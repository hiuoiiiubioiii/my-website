2. The Golden Rules of the Comma

The comma in C has a dual personality. It can act as a simple Separator or as an actual Operator.

Rule 1: Precedence
The comma operator has the LEAST (lowest) precedence of all operators in C. It is even lower than the assignment operator (=).
Rule 2: Associativity
It evaluates expressions from Left to Right.
Rule 3: Operator Functionality
When acting as an operator, it evaluates the first operand (left), rejects/discards its value, evaluates the second operand (right), and returns the right value.

3. Comma as a Separator

When declaring variables or passing arguments to functions, the comma is just a separator. It does not perform any operation.

int a, b, c;
int a = 10, b = 5, c = 10;

Here, it merely tells the compiler that you are listing multiple variables of the same data type.

4. Concept: Precedence Clash (No Brackets)

What happens when we use the comma as an operator alongside the assignment operator without any parentheses?

int a;
a = 5, 4;
  • We have two operators: = and ,
  • Because = has higher precedence, a = 5 executes first!
  • The comma operator then looks at 4. It evaluates it, but the value is rejected because it isn't assigned to anything.
Memory: Variable a 5

5. Concept: The Power of Parentheses

Let's add parentheses to the exact same code. Parentheses have the highest precedence.

int a;
a = (5, 4);
  • The expression inside () evaluates first.
  • Left operand (5): Evaluated and discarded.
  • Right operand (4): Evaluated and returned.
  • The expression becomes: a = 4;
Memory: Variable a 4

6. Concept: The Declaration Trap

Watch out for this very common syntax error!

int a = 5, 4; // ERROR!
Why does this throw an error?
Because this line begins with a data type (int), the compiler treats the comma as a separator. It creates a=5, sees the comma, and expects another valid variable name (like b). Finding a hardcoded 4 instead causes a compilation error!

The Fix: Use parentheses: int a = (5, 4);

7. Concept: Chaining and Functions

The comma operator can chain multiple expressions. Let's see how it behaves with functions like printf.

int a;
a = (printf("Jenny"), 2, 3, 4);
  • Associativity is Left-to-Right.
  • Step 1: printf("Jenny") executes. "Jenny" prints to the screen. Its return value is discarded.
  • Step 2: 2 is evaluated and discarded.
  • Step 3: 3 is evaluated and discarded.
  • Step 4: 4 is the right-most value. It is returned.
Console Output Jenny
Memory: a 4

8. Concept: Increment Operations & Comma

Let's trace how the comma works with post and pre-increment operators. This is highly tested in exams!

Example A: With Parentheses

int a = 8, b;
b = (a++, ++a);
  • () evaluated first. Left to right.
  • a++ (Post-increment): Current value 8 is evaluated but discarded. Memory a becomes 9.
  • ++a (Pre-increment): Memory a increments to 10. The value 10 is evaluated and returned.
  • b = 10.
Memory: a 8, 9 10
Memory: b 10

Example B: Without Parentheses

int a = 8, b;
b = a++, ++a;
  • Precedence alert! = is evaluated before ,.
  • b = a++ executes first. b gets the original value of a (8). Then a becomes 9.
  • The comma moves execution to ++a. a becomes 10. Value is discarded.
Memory: a 8, 9 10
Memory: b 8
