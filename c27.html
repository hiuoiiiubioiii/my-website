Control Statements in C

A complete breakdown of decision-making and the if statement.

1. Why Do We Need Control Statements?

By default, a C program executes in a Sequential Flow. The compiler reads and executes line 1, then line 2, then line 3, strictly from top to bottom.

Sequential Flow Example:

void main() {
    int a, b, sum;
    printf("Enter a & b: ");
    scanf("%d %d", &a, &b);
    
    // Executes no matter what:
    sum = a + b;
    printf("Sum = %d", sum);
    printf("End of program");
}

The Problem: What if you don't want every line to execute? What if you only want to calculate the sum if both 'a' and 'b' are equal to 1? To do this, we need to control the flow of execution based on specific conditions.

🛒 The Supermarket Analogy

Imagine your mom sends you to the market. She gives you conditional instructions:

  • Condition 1: "Buy oranges."
  • Condition 2: "If oranges aren't there, buy kinnow."
  • Condition 3: "If kinnow isn't there, buy mosambi."

You make decisions based on what is available. Control statements allow computers to make these exact same logical choices!

2. Types of Control Statements

Control statements are also called Decision-Making Statements or Selection Statements. They allow us to skip or execute blocks of code.

3. Deep Dive: The Simple `if` Statement

The `if` statement evaluates a condition. If the condition is true, it executes the code block. If false, it skips it entirely.

General Syntax

if (condition) 
{
    // if statement block
    // (executes ONLY if condition is true)
}
// after if statements 
// (executes normally, regardless of the condition)

4. Code Execution & Tracing

Let's trace a real program. First, an important rule about booleans in C:

🧠 "True" vs "False" in C:

  • 0 (Zero) is considered FALSE.
  • ANYTHING ELSE (1, 5, -10, etc.) is considered TRUE.
void main() {
    int a;
    printf("Enter a: ");
    scanf("%d", &a);
    
    if (a) {
        printf("Inside if block \n");
    }
    printf("Out of if block");
}

Output Tracing:

Scenario A: Input is 5
Enter a: 5
Inside if block
Out of if block

Why? `if(5)` is evaluated as TRUE. Block executes.

Scenario B: Input is 0
Enter a: 0
Out of if block

Why? `if(0)` is evaluated as FALSE. Block is completely skipped.

⚠️ Exam Trick: The Curly Braces Rule

Instructor's Golden Rule: If you do not use curly braces {}, the `if` statement only controls the very first statement immediately following it.

Look closely at this code (no curly braces):

if (a)
    printf("Inside if block \n");
    printf("Value of a = %d \n", a);

printf("Out of if block");
Output if user enters 0 (False)
Enter a: 0
Value of a = 0
Out of if block

Even though the condition was false, Value of a = 0 printed! Because there were no braces, the compiler assumed only the very first `printf` belonged to the `if` statement. The second `printf` executed normally as part of the sequential flow.

5. Tricky Interview & Exam Questions

Professors love putting these on exams to test your deep understanding.

Trick #1: The Accidental Semicolon

if (a); 
{
    printf("Inside if block");
}

What happens? It will NOT give a syntax error. A semicolon ; means "end of statement." Putting it directly after the `if` terminates the statement right there. The code block beneath it is no longer attached to the condition; it will execute every single time, regardless of what `a` is.

Trick #2: Assignment vs. Equality (`=` vs `==`)

int age;
scanf("%d", &age); // User enters 15

if (age = 25) {
    printf("You can go get coffee with me.");
}

What happens? Even though the user inputs 15, the program will print "You can go get coffee with me."

Why? Inside the `if` statement, there is a single equals sign =. This is the assignment operator. The computer assigns the value 25 to `age`. The condition then evaluates the number 25. Because 25 is non-zero, it is TRUE.

Fix: You must use the double equals sign == to check for equality.

Trick #3: Logical NOT (`!`)

if (!age) {
    printf("Age is zero");
}

What happens? The ! operator inverts the boolean value.

  • If user enters 25 (True) → !25 becomes False (0). Block is skipped.
  • If user enters 0 (False) → !0 becomes True (1). Block executes!
