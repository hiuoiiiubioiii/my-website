Tricky while Loops in C

In-Depth Masterclass Feed based on the Video Lecture

⚠️ The Golden Rule of C:
• 0 (Zero) means FALSE.
• Any non-zero value (1, 2, -10, 100) means TRUE.

1. The Standard Format Basics

while (condition) 
{
    // Body of loop
    statements;
    
    // Modify/Update Expression
    i++; 
}
Instructor Note: Initialization happens before the loop. The condition is checked at the start. Modification happens inside the body. In a for loop these are on one line, but in a while loop, they are spread out.

2. The Equality Operator (==) Normal

int i = 1;
while (i == 1) 
{
    printf("%d ", i);
    i++;
}
printf("\nEnd of Program");
1 ➔ 2

Condition 2 == 1 becomes False. Loop breaks.

3. The Assignment Trap (=) Interview Favorite

int i = 1;
while (i = 1)  // Single '=' assigns 1 to i
{
    printf("%d ", i);
    i++;
}
Why is this an infinite loop?
A single = is an assignment, not a comparison. Inside the condition, i is assigned the value 1. The result of the expression is 1. Since 1 is non-zero, the condition is always TRUE. Even though i++ runs, the loop starts over and resets i back to 1!
i is always 1
1 1 1 1 1 1 1 1... (Infinite)

4. Hardcoded True & Variable Wrapping Advanced Range

int i = 10;
while (1) // 1 is always TRUE
{
    printf("%d ", i);
    i++;
}
Does it print to infinity mathematically? No! Variables have memory limits. A 16-bit signed integer ranges from -32768 to 32767. When it hits the maximum positive number, adding 1 causes an overflow and "wraps around" to the lowest negative number.

5. Using a Variable as the Condition Tricky Halt

int i = 10;
while (i) // Same as while(i != 0)
{
    printf("%d ", i);
    i++;
}
This operates like the wrapping circle above. It counts up to 32767, wraps to -32768, and starts counting up toward 0.

The Twist: When i increments from -1 to 0, the condition becomes while(0). Since 0 is FALSE, the loop finally stops!
10 11 ... 32767 -32768 ... -2 -1
End of Program

6. Pre vs Post Increment in Conditions Logic Test

Post-Increment

int i = 0;
while (i++) { printf("%d ", i); }
Post-increment: Use the value, then increment.
1. Checks condition: while(0) ➔ FALSE.
2. Increments i to 1, but skips the loop body.
Output: Nothing prints!

Pre-Increment

int i = 0;
while (++i) { printf("%d ", i); }
Pre-increment: Increment first, then use.
1. Increments i to 1.
2. Checks condition: while(1) ➔ TRUE.
3. Loop runs until wrapping brings it back to 0.

7. Character Data Type Wrapping ASCII Concepts

char ch = 'a'; // ASCII 97
while (ch) 
{
    printf("%d ", ch);
    ch++;
}
Characters are stored as integers behind the scenes. Signed char ranges from -128 to 127.
Starts at 97, goes to 127, wraps to -128, counts up to -1, and stops when ch++ hits 0.
97 ➔ 98 ➔ ... ➔ -1 ➔ 0 (STOP)
97 98 ... 127 -128 -127 ... -1

8. Homework / Edge Cases Errors & Traps

Trap 1: The Deadly Semicolon

char ch = 'a';
while (ch);  // LOOK CLOSELY!
{
    ch++;
}
Placing a semicolon right after the while condition creates an empty loop body. It means while(ch) { nothing }. Since ch never changes inside that empty body, it causes a silent, frozen infinite loop on line 2!

Trap 2: Missing Condition

while () { i++; } // Compilation Error!
Unlike a for(;;) loop, a while loop must have a condition inside the parentheses. Leaving it blank throws a syntax error.
