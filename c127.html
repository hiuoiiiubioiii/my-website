Understanding fseek() in C

Complete visual notes based on Jenny Khatri's Lecture

🧐 1. The Problem: Sequential vs. Random Access

Up until now, you have learned to read/write files sequentially. If a file contains "Jenny Khatri is awesome" and you open it in read mode ("r"), the cursor automatically starts at the very beginning (at the 'J').

The Issue: What if you only want to read the word "Khatri"? You shouldn't be forced to read "Jenny " first.

Instructor Note: You might know the rewind() function. However, rewind() strictly moves the cursor from its current position back to the absolute beginning of the file. It is not flexible. To move anywhere we want, we need fseek().

🛠️ 2. Syntax & Arguments

The fseek() function forcefully moves the file pointer (cursor) to a specific position so you can read, write, or update exactly where you want.

int fseek(FILE *file_pointer, long int offset, int position);

Breakdown of the 3 arguments:

  • FILE *file_pointer: The pointer to the file you currently have open (e.g., fp).
  • long int offset: How many bytes (characters) you want to move. Use a positive value to move forward, and a negative value to move backward.
  • int position: The starting point from which you apply your offset.

Return Value: It returns 0 on success, and a non-zero value if it fails.

📍 3. The Three Position Macros

For the third argument (position), C provides three predefined macros inside the <stdio.h> library to tell the cursor where to start counting from:

  • SEEK_SET: Represents the beginning of the file (Index 0).
  • SEEK_CUR: Represents the current position of the cursor.
  • SEEK_END: Represents the end of the file.

💻 5. Practical Coding Session (VS Code)

Here is the complete C program the instructor writes to prove the logic from the whiteboard.

#include <stdio.h>
#include <stdlib.h>

void main() {
    FILE *fp = NULL;
    char ch;

    // Open file in read mode
    fp = fopen("abc.txt", "r");

    if (fp == NULL) {
        printf("error: Can't open file");
        exit(1);
    }

    /* TEST 1: Move 6 bytes forward from beginning */
    fseek(fp, 6, SEEK_SET); 
    ch = fgetc(fp);          // Reads 'K', cursor moves to index 7
    printf("%c\n", ch);      // Output: K

    /* TEST 2: Move 3 bytes backward from current pos (7) */
    fseek(fp, -3, SEEK_CUR); 
    ch = fgetc(fp);          // Reads 'y'
    printf("%c\n", ch);      // Output: y

    /* TEST 3: Move 3 bytes backward from the end */
    fseek(fp, -3, SEEK_END); 
    ch = fgetc(fp);          // Reads 'l' (from "faculty")
    printf("%c\n", ch);      // Output: l

    fclose(fp);
}

⚠️ 6. The "Gotcha": File Modes & Updating

Using fseek() to read is easy. But what if we want to update the file? Suppose we want to overwrite the "Kh" in Khatri with "hi" (Making it hiatri).

fseek(fp, 6, SEEK_SET); 
fputs("hi", fp);

The code above looks correct, but your file mode determines if it actually works:

❌ Mode "r" (Read Only): Fails. fputs() is a write command. You cannot write to a file opened in read mode.
❌ Mode "w+" (Write/Update): Fails logically. The "w+" mode instantly erases the entire contents of the file the second it opens. You cannot "seek" to index 6 because indices 0 through 6 no longer exist!
✅ Mode "r+" (Read/Update): The Correct Answer! This mode allows reading, seeking, and writing without erasing the existing contents first.
