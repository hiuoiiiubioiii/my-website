Arrays in C: Initialization

Instructor: Jayanti Khatri Lamba | Topic: 1D Arrays

1. Introduction

Once you have declared an array, you need to put data into it. The instructor outlines that there are two ways to initialize an array in C:

  • At Compile-Time: Values are hardcoded into the program when you write the code.
  • At Run-Time: Values are entered by the user while the program is actually running.

2. Compile-Time Initialization

You initialize the array at the exact moment you declare it in your code. You define exactly what numbers go into the array before compiling. The instructor demonstrates 8 variants on the whiteboard.

Variant 1: Standard Initialization Correct ✅

Declare size and provide the exact number of elements.

int a[5] = {0, -1, 11, 10, 2};

Variant 2: Omitting the Size Correct ✅

If you leave the brackets [] empty but provide values, the compiler automatically counts the elements (7 here) and sets the size behind the scenes.

int a[] = {0, 1, 2, 0, -1, 6, 7};

Variant 3: Partial Initialization Correct ✅

You declared space for 5 items but only provided 3. The remaining empty slots are automatically filled with zeros.

int a[5] = {0, 1, -1};

If you declare an array but do not initialize it, the memory slots are filled with unpredictable "Garbage Values".

Variant 5: Too Many Elements Error ❌

You declared a size of 5 but tried to force 7 elements into it. The compiler throws an error.

int a[5] = {1, 2, 3, 4, 5, 6, 7}; // ERROR

Variant 6: Individual Index Initialization Correct ✅

Declaring first, then assigning index by index. (Valid, but tedious and not preferred).

int a[5];
a[0] = 1;
a[1] = 2;
a[2] = -1;
a[3] = 11;
a[4] = 10;

Variant 7: The "All Zeros" Shortcut Correct ✅

Relying on the rule from Variant 3, providing a single 0 fills the rest of the array with zeros. A great trick to clear garbage values!

int a[5] = {0};

Variant 8: Empty Braces Error ❌

You cannot leave braces completely blank. You must provide at least one data item.

int a[5] = {}; // ERROR

📝 Note on Other Data Types

This syntax works identically for other types, like float or char. Note that characters must use single quotes.

char b[] = {'j', 'e', 'n', 'n', 'y'};

3. Run-Time Initialization

Instead of hardcoding numbers, ask the user to type them in while the program is running. Because you need to accept multiple inputs sequentially, you must use a Loop combined with the scanf function.

The Code:

int a[5];
int i; // Loop counter

printf("Enter the elements of array: ");

for(i = 0; i < 5; i++) {
    scanf("%d", &a[i]);
}
Whiteboard Visualization: Memory & The Loop

When the code runs, memory is allocated. Integers take 4 bytes, so the array addresses jump by 4. scanf looks at the current value of i to know which address to store the user's input in.

Tracing the Loop:

  • Loop 1: i = 0. scanf stores input at &a[0] (Address 2000). User types "10", it goes into slot 0.
  • Loop 2: i++ makes i = 1. scanf stores input at &a[1] (Address 2004). User types "11", it goes into slot 1.
  • Repeats until i reaches 5, then the loop terminates.

Note: To print the array back to the screen, use the exact same for loop, but replace scanf with printf("%d", a[i]); (removing the & symbol).

4. When to Use Which Method?

Use Compile-Time Initialization when: The array size is very small (like 5 or 10 elements) AND you already know exactly what the values are going to be.

Use Run-Time Initialization when:

  • The array is large (e.g., int a[100];). You don't want to type 100 numbers inside {} manually.
  • You want the program to be dynamic (different users will enter different data).
  • You want to populate an array based on specific logic or rules.

Advanced Run-Time Logic Example

Imagine you have an array of 100 slots. You want the first 30 slots to hold the number 1, and the remaining 70 slots to hold the number 0. You can place if/else statements inside your run-time loop to build complex arrays instantly:

int a[100], i;

for(i = 0; i < 100; i++) {
    if (i < 30) {
        a[i] = 1;
    } else {
        a[i] = 0;
    }
}
