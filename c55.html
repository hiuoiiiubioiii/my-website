2-D Arrays in C

Memory Representation & Accessing Elements

📝 1. Declaration & Initialization

A 2-D array is essentially a matrix containing rows and columns. In C, we declare it by specifying the data type, the name of the array, the number of rows, and the number of columns.

int a[2][3] = { {1, 2, 0}, {-1, 5, 4} };
  • int : The data type (Integer).
  • a : The name of our array.
  • [2] : Row size (It will have Row 0 and Row 1).
  • [3] : Column size (It will have Col 0, Col 1, Col 2).

Instructor Note: The inner braces `{}` group the data by rows. If you leave a space empty (like writing {1, 2} instead of {1, 2, 0}), the compiler automatically fills the empty spot with a 0.

💻 3. How the Computer Actually Stores It

Here is the core concept: Computer memory is linear. It is not a 2D grid; it's a long, continuous tape of memory slots. So, how does the computer fit our matrix into a straight line?

It uses Contiguous Memory Allocation and stores the data using Row-Major Order. This means it lays down entire rows back-to-back.

  • First, it stores all elements of Row 0.
  • Immediately after, it stores all elements of Row 1.

Understanding the Memory Math:

Assume the Memory Manager assigns a Base Address of 100 to the very first element. Because an int data type takes up 4 bytes of memory in this compiler, the address increases by 4 for every single element sequentially (100, 104, 108, 112...).

Conclusion: A 2-D array is simply an "Array of 1-D Arrays".

🎯 4. Accessing Specific Elements

To grab a specific piece of data via code, you must provide two indices: one for the row (usually variable i) and one for the column (usually variable j).

// Syntax to access an element
arrayName[rowIndex][columnIndex];

Examples from the whiteboard:

  • To print the 0 (Top right of the matrix):
    printf("%d", a[0][2]);
  • To access the -1:
    a[1][0] (Row 1, Column 0)
  • To access the 5:
    a[1][1] (Row 1, Column 1)

🧮 5. Address Calculation (Teaser)

If you ask the computer for a[1][0], how does it instantly know to look at memory address 112?

For a standard 1-D Array, the compiler uses a simple formula:

Address = BaseAddress + (Index * Size_of_DataType)

However, for a 2-D array, it's more complex because the formula must account for both the Row Index i and Column Index j, as well as the total number of columns in the array.
