C Loops Masterclass

A complete visual breakdown of for, while, and do-while loops

1. Type of Control (The Checkpoint)

The most fundamental difference between the three loops is when the program checks the condition. Are you checked at the door, or are you checked as you leave?

Entry-Controlled
for while

Condition is checked BEFORE entering the loop.

1. Check Condition
↓
2. Execute Body of Loop

Exit-Controlled
do-while

Condition is checked AFTER executing the loop once.

1. Execute Body of Loop
↓
2. Check Condition

2. Syntax & Structure

How we write them on the whiteboard (and in the IDE). Look closely at where the pieces live.

Instructor Warning: Always write C keywords (for, while, do) in strictly lowercase letters. Using capital letters will throw an error!

A. The for Loop

Everything is packed neatly into one line. The most popular loop for programmers!

for (initialization; condition; update) 
{
    // Body of loop
    Statement 1;
}

B. The while Loop

Initialization happens before the loop, and updates happen inside the body.

initialization;
while (Condition) 
{
    // Body of loop
    Statement 1;
    update;
}

C. The do-while Loop

Crucial Syntax Rule: You must put a semicolon ; at the very end of the while(condition) statement. Do not put it after the curly brace! 
initialization;
do 
{
    // Body of loop
    Statement 1;
    update;

} while (Condition); <-- Do not forget this semicolon!

3. The "False Condition" Test

What happens if the initial condition is FALSE on the very first try? Will the body of the loop execute?

for loop

0

Executes Zero Times.
Because it is entry-controlled, it never enters the body.

while loop

0

Executes Zero Times.
Same as the for loop. The door is locked immediately.

do-while loop

1

Executes At Least Once.
It runs the "do" block before it even realizes the condition is false!

4. When to Use Which Loop?

When to use for

Rule: Used when the exact number of iterations is known.

Example: You want to print numbers from 1 to 10. You know definitively that this requires exactly 10 iterations. You use a for loop.

When to use while

Rule: Used when the number of iterations is NOT exactly known.

Example: You want to keep asking a user to enter an integer, and you only want the program to stop when they enter a 0. User A might enter 6 numbers before typing 0. User B might type 0 on the second try. You cannot predict it, so you use a while loop.

When to use do-while

Rule: Used when iterations are unknown, AND the body of the loop must be executed at least once.

Example: A menu-driven program (like an ATM). You must show the user the menu at least one time before they can choose the "Exit" option.

💡 Pro-Tip: The Infinite Loop

During the user-input example, the instructor introduced the most famous way to write an infinite loop in C programming:

while (1) 
{
    printf("Enter an integer (0 to stop): ");
    scanf("%d", &number);

    if (number == 0) {
        break; // This breaks the infinite loop!
    }
}

How it works: In C, 1 represents "True". So while(1) means "While True". It will run forever unless you put an if condition paired with a break; statement inside the loop body to force the program out!
