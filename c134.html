Dynamic Memory Allocation: calloc()

Complete Study Notes & Visualizations from the Whiteboard Session

1. Introduction & Basic Definition 📖

Building on our knowledge of malloc(), we are now looking at calloc(). Both are used for Dynamic Memory Allocation (DMA) at runtime, saving memory on the heap.

  • Full Form: Contiguous Allocation.
  • Location: It is a built-in function found inside the <stdlib.h> header file.
Instructor Says: "calloc() is used to dynamically allocate multiple blocks of memory, and each block is exactly the same size. This is the first major difference from malloc(), which allocates one single large block."

2. Syntax & Arguments ⚙️

Unlike malloc which takes only 1 argument, calloc requires two arguments.

void* calloc(size_t n, size_t size);
  • size_t: Means an unsigned integer (cannot take negative values, only 0 or positive).
  • Argument 1 (n): The total number of blocks (or elements) you want to store.
  • Argument 2 (size): The size in bytes of each individual block.

Writing the Code Step-by-Step

If we want to allocate memory for an array of 5 integers, we do it like this:

int *ptr = (int *)calloc(5, sizeof(int));
  1. We ask calloc for 5 blocks.
  2. Each block is the size of an integer (sizeof(int), usually 4 bytes).
  3. calloc returns the base address (the address of the very first byte) as a void * (void pointer), because it doesn't know what data type you will store yet.
  4. We use Typecasting (int *) to tell the compiler: "Treat this address as a pointer to an integer."

The Biggest Difference (Initialization): calloc automatically initializes all the memory it creates to 0. malloc does not initialize the memory, leaving leftover garbage values inside it.

Performance Note: Because calloc takes the extra step to write zeroes into the memory, it has a slight "overhead" and is technically a bit slower than malloc. Use malloc if you plan to immediately overwrite the values. Use calloc if you specifically need the memory to start out empty/zeroed.

4. Static vs. Dynamic Allocation ⚖️

During the lesson, the instructor pointed out a quick distinction on the whiteboard:

  • int *ptr; ➔ This is Static Memory Allocation (SMA). The memory for the pointer itself is allocated at compile time on the Stack.
  • ptr = (int *)calloc(...); ➔ This is Dynamic Memory Allocation (DMA). The memory block it points to is allocated at runtime on the Heap.

5. Practical Code Demo (VS Code) 💻

To prove the zero-initialization concept, the instructor ran the following code. She specifically commented out the scanf input loop to see what was inside the memory before the user touched it.

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;
    int n, i;

    printf("Enter total no of values: ");
    scanf("%d", &n);

    // Using calloc to allocate memory based on user input
    ptr = (int *)calloc(n, sizeof(int));

    /* 
    User input code is COMMENTED OUT. 
    We want to print the default memory state!
    */

    printf("\nThe entered values are: \n");
    for(i=0; i<n; i++) {
        // Printing what is currently in the memory
        printf("%d\t", *(ptr+i));
    }

    free(ptr);
    return 0;
}

Live Execution Results:

  • Test 1 (Using calloc): She inputs 3. The output printed is 0 0 0. Proving `calloc` initializes to zero.
  • Test 2 (Using malloc): She swaps `calloc` for `malloc` and runs it. The output is 7472000 349102 -102948. Proving `malloc` leaves garbage values.
