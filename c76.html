Pointer Arithmetic: Subtraction

Complete Class Notes, Visualizations, & Formulas

📝 1. The "Rules" of Pointer Math

Before doing math with memory addresses, the compiler has strict rules on what operations are valid. Let p and q be two pointer variables.


p + q
❌ INVALID (Cannot add pointers)
p - q
✅ VALID (Subtracting pointers)
q - p
✅ VALID (Order changes sign)
p - 2
✅ VALID (Subtracting integer)

🧮 3. Deep Dive 1: Pointer - Pointer (q - p)

What happens if we calculate d = q - p;?
The Wrong Way: Standard math says 1012 - 1000 = 12. This is incorrect! The compiler wants to know how many elements apart they are, not how many individual bytes apart they are.

FORMULA:
(Address 1 - Address 2) / Size of Data Type

Calculating q - p:

Address of q: 1012

Address of p: 1000

(1012 - 1000) / 4 bytes = 12 / 4

Result: 3

💡 Key Takeaway: Subtracting two pointers returns an INTEGER representing the number of elements between them.

🔙 4. Deep Dive 2: Pointer - Integer (q - 2)

What happens if we calculate q = q - 2;?
Subtracting an integer means moving the pointer backward by that many elements.

FORMULA:
Current Address - (Integer Value * Size of Data Type)

Calculating q - 2:

Current Address of q: 1012

Move back 2 integers: 2 * 4 bytes = 8 bytes

1012 - 8

Result: Address 1004 (Index 1)

⚠️ Out of Bounds Warning: If you subtract too much (e.g., q = q - 4 from index 1), the pointer will point to an address outside the array (like 988). Accessing this address yields "Undefined Behavior" (garbage values).

🔗 5. The "Same Array" Rule

Rule: You can only subtract pointers if they point to the same array.

If p points to array_A and q points to array_B, the compiler will allow p - q syntactically. However, because you don't know how far apart these two arrays are stored in your computer's RAM, the result will be a completely meaningless, garbage number.

💻 7. Practical Computer Demo (VS Code)

Here is the exact code and output demonstrated in the IDE to prove the math.

#include <stdio.h>

void main() {
    int a[] = {2, 4, 3, 0, -7};
    int *p = a;         // index 0
    int *q = &a[3];     // index 3

    printf("%d\n", q - p);
    printf("%d\n", p - q);

    q = q - 2;          // moves q to index 1
    printf("value = %d\n", *q);

    q = q - 2;          // moves q out of bounds (index -1)
    printf("value = %d\n", *q);
}

📌 Cheat Sheet: Pointer Math Rules

Keep these rules memorized for exams!

  • Pointer + Pointer ERROR (Invalid)
  • Pointer - Pointer INTEGER (Elements between them)
  • Pointer + Integer POINTER (Moves forward)
  • Pointer - Integer POINTER (Moves backward)
