Pointer Assignment in C

Scroll down to follow the lecture. Learn how to assign addresses vs. values using pointers.

1. Initial Setup & Memory Visualization

We begin by setting up standard integer variables and declaring our pointers. To understand pointers, you must visualize how they look in the computer's memory.

int a = 10, b = 11;
int *p, *q;
⚠️ Instructor Clarification:

In the declaration line int *p, *q;, the asterisk (*) is NOT acting as a dereferencing/indirection operator. It simply tells the compiler: "These variables are pointers that will hold memory addresses."

Here is what the memory looks like right now (assuming hypothetical addresses):

Addr: 1000
b
11
Addr: 1046
p
Garbage
Addr: 2000
q
Garbage
Addr: 3000

2. Direct Pointer Assignment (Address Copy)

Now, let's assign the address of variable a to pointer p, and then assign p directly to q.

p = &a; 
q = p;  // Direct Assignment

What is happening? You are taking whatever value is currently stored inside p (the address 1000) and copying it into q. Now, both pointers are pointing to the exact same variable (a).

a
10
Addr: 1000
p
1000
Addr: 2000
👉 Points to 'a'
q
1000
Addr: 3000
👉 Points to 'a'

If we run printf("a = %d %d %d", a, *p, *q);, the output will be:

Console: 10 10 10

3. The Danger Zone (Uninitialized Dereferencing)

What happens if we alter the code and try to assign values instead of addresses, but forget to initialize q first?

p = &a;
*q = *p; // DANGER! 'q' has no address yet

The right side (*p) works fine; it grabs the value 10. But the left side tells the computer: "Take this 10 and store it at the memory address currently held inside q."

p
1000
👉 Points to 'a'
q
?@#*
💥 Points to nowhere!
💥 Program Crash (Segmentation Fault)

Because q holds a random garbage value, the computer attempts to access a random, illegal memory location to store the '10'. The program instantly crashes and stops working.

4. Value Assignment Using Pointers

To fix the crash, we must ensure q is pointing to a valid memory location before we dereference it. Let's point it to b.

p = &a;
q = &b;
*q = *p; // Overwriting the value inside 'b'

This is Value Assignment. We are saying: "Go to the address inside q (which is b's address, 1046) and overwrite whatever is there with the value found at the address inside p (which is 10)."

a
10
Addr: 1000
b
1110
Addr: 1046
p
1000
👉 Points to 'a'
q
1046
👉 Points to 'b'

If we print a, *p, *q now, the output is:

Console: 10 10 10

5. VS Code Demo: Pointer Data Type Rules

In the final part of the lecture, the instructor demonstrates a vital rule in her IDE. She changes the pointer declarations to float, but leaves the variables as int.

float *p, *q;
int a = 10, b = 11;

p = &a; // IDE Warning generated here!
🚨 Compiler Warning: Incompatible Pointer Type

A pointer's data type must strictly match the data type of the variable whose address it is holding. You cannot store the address of an int inside a float *.

To fix this, she changes variables a and b to floats, and uses %f in the printf statement, resulting in a successful output of 10.000000.
