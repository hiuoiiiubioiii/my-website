Relational Operators in C

Scroll down to learn, trace memory, and execute code.

💡

1. Core Concepts

Relational operators (also known as Comparison Operators) are used to check the relationship between two values. They are heavily used in decision-making (like if-else) and loops.

The result of a relational operation is always a boolean value:

True = Stored as 1
False = Stored as 0

Since the output is stored as an integer in memory, we use the %d format specifier to print the result in C.

📋

2. The 6 Operators

Here is what the instructor wrote on the board for the types of relational operators:

<   (Less than)
>   (Greater than)
<=  (Less than or equal to)
>=  (Greater than or equal to)
==  (Equal to / Equality operator)
!=  (Not equal to / Non-equality)

⚠️ Crucial Distinction!

= (Single equals) is an Assignment Operator. It assigns a value to a variable.

== (Double equals) is a Relational Operator. It compares two values.

🔤

3. Comparing Different Data Types

You can compare integers, floats, and characters. You can also compare full expressions.

A. Comparing Characters

C compares characters using their integer ASCII values.

'a' < 'b'
↓       ↓
97 < 98  ➡️ 1 (True)

B. Comparing Strings (The Danger Zone)

You CANNOT compare strings using relational operators in C!

"Jenny" < "lectures"
Compares Base Memory Addresses!
[Addr: 1000] < [Addr: 2000]

Because strings are stored at random memory addresses, comparing them this way yields unpredictable garbage results. You must use functions like strcmp().

⚙️

4. General Syntax

Relational operators don't just work on single numbers. You can compare entire mathematical expressions on either side.

[ AE1 ]   Relational Operator   [ AE2 ]

(AE stands for Arithmetic Expression)

Example: a + b < c + d
Example: -14 > 3 → 0

Precedence & Associativity

Rule 1: Arithmetic operators (+, -) have higher precedence than Relational operators. Solve math first!

Rule 2: Relational operators have Left-to-Right Associativity.

6. Tricky Edge Cases

What if you run: f = a > b > d < c; ?

18 > 10 > 10 < 9
1 > 10 < 9
0 < 9 → Stores 1 in f

What if you run: printf("%d", '+' == '+'); ?

The compiler checks ASCII values: 43 == 43 → Prints 1
(Even '<' > '>' is valid syntax because it compares underlying integers!)
🎯

7. The Floating-Point Trap

float f = 2.2;
float g = 2.22;

Comparing f < g works perfectly fine. However...

Avoid `==` with Floats!

If you have high precision floats (e.g., 2.0000001 vs 2.000001), the computer stores them as approximations. Two floats that look identical might be mathematically evaluated as slightly different by the computer, causing == to return False unexpectedly.

📝

8. Assignment

Given the final memory state from earlier:

a = 18, b = 10, c = 9, d = 10, e = 10

Evaluate this expression exactly as the computer would:

a + c == b >= e < c + d != 1

Step-by-step logic:

  1. Insert values: 18 + 9 == 10 >= 10 < 9 + 10 != 1
  2. Arithmetic first: 27 == 10 >= 10 < 19 != 1
  3. Relational (>=, <) Left-to-Right: 27 == 1 < 19 != 1
  4. Next Relational: 27 == 1 != 1
  5. Equality (==, !=) Left-to-Right: 0 != 1
  6. Final Evaluation: 1

Final Output: 1 (True)
