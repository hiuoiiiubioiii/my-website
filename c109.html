1. The Progression of Data Storage

Before understanding what a structure is, we must understand why we need them. Let's look at how we store data in C.

A. Primitive Variables

If you want to store a single value, you use a standard variable.

int a = 5;

B. Arrays

If you want to store the roll numbers of 60 students, creating 60 individual variables is inefficient. We use arrays to store a collection of homogeneous (same type) data items.

int rollno[60];

2. The Real-World Problem

In real-world software, data is rarely just one type. A "Student" isn't just a number. A student consists of Heterogeneous Data (mixed types):

  • rollno ➔ Integer
  • name ➔ String (char array)
  • marks ➔ Decimal (float)

Attempt 1: Individual Variables (The Messy Way)

If we try to store just 2 students without structures, we need 6 variables. For 60 students, we'd need 180 variables!

Student 1
int rollnoS1;
char nameS1[20];
float marksS1;
Student 2
int rollnoS2;
char nameS2[20];
float marksS2;

Attempt 2: Parallel Arrays

We could create three separate arrays: rollno[60], name[60][20], and marks[60]. But the data is scattered. The solution? Group them under one single name.

3. What is a Structure? & Syntax

A structure is a user-defined data type. You take primitive data types (int, char, float) and group them together under a single name.

struct Student 
{
    int rollno;
    char name[20];
    float marks;
};

Syntax Breakdown:

  • struct : The mandatory C keyword.
  • Student : The Structure Tag (your new data type name).
  • int rollno; ... : The Members of the structure.
  • }; : The closing brace and mandatory semicolon.

4. Crucial Rules for Declarations ⚠️

The instructor explicitly highlights common mistakes. Pay close attention!

Rule 1: Semicolon is Mandatory

You MUST put a semicolon ; after the closing curly brace }.

Rule 2: No Duplicate Members

Two members inside the exact same structure cannot share the same name.

struct Student {
    int rollno;
    float rollno; // ❌ ERROR!
};

Rule 3: NO Initialization Inside

You cannot assign values to variables when declaring the structure.

struct Student {
    int rollno = 1; // ❌ ERROR!
};

5. Memory Allocation (Why Rule 3 exists)

When you write struct Student { ... };, NO memory is allocated in the computer. You are just creating a template/blueprint. Because no memory exists yet, you cannot store a value like 1 inside it!

⚙️ struct Student (BLUEPRINT - 0 Bytes)

Tells compiler: "When I create a student, it will look like this."

Creating an Object

To actually store data, you must create a variable (an object) of your new data type in the main() function.

void main() {
    int a;              // Primitive (4 bytes)
    struct Student s1;  // Object (Memory allocated here!)
}

How much memory does `s1` take?

📦 struct Student s1 (Total: 28 Bytes)
int rollno; 4 Bytes
char name[20]; 20 Bytes
float marks; 4 Bytes

4 + 20 + 4 = 28 bytes of memory allocated consecutively.
