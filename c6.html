Writing the Code

The Source File

The journey begins when you write your program using an IDE (like VS Code) or a text editor (like Notepad).

  • This code is written in a High-Level Language (English-like).
  • You must save the file with the .c extension.
  • This original file is known as your Source Code.

2

Preprocessing

Handling the # directives

The compiler doesn't understand lines that start with a hashtag (#). The Preprocessor runs first to handle these.

#include <stdio.h>
#include <conio.h>
#define max 100
  • Expansion: It takes the declarations from header files (like stdio.h) and replaces the #include line with the actual code.
  • Macro Replacement: It finds every instance of words like max and replaces them with the value (100).
  • Cleanup: It removes all # lines and strips away all comments.

3

Compilation

Checking Syntax & Translating

The Compiler takes the expanded .i file and checks it for grammar and syntax rules.

  • Syntax Error Loop: If there is an error (e.g., missing semicolon), execution stops here. You must go back to Step 1 and edit your source code.
  • Translation: If no errors are found, the compiler translates the high-level C code into Assembly Language.
  • Assembly uses low-level mnemonics (like ADD, MOV) but is still text-based.

↓Syntax Error? (If yes, stop)

4

Assembly

Converting to Machine Code

Computers only understand binary (0s and 1s). The Assembler bridges the final gap between text and machine language.

  • It translates the .asm file into an Object File.
  • This file contains binary machine code.
  • Crucial Note: You cannot execute this file yet! It is incomplete because it doesn't know the definitions of built-in functions.
first.asm
↓
Assembler

Object File
first.obj (or .o)
5

Linking

The Missing Piece: Libraries

Your object file knows you called functions like printf() or scanf(), but it doesn't have the code defining how they work. The definitions live in C's System Libraries.

  • The Linker combines your first.obj file with the necessary System Libraries.
  • If you wrote a large program across multiple files, the Linker merges all those object files together.
  • Result: A fully complete, ready-to-run Executable File.

Loading & Execution

From Hard Drive to CPU

Your .exe file is saved on your hard disk. However, the CPU can only execute programs that are in Main Memory (RAM).

  • Loader: A program that takes the .exe from your hard disk and loads it into the RAM.
  • Execution: The CPU begins processing the instructions starting exactly at the main() function.
  • During execution, you might provide Input Data.
  • If a Runtime/Logic Error occurs here, the program crashes, and you must debug your source code. If all is well, you get your Output!
