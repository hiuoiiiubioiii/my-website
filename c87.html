In-Depth Breakdown: Call by Value vs. Call by Reference

📌 1. Introduction to Function Calling

Before diving into memory logic, let's recap the basics of function calling in C. To safely use a function, you must either define it before calling it, or declare it at the top of your program.

Ways to Pass Arguments

You can pass arguments to a function in two ways:

  • Direct Values: Sum(5, 7);
  • Variables: int x = 5, y = 7; Sum(x, y);

When dealing with variables, we encounter two distinct methods of passing data: Call by Value and Call by Reference.

📄 2. Deep Dive: Call by Value

In this method, you pass the actual values (or a copy of the variables) to the function. Let's look at the code written on the whiteboard.

void fun(int x, int y); // Function Declaration

void main() {
    int x = 5, y = 7;
    fun(x, y);          // Function Call (Actual Parameters)
    printf("x=%d, y=%d", x, y); 
}

void fun(int x, int y) { // Function Definition (Formal Parameters)
    x = 7;
    y = 5;
    printf("x=%d, y=%d\n", x, y);
}

🧠 How Memory Works Here (RAM Visualization)

When main() runs, it gets its own memory scope. When fun() is called, it gets a completely separate memory scope. The values from main are simply copied over.

x
5
Addr: 1000
y
7
Addr: 1046

*Original variables remain untouched.

fun() Memory Scope
x (copy)
5 7
Addr: 2000
y (copy)
7 5
Addr: 2046

*Changes happen only in this local scope.

The Drawback

Because fun() only alters the copies stored in its own local memory scope, the original variables in main() are never touched. Once fun() finishes executing, its memory vanishes. Therefore, the printf in main() will still output: x=5, y=7.

🔗 3. Deep Dive: Call by Reference

To fix the drawback of Call by Value, we use Call by Reference. Instead of passing a copy of the value, we pass the address in memory where the original value lives using pointers (*) and the address-of operator (&).

void fun(int *x, int *y); // Declaration updated with pointers

void main() {
    int x = 5, y = 7;
    fun(&x, &y);          // Passing Addresses instead of values
    printf("x=%d, y=%d", x, y); 
}

void fun(int *x, int *y) { // Receives memory addresses
    *x = 7;               // Modifies value at address directly
    *y = 5;
    printf("x=%d, y=%d\n", *x, *y);
}

🧠 Pointer Memory Visualization

This time, fun() creates Pointer Variables. The line *x = 7; literally translates to: "Go to the memory address stored in x (1000) and change the value at that exact location to 7."

⚠️ Common Student Mistake: If you change your function call to pass addresses (&x), you must also change the function declaration at the very top of your file to expect pointers (int *). If you leave it as void fun(int, int), the compiler will throw an error!

💻 4. Practical Demonstration (VS Code)

When the instructor runs this code live in Visual Studio Code, here is exactly what happens in the terminal.

Output: Call by Value

> inside fun(called function) x=7, y=5
> inside main(calling function) x=5, y=7

Proof: The values in main did not change.

Output: Forgetting the Declaration Update

If you switch to &x, &y but forget to update the declaration at the top:

error: conflicting types for 'fun'
note: expected 'int' but argument is of type 'int *'

Output: Correct Call by Reference

After fixing the declaration to void fun(int *x, int *y):

> inside fun(called function) x=7, y=5
> inside main(calling function) x=7, y=5

Proof: The pointers successfully reached into main's memory and permanently altered the original variables globally!

🎓 5. Key Takeaways Summary

  • Call by Value: Passes a copy of the variable. Changes made inside the function are local and do not affect the original variables. Safe to use when you just need to calculate something without altering original data.
  • Call by Reference: Passes the memory address of the variable using pointers (*) and address-of operators (&). Changes made inside the function alter the original variables in memory.
  • Terminology: Variables passed in the function call are Actual Parameters. Variables received in the function definition are Formal Parameters.
