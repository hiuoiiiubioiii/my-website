Storage Classes in C

Complete Visual Notes: The auto Class

1. What is a Storage Class?

When you declare a variable (e.g., int x = 10;), the data type (int) tells the compiler what type of value it holds. But there is hidden information associated with every variable, dictated by its Storage Class.

The 4 Hidden Properties:

Default ValueIf uninitialized, what value does it hold?
ScopeWhere in the code can it be accessed?
LocationWhere is it physically saved in memory?
LifetimeHow long does it stay alive before deletion?

The 4 Types of Storage Classes in C:

  1. Automatic (Focus of this lesson)
  2. Register
  3. Static
  4. External
2. The auto Storage Class

Crucial Rule: If you write an integer inside a function like int x = 10;, the compiler automatically assumes you mean auto int x = 10;. It is the default for local variables!

3. Scope & Lifetime Visualized (Whiteboard)

To prove how Scope and Lifetime work, the instructor drew nested blocks. A variable created inside a block dies when that block ends.

Main Function Scope (Outer Block)
auto int x = 10; // This 'x' belongs to the main function
Inner Block Scope
auto int x; // A completely new 'x'. Since it has no value, it gets a GARBAGE value!

printf("%d", x); // Prints GARBAGE VALUE.

// 🪦 The inner 'x' is DESTROYED here. Its lifetime is over.

printf("%d", x); // Prints 10, accessing the outer 'x' which is still alive!
4. Live Laptop Demonstrations

The instructor switched to VS Code to prove the theories. Here is exactly what she tested:

Test 1: Scope & Block Shadowing

    {

        printf("x=%d\n", x); // Prints inner block variable

    printf("%d", x);       // Prints outer block variable

Test 2: Default "Garbage" Value

She removed the `= 20` to see what happens when an auto variable is uninitialized.

    {

        printf("x=%d\n", x);
    }
    printf("%d", x);
    return 0;
}
Output:

Test 3: The Global Scope Error

She moved an `auto` variable outside of the `main()` function to the top of the script.

    ... 
}
Compiler Error:

Takeaway: `auto` variables are strictly local. They CANNOT be global!

5. Exam Cheat Sheet: `auto` Class
  • Keyword: auto
  • Default Nature: It is the default storage class for all local variables. (e.g., int x; is exactly the same as auto int x;).
  • Initial / Default Value: Garbage value (unpredictable numbers).
  • Scope: Local (Limited to the specific Block or Function where it was created).
  • Lifetime: Ends instantly when the block or function ends.
  • Memory Location: Stored in the RAM's Stack segment.
  • Global usage? Absolutely forbidden. Will trigger a compiler error.
