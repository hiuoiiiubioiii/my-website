Part 2 The "Need" for Arrays

Why use arrays when we have standard variables like int, float, and char?

If you declare a standard variable:

int a = 5;

Memory representation of a single variable:

a
5
1000

The Problem: A single variable holds one value at a time. If you write a = 6;, the 5 is overwritten and lost forever.

What if a teacher needs to store the roll numbers of 60 students? Creating 60 separate variables (int a, b, c...) is messy, tedious, and bad programming practice. We need a way to store multiple values under a single variable name.

Part 3 Introduction & Memory Allocation

The solution is the Array. Under one variable name, we can store 60 numbers.

int rollno[60];

Breakdown:

  • int = The data type.
  • rollno = The name of the array.
  • [60] = The size (holds 60 items).

How Memory is Allocated (Contiguous Memory):

Assuming an int takes 4 bytes: 60 items × 4 bytes = 240 bytes total.

...
1000
...
1004
...
1008
...
1012
...
1236

Crucial Rule: These 240 bytes are allocated back-to-back. They are not scattered around the computer's memory.

Part 4 The Strict Definition of an Array

Based on the whiteboarding, an array must follow two strict rules:

  1. Multiple Items: A collection of more than one data item under one name.
  2. Homogeneous Data: All items in the array must be of the exact same data type.

Testing the Homogeneous Rule for int a[5]:

  • ✅ Valid: 1, 2, 3, 4, 5 (All positive integers)
  • ✅ Valid: 10, 20, 0, -1, 0 (Negatives and zeros are integers)
  • ❌ Invalid: 10, 20, 1.2, 0, 2.2 (Cannot mix integers with floating-point decimals!)

The Final Definition:
"An array is a collection of homogeneous data items stored in contiguous memory locations."

Part 6 Syntax & Declaration Rules

General Syntax: Datatype name_of_array[size_of_array];

The instructor tested several variations on the whiteboard to show what the C compiler will accept for the size inside the brackets [].

Declaration Written Valid? Instructor's Explanation
int a[]; ❌ No You must specify a size if you are only declaring the array without initializing it instantly.
int a[-5]; ❌ No Array size cannot be a negative number.
int a[2+2]; ✅ Yes Expressions are allowed! The compiler evaluates 2+2 to a constant size of 4.
int a[3*2]; ✅ Yes Evaluates to a constant size of 6.
int a[11/2]; ✅ Yes Evaluates to 5 (integer division drops the decimal).
int b = 11/2;
int a[b];		 ❌ No Standard C Rule: You cannot pass a variable as the size during declaration. The size must be a constant at compile time.

Part 7 Best Practices: Using Macros

Since passing a standard variable as an array size (like int n; int a[n];) is traditionally invalid or bad practice in C, how do we make our code flexible so we don't have to hardcode numbers everywhere?

Answer: Use Preprocessor Macros.

#include <stdio.h>
#define N 5   // Define the macro here

int main() {
    int a[N]; // The compiler sees this as int a[5];
    return 0;
}

Why this works: The C preprocessor finds every instance of N and replaces it with 5 before the code actually compiles. Therefore, the compiler sees a constant number, which perfectly satisfies the rules of array declaration!
