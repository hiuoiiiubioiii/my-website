Dynamic Memory Allocation in C

Complete In-Depth Study Notes on the malloc() Function

↓ Scroll to start learning ↓

1. Introduction to malloc()

What it stands for: malloc stands for Memory Allocation.

Primary Use: It is a function used to dynamically allocate a single, contiguous block of memory at runtime. While often used to allocate memory for structures, it is heavily used to create dynamic arrays where the size is decided by the user.

The Stack vs. The Heap Normal variables (like int a[30];) are allocated on the Stack at compile time. Memory allocated by malloc() comes from the Heap section of your computer's memory at runtime.

The Role of Pointers: Because malloc() reserves memory far away in the Heap, it returns the address of that memory. Therefore, you absolutely must use pointers to catch and use that address. Without pointers, dynamic memory allocation is impossible.

2. The Exact Prototype of malloc()

If you look inside the C standard library, this is how malloc() is defined:

void* Return Type
malloc( Function Name
size_t size Argument
);

Breaking down the components:

  • size_t size: This is the argument you pass in. size_t is essentially an unsigned int. You can only request a positive number of bytes (or zero). You cannot request "negative" memory.
  • void* (The Return Type): malloc returns a void pointer (a generic pointer).

    Why? Because malloc just reserves raw bytes. It has no idea if you plan to store integers, floats, or characters in those bytes.

3. The Problem with Hardcoding & sizeof()

When calling malloc(), you pass it one argument: the number of bytes you want.

Bad Practice: malloc(20)

You should rarely hardcode the number of bytes. Why? Because an int might take 4 bytes on a 64-bit machine, but only 2 bytes on an older 32-bit machine. If you hardcode bytes, your code is not portable and might waste memory or crash.

Best Practice: Use the sizeof() operator.

malloc(3 * sizeof(int));

If you want an array of 3 integers, you let the compiler do the math. If an int is 4 bytes, it allocates 12 bytes. If an int is 2 bytes, it allocates 6 bytes. It is always safe.

4. The Master Syntax Formula

Because malloc returns a generic void*, you must Typecast the pointer into the data type you intend to use. Combining typecasting and sizeof() gives us the standard formula:

ptr = Pointer Variable
(int*) Typecasting
malloc( Function
n * sizeof(int) Total Bytes Required
);

(Where n is the number of elements the user wants to store).

5. Two Crucial Behaviors of malloc()

1. Garbage Values If malloc successfully allocates memory, the block is left uninitialized. If you print the block before putting your own data into it, it will print garbage values left over by previous programs.
2. Allocation Failure What if you ask for memory, but the Heap is full? malloc will fail. If it fails, it returns a NULL pointer. Always check if ptr == NULL after using malloc to ensure your program doesn't crash!

6. Whiteboard Code Breakdown & Visualization

Here is the complete C program the instructor wrote on the board. Notice the inclusion of stdlib.h (mandatory for memory functions) and the use of free() at the end.

#include <stdio.h>
#include <stdlib.h> // MANDATORY: malloc() and free() live here!

int main() {
    int n, i, *ptr; // These static variables live on the Stack

    printf("Enter total no of values: ");
    scanf("%d", &n); // Let's say user enters 3

    // Dynamically allocate memory on the Heap
    ptr = (int*) malloc(n * sizeof(int));

    // Check for allocation failure
    if (ptr == NULL) {
        printf("Memory allocation failed!");
        return 1;
    }

    printf("Enter the values:\n");
    for(i = 0; i < n; i++) {
        // ptr holds base address (e.g., 2000). Adding 'i' moves 
        // the pointer by integer sizes (4 bytes).
        // i=0 -> 2000, i=1 -> 2004, i=2 -> 2008.
        scanf("%d", (ptr + i)); 
    }

    printf("The entered values are:\n");
    for(i = 0; i < n; i++) {
        // * is the dereference operator. It gets the VALUE at the address.
        printf("%d\t", *(ptr + i)); 
    }

    // MANDATORY: You must manually free dynamic memory when done!
    free(ptr); 

    return 0;
}

7. Why is Dynamic Allocation Better?

During the IDE demonstration, the instructor proves why malloc() is superior to traditional Static Arrays (like int a[30];) using an excellent real-world analogy.

🫓
The Hostel Mess / Cafeteria Analogy

Static Arrays (The Bad Way): You declare int a[30];. It's like telling the cafeteria in advance to make you 30 chapatis because you might be really hungry. But you show up and only eat 5. You have permanently wasted 25 spaces of memory. Conversely, if you are starving and need 35, the program crashes because you maxed out at 30.

Dynamic Allocation (The malloc Way): You go to the cafeteria, assess your hunger at that exact moment (Runtime), and ask for exactly 5 chapatis (n=5). The program gives you exactly what you need—no memory is wasted, and you never run out of space.
