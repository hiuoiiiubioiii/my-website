1. What is Recursion?

At its core, recursion is a very simple concept to define:

Definition: When a function calls itself (either directly or indirectly) inside its own body, the process is called recursion. The function is known as a recursive function.

Let's look at the visual comparison drawn on the whiteboard:

Normal Function Call

void main() {
    add(); // Calling a different func
}

void add() {
    // Definition of add
}

Recursive Function Call

void add() {
    // some code
    add(); // Calling ITSELF!
    // some code
}

2. The Danger of Recursion (The "Jenny" Analogy)

To explain why recursion can be dangerous, the instructor uses a real-world classroom analogy.

Normal Loop (Safe)

If the instructor needs a student named "Rahul", she will call: "Rahul? Rahul? Rahul?"
She keeps calling his name. Eventually, Rahul answers "Yes ma'am," and she stops calling him.

Recursive Loop (Dangerous)

Imagine the instructor's name is Jenny. What happens if she starts calling her own name?
"Jenny? Jenny? Jenny?"
Because she is Jenny, no one is going to come up to her. She will keep calling her own name forever. This is an infinite loop.

Takeaway: If a recursive function keeps calling itself without a strict rule to make it stop, it will run forever until your computer crashes.

3. Mathematical Breakdown (The Factorial Example)

Recursion is used to break a massive problem down into tiny, identical problems. The classic example is finding the factorial of a number.

5! = 5 × 4 × 3 × 2 × 1 = 120

We can rewrite this recursively. Notice how the problem gets smaller at each step:

5! = 5 × 4!
4! = 4 × 3!
3! = 3 × 2!
2! = 2 × 1!
1! = 1 × 0!
0! = 1 (STOP HERE)

The General Formula: n! = n × (n-1)!

If we don't stop at 0!, the math continues to -1!, -2!, to negative infinity! This proves why we must explicitly tell the computer to stop.

4. The Golden Rule: The Base Condition

Because a recursive function loops forever by default, you must provide a stopping point. The instructor circles this heavily on the whiteboard.

Base Condition (Termination Condition): The condition inside a recursive function that stops the recursion and returns a result instead of making another recursive call.

What happens if you forget it?

If you don't define a proper base condition, the function calls itself infinitely. Every function call takes up memory in the computer's RAM. Eventually, the memory is completely exhausted, resulting in a system crash known as a Stack Overflow.

For the factorial example, the base condition is: if (n == 0) return 1;

5. In-Depth Code Walkthrough

Here is the exact code written on the whiteboard. Notice the placement of the `printf` statements. One happens before the recursive call, and one happens after.

#include <stdio.h>

void display(int n) {
    // 1. BASE CONDITION (When to stop)
    if(n < 1) {          
        return;
    }
    else {
        // 2. Execution BEFORE recursive call
        printf("%d ", n);   
        
        // 3. RECURSIVE CALL
        display(n - 1);       
        
        // 4. Execution AFTER recursive call (Pending statement)
        printf("%d ", n);   
    }
}

void main() {
    int n = 3;
    display(n);
}

6. Visualizing Memory: The Stack Trace

This is the most critical part of the lecture. When a function calls another function, the current function pauses and waits for the new function to finish. The computer uses a data structure called a Stack to remember where it left off.

Recursion happens in two distinct phases: Winding (going forward/down) and Unwinding (returning backward/up).

Phase 1: Winding (Moving Forward)

The stack grows. Every time `display` is called, it executes the first `printf`, but it pauses before hitting the second `printf`.

Step 1
main() calls display(3)
main()Waiting...
→
Step 2
display(3) prints 3,
calls display(2)
display(3)n=3, paused at 2nd printf
main()Waiting...
→
Step 3
display(2) prints 2,
calls display(1)
display(2)n=2, paused at 2nd printf
display(3)Waiting...
main()Waiting...
→
Step 4
display(1) prints 1,
calls display(0)
display(1)n=1, paused at 2nd printf
display(2)Waiting...
display(3)Waiting...
main()Waiting...
Output on screen so far:
3 2 1

Phase 2: Hitting the Base Condition

At display(0), n < 1 becomes TRUE. It hits the return; statement. The function dies immediately and is popped off the stack. Control is handed back to the paused display(1).

Phase 3: Unwinding (Returning Backward)

Now the computer moves backward down the stack, "waking up" the paused functions. As each function wakes up, it executes its pending statement (the second printf) and then dies.

Step 5
Return to display(1).
Prints 1, then dies.
display(0)Returned
display(1)Executes 2nd printf, ends.
display(2)Waiting...
display(3)Waiting...
main()Waiting...
→
Step 6
Return to display(2).
Prints 2, then dies.
display(1)Destroyed
display(2)Executes 2nd printf, ends.
display(3)Waiting...
main()Waiting...
→
Step 7
Return to display(3).
Prints 3, then dies.
display(2)Destroyed
display(3)Executes 2nd printf, ends.
main()Waiting...
Final Output on screen:
3 2 1 1 2 3

7. Summary & Final Takeaways

  • Two Phases: Every recursive program has a Winding phase (breaking the problem down) and an Unwinding phase (building the solution back up).
  • Pending Statements: Code written after the recursive call does not execute until the Unwinding phase. It is stored in memory waiting for its turn.
  • Base Condition: Without it, the Winding phase never ends, causing a Stack Overflow.
