Void Pointers in C

Everything you need to know: Definitions, Memory Visualizations, and Code.

1. What is a Void Pointer? (The Empty Glass)

In English, "void" means empty. Think of an empty glass. Because the glass has no specific purpose assigned to it, you can fill it with water, milk, or juice.

In C, a Void Pointer is a pointer that has no associated data type. Because it has no specific type, it acts as a generic pointer and can store the memory address of any data type (integer, float, character, etc.).

Syntax: void *vp;

2. Standard Pointers vs. Void Pointers

Standard pointers have strict rules. An integer pointer can only hold the address of an integer. If you try to point it to a float, it breaks the rules.

// STRICT POINTERS
int *ip;    // Can only point to an int
float *fp;  // Can only point to a float
char *cp;   // Can only point to a char

// THE GENERIC VOID POINTER
void *vp;   

int a = 5;
float b = 1.11;
char c = '0';

vp = &a;    // ✅ PERFECTLY FINE
vp = &b;    // ✅ PERFECTLY FINE
vp = &c;    // ✅ PERFECTLY FINE

3. The Big Problem: Dereferencing

Assigning an address to a void pointer is easy. The problem happens when you try to print the value stored at that address (dereferencing).

COMPILER ERROR!
printf("%d", *vp); will result in an error: "invalid use of void expression".

Why does this happen? Let's look at memory.

When you tell the compiler to print *ip (an integer pointer), it goes to the memory address and reads exactly 4 bytes because it knows it's an integer. For a character pointer (*cp), it reads exactly 1 byte.

But when you tell the compiler to print *vp, the compiler arrives at the address (e.g., 1000) and panics. It has no data type, so it says: "How many bytes am I supposed to read? 1? 4? 8?"

4. The Solution: Typecasting

To fix the error, you must Typecast the void pointer before you dereference it. You have to temporarily tell the compiler what kind of data it is looking at.

* (int *) vp

Actually gets the value stored at the address.

Void Pointer
The pointer holding the raw memory address.

To print our integer a = 5, the correct code is:

printf("%d", *(int *)vp);

5. Why do we even use Void Pointers?

You might be thinking, "Why do this if standard pointers are easier?"

  • 1. Code Efficiency: You can declare just one void pointer and reuse it to print integers, floats, and characters. You don't need to waste memory declaring ip, fp, and cp.
  • 2. Dynamic Memory Allocation: Advanced C functions like malloc() and calloc() return a void pointer by default. They give you raw memory blocks, and you use typecasting (like (int *)malloc...) to define what goes inside those blocks.

6. The Complete Practical Program

Here is the exact code the instructor writes in Visual Studio Code to prove everything we've learned.

#include <stdio.h>

void main() {
    // 1. Declare one generic void pointer
    void *vp;               
    
    int a = 5;
    float b = 1.56;
    char c = 'c';

    // --- ACCESSING THE INTEGER ---
    vp = &a;                
    // printf("%d", *vp);   <-- THIS GIVES AN ERROR
    printf("a=%d\n", *(int *)vp);  // ✅ Correct Typecasting

    // --- ACCESSING THE FLOAT ---
    vp = &b;                
    printf("b=%f\n", *(float *)vp); // ✅ Correct Typecasting

    // --- ACCESSING THE CHARACTER ---
    vp = &c;                
    printf("c=%c\n", *(char *)vp);  // ✅ Correct Typecasting
}
Console Output:

b=1.560000
