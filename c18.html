1. Introduction to Bitwise Operations

Unlike arithmetic (`+`, `-`) or logical (`&&`, `||`) operators, Bitwise Operators manipulate data at the absolute lowest level: the bit level.

Computer memory cannot directly store decimal math like 5 + 4. It stores data in binary formats (0s and 1s). While the CPU inherently works in binary, C allows us to manually manipulate these bits using specific operators.

Memory Rules & Hierarchy

The smallest unit is a bit. Bitwise operators require a minimum of 1 Byte (8 bits) to perform an operation.

Bit 0 or 1
Byte 8 bits
Kilobyte (KB) 1024 Bytes
⚠️ Data Type Restriction: Bitwise operators can ONLY be applied to integer types (int) and character types (char). They cannot be applied to floating-point numbers (float, double).

2. The 6 Bitwise Operators

C provides 6 specific operators for bitwise manipulation. Five of them require two operands (Binary Operators), and one requires only one operand (Unary Operator).

& Bitwise AND
| Bitwise OR
^ Bitwise XOR
<< Left Shift
>> Right Shift
~ Bitwise NOT
(Unary)

3. Prerequisite: The "8-4-2-1" Shortcut

To understand bitwise operators, you must convert decimals to binary. We use the powers of 2 (from right to left) to create a shortcut table. Place a 1 under the numbers that add up to your target decimal, and 0 everywhere else.

Math 2³ (8) 2² (4) 2¹ (2) 2⁰ (1) Result
Make 10 (8+2) 1 0 1 0 1010
Make 5 (4+1) 0 1 0 1 0101

*4 bits can represent up to 15. For larger numbers, extend to 8 bits (128, 64, 32, 16, 8, 4, 2, 1).

4. Bitwise AND (&)

Rule: The output bit is 1 ONLY IF BOTH corresponding input bits are 1. Otherwise, it is 0.

Do not confuse this with Logical AND (&&), which just returns 1 (True) or 0 (False). Bitwise AND calculates a new decimal number.

8. Complex Expressions & Short-Circuiting

When multiple operators exist in one statement, the computer relies on Operator Precedence. The order of evaluation here is: Arithmetic (+) → Bitwise (&) → Logical AND (&&) → Logical OR (||).

int a = 10, b = 6; printf("%d", a & b && b + 1 || 0);

Step-by-Step Breakdown:

Step 1: Arithmetic (+)
Solve b + 1. 6 + 1 = 7.
Expression becomes: a & b && 7 || 0
Step 2: Bitwise AND (&)
Solve a & b (10 & 6).
1010 & 0110 = 0010 (which is 2 in decimal).
Expression becomes: 2 && 7 || 0
Step 3: Logical AND (&&)
Solve 2 && 7.
In logic, any non-zero number is True (1). True && True = True (1).
Expression becomes: 1 || 0
Step 4: Logical OR (||) & Short-circuiting
Solve 1 || 0.
Because the left side is True (1), the OR statement is guaranteed to be True. The execution stops here and returns 1.
Final Printed Output: 1

The Short-Circuit Trap 🪤

What if the code was: printf("%d", 1 || b++); and then we printed b?

Because the left side is 1 (True), the Logical OR short-circuits. The computer skips the right side entirely. Therefore, b++ is never executed. The value of b remains 6!
