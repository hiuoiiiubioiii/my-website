🧠 What is a+ Mode?

The a+ mode is the final mode in the series of 6 standard file opening modes (r, w, a, r+, w+, a+). It stands for Reading and Appending.

Core Rules of a+ Mode:

  • Purpose: Opens a file for both Reading and Appending.
  • Appending vs. Writing: Appending is strict. You can only write data at the very end of the file. You cannot overwrite data in the middle.
  • File Existence Check:
    • If the file exists: Previous content is kept safe.
    • If the file does not exist: A brand new file is created.
  • The File Pointer (Cursor): There is only one pointer. When you read, it moves through the text. When you append, it automatically jumps to the end.

💻 Writing the Logic (Whiteboard Code)

Here is the complete C program Jenny wrote on the whiteboard, implementing both reading and appending simultaneously.

// Standard inclusions
#include <stdio.h>
#include <stdlib.h>

void main() {
    // 1. Create file pointer and variables
    FILE *fp = NULL;
    char ch;

    // 2. Open file in a+ mode
    fp = fopen("abc.txt", "a+");

    // 3. Error handling
    if (fp == NULL) {
        printf("File doesn't exist / Cannot open");
        exit(1); 
    }

    // --- APPENDING LOGIC ---
    // Pointer automatically jumps to EOF for appending
    fputs("khatri", fp); 

    // --- CRUCIAL STEP ---
    // Move cursor from the end back to the beginning!
    rewind(fp);

    // --- READING LOGIC ---
    while (!feof(fp)) {
        ch = fgetc(fp);
        printf("%c", ch);
    }

    // 4. Close the file
    fclose(fp);
}

⚠️ The rewind() Edge Case

A common mistake students make in a+ mode is appending data and then immediately trying to read the file without using rewind().

What happens if you forget rewind()?
Your console will be completely blank!

Because appending forces the cursor to the absolute end of the file, your while(!feof(fp)) loop immediately detects the End-Of-File and terminates before printing a single character. rewind(fp) pushes the cursor back to index 0.

📊 Exam Prep: Comparing r+, w+, and a+

These three modes are highly testable. Here is how they differ:

Feature r+ Mode w+ Mode a+ Mode
Primary Actions Read & Write Write & Read Read & Append
If file missing? Returns NULL Creates new file Creates new file
Previous Content? Kept safe (Overwrites based on cursor) Erased entirely Kept safe
Cursor Location Beginning of file Fresh start (Beginning) Strictly End of file (for writing)

🛠️ Practical Demos (VS Code)

During the computer demonstration, Jenny proved three specific scenarios:

  • Scenario 1: Standard Execution
    She opened an existing abc.txt containing "Jenny khatri is awesome faculty". She appended "this is awesome", used rewind(), and read the file. The console successfully printed both strings together.
  • Scenario 2: Creating a Missing File
    She changed her code to target a.txt (which didn't exist). The program didn't crash; instead, it successfully created a brand new a.txt on her hard drive and appended the text into it.
  • Scenario 3: Proving the Bug
    She intentionally deleted the rewind(fp) line. She ran the code, and the console was totally empty. Upon checking the actual text file, the text was appended multiple times, proving that writing worked, but reading failed due to cursor placement.
