📝 Question 1: Global Variables & Defaults

Code Snippet:

#include <stdio.h>
int a;
int main()
{
    printf("a = %d", a);
    return 0;
}
Instructor's Whiteboard Notes

👉 int a; is declared outside of main(). It is a Global Variable.

❌ auto int a;
You cannot use the `auto` keyword with global variables. It is strictly for local variables.

👉 Default Storage Class: extern

👉 Default Initial Value: = 0

💡 Core Rule: Global/extern variables are automatically initialized to 0 by the compiler if you do not assign a value to them.

Final Output for Q1:

a = 0

📝 Question 2: The `static` Keyword & `for` Loop Execution

This is the most complex part of the lesson. Pay close attention to how the static keyword alters the standard flow of a function.

#include <stdio.h>
int fun()
{
    static int num = 16;
    return num--;
}

int main()
{
    for(fun(); fun(); fun())
        printf("%d ", fun());
    return 0;
}
💡 Concept 1: The static Keyword
A static variable is initialized only once. When fun() is called multiple times, the line static int num = 16; is ignored after the first run. The variable retains its previous value across function calls.
💡 Concept 2: Post-Decrement (num--)
The statement return num--; means: Return the current value of `num` first, and then decrease `num` by 1.

🧠 Step-by-Step Visual Dry-Run

Here is exactly how the instructor traces the for loop on the whiteboard, continuously crossing out the old value of num to write the new one.

Step 1: Loop Initialization

Code: for(fun(); ...; ...)

Inside `fun()`: `num` initializes to 16. It returns 16, then decrements.

num = 16 15

(Returned value 16 is ignored by the loop setup)

Step 2: Loop Condition (Check 1)

Code: for(...; fun(); ...)

Inside `fun()`: Static initialization line is ignored! Returns 15, then decrements.

num = 15 14

Condition check: 15 is not 0 (True). Enter loop.

Step 3: Inside Loop (Print 1)

Code: printf("%d ", fun());

Inside `fun()`: Returns 14, then decrements.

num = 14 13
Screen: 14

Step 4: Loop Update

Code: for(...; ...; fun())

Inside `fun()`: Returns 13, then decrements.

num = 13 12

Step 5: Loop Condition (Check 2)

Code: for(...; fun(); ...)

Inside `fun()`: Returns 12, then decrements.

num = 12 11

Condition check: 12 is True. Enter loop.

Step 6: Inside Loop (Print 2)

Code: printf("%d ", fun());

Inside `fun()`: Returns 11, then decrements.

num = 11 10
Screen: 14 11

Steps 7 to 15: The Pattern Continues

The loop cycles through Update ➡️ Condition ➡️ Print.

Updates drop the memory to 9, 6, 3.
Conditions drop the memory to 8, 5, 2.
Prints drop the memory to 7, 4, 1, and output the returned values: 8, 5, 2.

Screen: 14 11 8 5 2

Step 16: Final Loop Update

Code: for(...; ...; fun())

Inside `fun()`: Returns 1, then decrements.

num = 1 0

Step 17: Loop Condition (Final Check)

Code: for(...; fun(); ...)

Inside `fun()`: Returns 0, then decrements.

num = 0 -1

Condition check: The returned value is 0. In C, 0 means FALSE. The loop terminates immediately!

Final Output for Q2:

14 11 8 5 2
