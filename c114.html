1. Introduction to Unions

What is a Union? Just like a Structure (struct), a Union is a user-defined data type that can hold members of different data types under a single name.

While they look almost identical in syntax, there is one massive difference between the two: How memory is allocated.

2. Structures vs. Unions (Side-by-Side)

To understand Unions, we must compare them to Structures. Notice how identical the syntax is—you simply replace the keyword struct with union. You access members the exact same way using the dot (.) operator.

The Structure (struct)

struct demo {
    int a;
    char b;
    float c;
};

void main() {
    struct demo S = {1, 'a', 4.0}; 
    printf("%d %c %f", S.a, S.b, S.c); 
}

The Union (union)

union abc {
    int a;
    char b;
    float c;
};

void main() {
    union abc u;
    // Initialization is different!
}

3. The Core Concept: Memory Allocation

This is the most important concept in the lecture. The way RAM handles these two data types is fundamentally different.

Memory in a Structure

  • Rule: Every member gets its own separate memory block.
  • Calculation: Total memory = sum of the sizes of all members.
  • int a (4) + char b (1) + float c (4) = 9 Bytes Total.

Structure Memory Layout (9 Bytes)

a (4 bytes)
b (1 byte)
c (4 bytes)

Memory in a Union

  • Rule: All members share a single, common memory block.
  • Calculation: Total memory = size of the largest member in the union.
  • Since float and int require 4 bytes, the Union just reserves the max required: 4 Bytes Total.

Union Memory Layout (4 Bytes)

Shared 4-Byte Block
↑ u.a ↑ u.b ↑ u.c

All variables point to this exact same physical location in RAM.

4. The Drawback: Overwriting Data

Because all variables share the exact same 4 bytes of memory, a Union can only hold one value at a time. If you try to update a new member, the previous member's data gets destroyed/overwritten.

// Instructor's Whiteboard Example:
u.a = 1;        // 1 is stored in the 4-byte block.
u.b = 97;       // The 1 is erased. 97 is now stored in the block.
u.c = 9.2;      // The 97 is erased. 9.2 is now stored in the block.

What happens if you print them now?

printf("a = %d", u.a); // Output: Garbage value (1 was overwritten)
printf("b = %c", u.b); // Output: Garbage value (97 was overwritten)
printf("c = %f", u.c); // Output: 9.200000 (Correct, it was the last thing written)

5. Why Use Unions? (Advantages)

If data keeps getting overwritten, why would we ever use a Union instead of a Structure?

Memory Conservation: In scenarios where you have many variables inside a data structure, but you guarantee that the program will only ever need to use one variable at any given time, a union saves a massive amount of memory.

Example: If a structure has ten 4-byte integers, it uses 40 bytes. If a union has ten 4-byte integers, it only uses 4 bytes. In the early days of programming, memory was very expensive, making unions incredibly useful. Today, they are mostly used in embedded systems with strict memory limits.

6. Using Pointers with Unions

Just like structures, you can use pointers to access union members. The instructor demonstrated this syntax on the whiteboard:

union abc *ptr;      // Declare a pointer of type union abc
ptr = &u;            // Pointer stores the memory address of union variable 'u'

// Accessing members using the Arrow (->) operator:
ptr->a = 1;
ptr->b = 97;
ptr->c = 78.90;

The pointer ptr sits in memory, holding the base address of the union u, allowing you to point directly to that shared memory block to overwrite data.

7. Practical VS Code Demonstration

To prove the concept, the instructor ran the exact pointer code from above in VS Code.

// The Code Run:
u.a = 1;
u.b = 97;
u.c = 78.90;

printf("a value is: %d\n", ptr->a);
printf("b value is: %c\n", ptr->b);
printf("c value is: %f\n", ptr->c);
// Terminal Output:
a value is: 1117837837 (Garbage value)
b value is: (Nothing/Garbage character)
c value is: 78.900002 (Correct value, because it was the last one entered)
