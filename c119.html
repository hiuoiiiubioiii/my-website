1. Why Do We Need File Handling? (The "Need")

Before learning how to write the code, we must understand the memory limitations of a standard C program. The instructor uses a basic addition program (5 + 10 = 15) to demonstrate this.

💻 Console

a = 5
b = 10
sum = 15

Output Screen

⚡ RAM

Volatile Memory

Variables live here while the program runs.

Data lost on program exit or power off.

💾 Hard Disk

Non-Volatile Memory

Data stored permanently in files.

(Sequence of Bytes)

  • The Problem with RAM: Programs execute in RAM. Because RAM is volatile, the moment your program ends or the computer turns off, your output (like `sum = 15`) is destroyed.
  • The Solution (Hard Disk): To save program outputs for future use, you must store data in the Hard Disk, which is non-volatile (permanent).
  • Handling Large Inputs: If a program needs thousands of numbers as input, typing them into the console manually is impossible. Storing that data in a file on the Hard Disk allows the C program to automatically fetch it.

2. What Exactly is a "File"?

On the whiteboard, the instructor points to the blocks inside the Hard Disk and defines a file simply:

"A file is a sequence of bytes."

  • It is a block of memory located on your Hard Disk designed for permanent data storage.
  • File Handling is the process of writing C commands to interact with these sequences of bytes (rather than opening them manually with your mouse).

3. Types of Files in C

The instructor categorizes files into two main types based on how data is stored and read:

📝 Text Files (.txt)

Stores data as plain text (alphabets, numbers, symbols).

  • Human Readable: You can open it in Notepad and easily read it.
  • Drawback: Because it is easily readable, it is less secure.

⚙️ Binary Files (.bin)

Stores data as a stream of compiled 0s and 1s.

  • Not Human Readable: Opening this in Notepad results in gibberish/symbols.
  • Benefit: Highly secure compared to text files because humans cannot decipher it easily.

4. Standard File Operations

When you write C code to handle files, you are typically performing one of these five primary operations:

  • 📄 Create: Making a brand new file on the hard disk.
  • 📂 Open: Accessing an existing file to use it in your program.
  • 👁️ Read: Fetching (pulling) data out of the file to use as input in your C program.
  • ✍️ Write: Sending data (output) from your C program to be stored permanently in the file.
  • 🔒 Close: Crucial Step! Once you are done, you must command C to close the file. This saves the file properly and frees up computer memory.

5. The File Pointer (The Bridge)

To perform any of the operations listed above, your C program needs a way to "point" to the specific file residing on the hard disk. We do this using a special pointer.

#include <stdio.h>

int main() {
    // Creating a file pointer
    FILE *ptr;
    
    return 0;
}
  • FILE is a custom data type. It must be typed in all capital letters.
  • It is pre-defined inside the standard input/output header file <stdio.h>, which is why we must include that header.
  • *ptr is the pointer variable you create to act as the bridge between your code and the physical file on the disk.

(Note: The exact code functions to open, read, write, and close using this pointer will be covered in the next lecture.)
