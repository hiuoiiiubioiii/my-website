Structure Padding in C

👩‍🏫 Taught by: Jenny's Lectures | ⏱ Read Time: ~7 mins | 📘 Topic: C Programming Memory Management

Welcome to the visual study notes for Structure Padding! As you scroll down, you'll uncover exactly how the compiler manages memory behind the scenes to optimize CPU performance.

1. The Naive Size Calculation

Let's look at a seemingly simple structure with two variables inside it:

struct abc {
    char a;  // Takes 1 byte
    int b;   // Takes 4 bytes
} s;

Question: What is the size of the variable s?

The Naive Answer: 1 byte (char) + 4 bytes (int) = 5 bytes.

The REAL Answer: It actually takes 8 bytes. To understand why, we need to look at how your CPU reads memory.

2. How the CPU Reads Memory (Word Addressable)

Historically, older processors were Byte Addressable, meaning they read memory exactly 1 byte at a time. However, modern CPUs are Word Addressable.

  • In 32-bit machines, 1 Word = 4 Bytes. The CPU fetches 4 bytes in a single cycle.
  • In 64-bit machines, 1 Word = 8 Bytes. The CPU fetches 8 bytes in a single cycle.

Note: For these visual examples, we are assuming a 32-bit architecture (fetching 4 bytes at a time).

🛒 The Store Analogy:

Imagine you need to buy a shirt, pants, t-shirt, and jeans. If you go to the store 4 separate times to buy one item per trip, it's highly inefficient. It's much faster to go once and grab all 4 items at the same time. The CPU thinks the exact same way!

Why is this bad? Because the CPU fetches data in 4-byte Words. To read the integer b, the CPU has to execute Two Cycles:

  1. Cycle 1: Fetches addresses 100-103. It gets the char, and only 3 parts of the int.
  2. Cycle 2: Fetches addresses 104-107. It finally gets the last part of the int.

Taking two cycles to read a single integer severely decreases your computer's speed and performance.

4. The Solution: Structure Padding

To fix this performance issue, the compiler automatically steps in. It forces variables to start at addresses that are multiples of their size. An int (4 bytes) must start at an address that is a multiple of 4 (like 100, 104, 108).

The compiler inserts empty, unused bytes known as Padding Bits or Memory Holes. Now, the int starts cleanly at address 104.

  • The Benefit: The CPU can read int b in exactly ONE cycle. Processing speed is highly optimized!
  • The Cost: Wasted memory space. The total size is now 1 (char) + 3 (padding) + 4 (int) = 8 Bytes.

5. The Worst-Case Scenario: Wasting Memory

Let's add another char c to our structure, placing it after the integer.

struct abc {
    char a;  // 1 byte
    int b;   // 4 bytes
    char c;  // 1 byte
} s;

6. How to Optimize as a Programmer

Since the compiler does padding automatically based on the order you type, it is your duty as a programmer to order your variables smartly. Always declare them in increasing (or decreasing) order of their sizes.

struct abc {
    char a;  // 1 byte
    char c;  // 1 byte
    int b;   // 4 bytes
} s;

By simply moving char c to sit next to char a, they can share the same Word! The compiler only needs to add 2 bytes of padding. We successfully reduced our structure size from 12 bytes down to 8 bytes.

🎯 Interview Prep Summary

  • What is Structure Padding? It is the automatic insertion of empty bytes between structure members by the compiler.
  • Why do we do it? To align data in memory to match the CPU's word size. This allows the processor to fetch data in one CPU cycle, heavily increasing speed.
  • What is the trade-off? Speed vs. Memory. You get a faster running program, but it consumes more memory (wasted padding bits).
  • Can we disable this? Yes! If you are in a highly memory-constrained environment, you can force the compiler to ignore padding using a technique called Structure Packing (which will be covered in the next topic). Note that structure packing will make your code slower.
