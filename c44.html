Nested while Loops in C

Instructor: Jayanti Khatri Lamba (Jenny's Lectures)

What is nesting? Simply put, a nested while loop is a while loop placed inside another while loop.

Instructor's Tip: In C programming, you are not restricted! You can mix and match loops based on your program's needs up to n levels deep. For example, you can place a while inside a for loop, or a do-while inside a for inside a while.

🧩 General Syntax on the Board

Here is exactly how the instructor mapped out the syntax on the whiteboard. Notice the specific blocks for the inner and outer loops.

while (Outer_Condition) 
{
    // Control enters here if Outer_Condition is TRUE

    while (Inner_Condition) 
    {
        // I.L.S.B: Inner Loop Statement Block
        // Control enters here if Inner_Condition is TRUE
        // Usually contains an increment/decrement for inner variable
    }

    // O.L.S.B: Outer Loop Statement Block
    // Executes AFTER the inner loop finishes (Inner becomes FALSE)
    // Usually contains an increment/decrement for outer variable
}

✍️ The Program & Dry Run

To prove how this works, the instructor traces variables exactly like a computer's memory. This is the most critical trap in nested loops.

int i = 0, j = 1;

while (i < 3) 
{
    while (j <= 3) 
    {
        // Post-increment: prints original j, then adds 1
        printf("%d ", j++); 
    }
    // Post-increment: prints original i, then adds 1
    printf("%d ", i++);     
}

Watch the variables change: Notice how j crosses out and increases to 4. Because she never reset j = 1 inside the outer loop, the inner loop gets completely skipped on iterations 2 and 3!

⚠️ The "Homework" Challenge

At the very end of the video, the instructor erases the inner statements and asks you to figure out the output. Here is the modified code:

int i = 0, j = 1;

while (i < 3) 
{
    while (j <= 3) 
    {
        // Empty block! Nothing happens here.
    }
}
// Pre-increment: adds 1 to variables, then prints them
printf("%d, %d", ++i, ++j);

🚨 The Solution: Infinite Loop

If you look closely, there are no increment statements (i++ or j++) inside the loops anymore!

  • The program checks i < 3 (0 < 3) ➔ True.
  • It enters the inner loop and checks j <= 3 (1 <= 3) ➔ True.
  • Because the block is empty, j never changes. It remains 1 forever.
  • Result: The program gets stuck in an Infinite Loop checking 1 <= 3 infinitely. It will never reach the printf statement at the bottom.
