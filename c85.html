🧠 1. The Core Concept: Why Declare?

Think about how variables work. Before you can use a variable, the compiler needs to know its name and data type.

// If you do this without declaring 'sum' first...
sum = a + b; // ERROR! Compiler asks: "What is sum?"

// The fix is declaring variables:
int a, b, sum;
sum = a + b; // Works perfectly.

The exact same rule applies to functions. You must introduce (declare) a function to the compiler before you actually use (call) it in your code.

🧩 2. Anatomy of a C Function

There are typically 3 parts to using a function in C. Look at the color-coded visualizer below to understand where each part lives in your script.

1. Declaration (Prototype)
2. Calling (Trigger)
3. Definition (Logic)
#include <stdio.h>

// 1. Tells compiler: "A function named 'sum' exists"
void sum(void);

void main() {
    // 2. Triggers the function execution
    sum();
}

// 3. The actual step-by-step logic
void sum(void) {
    int a, b, s = 0;
    printf("Enter two numbers: ");
    scanf("%d %d", &a, &b);
    s = a + b;
    printf("Sum: %d", s);
}

📝 3. Declaration Syntax Rules

A function declaration is a single line that acts as a blueprint. It tells the compiler 3 things:

return_type function_name ( type_of_arguments );
  • Semicolon is mandatory: Declarations must end with `;`.
  • Variable names are optional: Writing void sum(int a, int b); is valid, but void sum(int, int); is preferred. The compiler only cares about the data types.
  • Strict Matching: The return type and arguments in the declaration must exactly match the actual Definition.

⚠️ 4. The Empty Brackets Trap

If your function takes zero inputs, you might think leaving the brackets empty () is correct. It is not.

THE DANGER: In older C standards, an empty bracket () in a declaration means "this function can take ANY number of arguments."


If you declare void sum();, and accidentally call sum(5, 7); later, the compiler won't stop you. This leads to hidden logic errors.

The Solution: Always write (void) in the declaration. Example: void sum(void);. This strictly tells the compiler: "This function takes absolutely ZERO arguments." If you try to pass arguments now, it throws a safe, helpful error.

💻 5. Practical VS Code Scenarios

What happens if you break the rules? Let's look at the terminal errors you will encounter in VS Code.

Scenario A: You forget to declare entirely.

warning: implicit declaration of function 'sum' [-Wimplicit-function-declaration]
> sum();

Why? If C encounters a call without seeing a declaration, it implicitly assumes the function returns an 'int'. The program might still run, but it's risky.

Scenario B: You forget to declare, AND the function returns a float.

warning: implicit declaration of function 'sum'...
error: conflicting types for 'sum'
> float sum(void) { ... }

Why? Because you didn't declare it, the compiler assumed it returned an int. When it finally reads your definition and sees it returns a float, the two types conflict and crash the program. Takeaway: If returning anything other than int, you MUST declare.

🌍 6. Scope: Global vs. Local Declaration

Where you write your declaration matters.

void main() {
    // LOCAL DECLARATION
    char fun(void);

    char ch = fun();
    printf("%c", ch);
}

Local Scope: The code above is perfectly legal. However, because char fun(void); is declared inside main(), only the main function is allowed to use it.

Best Practice: Declare functions Globally (at the top of the file, above `main`). This allows any function in your entire script to call it.

🎯 7. Instructor's Homework Challenge

At the end of the video, the instructor edits the fun() definition to look like this:

char fun(void) {
    char c;
    printf("Enter a character: ");
    scanf("%c", &c);

    // Notice the single quotes here!
    return 'c';
}

Question:

If the user runs this program and types the letter 'H' on their keyboard when prompted, what exactly will print to the screen?

Answer: It will print the letter c.

Explanation: Because the return statement is returning 'c' (a literal character string enclosed in single quotes) instead of c (the variable without quotes), the function completely ignores the user's input of 'H' and hardcodes the return value to the letter 'c'.
