1. What is an Enum?

An Enum (short for enumeration) is a user-defined data type in C. Its main job is to assign readable, human-friendly names to integral constants (numbers).

Enum in C
↳ User-defined type
↳ Used to assign names to Integral Constants
👩‍🏫 Teacher's Analogy: The Classroom

Imagine a classroom. The teacher says, "Roll number 45, stand up." It takes a second for the students (even roll number 45) to process who that is.

Now imagine the teacher says, "Rahul, stand up." The reaction is instant.

In coding, using names (like Sun, Male) instead of raw numbers (0, 1) heavily increases the readability and maintainability of your program. Anyone reading your code will understand it immediately.

2. Syntax: Enum vs. Struct

To understand how to declare an enum, the instructor compares it directly to a struct, which you already know.

// Struct syntax
struct Student {
    int roll_no;
    float marks;
};
struct Student s;

// Enum syntax
enum days {
    Sun,
    Mon,
    Tue
};
enum days today;
👩‍🏫 The Rule of Restriction

If you declare a normal int a;, it can hold ANY integer. But if you declare an enum variable like enum week_days today;, it can ONLY hold values from the specific list you provided (Sun, Mon, Tue, etc.). If you try to assign today = Sat; but you didn't include 'Sat' in your list, the compiler throws an error!

3. Behind the Scenes: Integer Auto-Assignment

You are typing words, but the C compiler only understands numbers. How does it handle this?

enum week_days {
  Sun, // Automatically becomes 0
  Mon, // Automatically becomes 1
  Tue, // Automatically becomes 2
  Wed, // 3
  Thu, // 4
  Fri, // 5
  Sat // 6
}; 
enum week_days today = Sun;
printf("%d", today); // Output: 0

today = Tue;
printf("%d", today); // Output: 2

4. Customizing Values (The Rules)

You do not have to accept the default 0, 1, 2 assignments. You can manipulate them according to specific rules.

  • Rule 1 Changing the starting point:
    If you write { Sun = 1, Mon, Tue }, the compiler auto-increments the rest. Sun=1, Mon=2, Tue=3.
  • Rule 2 Random breaks in order:
    If you write { Sun = 1, Mon = 5, Tue }, Tue will automatically become the previous value + 1. So, Tue=6.
  • Rule 3 Duplicate values are allowed:
    If you write { Sun = 0, Mon = 0 }, this is perfectly valid in C.
  • Rule 4 NO FLOATS:
    Enums only hold Integral Constants. Writing { x = 0, y = 2.5 } will trigger a compiler error.

5. Enums vs. Macros (#define)

Why use Enums when we can just use #define MAX 50 to name our numbers? There are major differences:

1. Scope Differences:
↳ Macros (#define) are global. Processed by the preprocessor before compilation.
↳ Enums can be Local OR Global.

2. Duplicate Names:
↳ You cannot have duplicate names in the same Enum scope.
  { Sat, Sun } and another enum with { Sun } in the same function = ERROR.

3. Auto-Assignment:
↳ Enums auto-assign sequential values.
↳ Macros require you to assign a value explicitly every time.
👩‍🏫 Teacher's Tip: Local Scope

If you declare an enum inside void fun1() { ... }, that enum cannot be accessed outside of fun1(). This allows you to keep your variables strictly contained where they are needed, unlike Macros which bleed everywhere.

6. Practical Use Cases in Business Logic

When should you use an Enum? Whenever a variable has a limited, fixed set of possibilities. (e.g., 7 days, 12 months, 4 directions).

Use Case 1: Shopping Discounts
enum gender { Male, Female, Others };

if (customer == Female) ↳ 50% discount
if (customer == Male) ↳ 25% discount
if (customer == Others) ↳ 60% discount

Use Case 2: The Switch Statement

switch(today) {
    case Sun: 
        printf("Today is Sunday");
        break;
    case Mon:
        printf("Today is Monday");
        break;
}
👩‍🏫 Readability Win!

Writing case Sun: is infinitely easier for another developer to read than writing case 0:. It removes the need for them to guess what "0" stands for.
