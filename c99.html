Returning a Pointer from a Function in C

A comprehensive, visual breakdown of pointer arithmetic, stack memory, and constant pointers.

1. Introduction & Context

In previous lessons, we discussed returning a string from a function. Returning a string is essentially returning a character pointer (char *) because you are returning the base address of the first character.

The goal of this lesson is to learn how to return an integer pointer (int *) from a function by passing an array, modifying the pointer using pointer arithmetic, and returning the new memory address.

2. The Complete Code (From the Whiteboard)

Here is the complete C program the instructor constructs on the whiteboard. Take a look, and then we will break down exactly how it operates in the computer's memory.

#include <stdio.h>

int* returnPointer(int a[]); // Function Prototype

int main() {
    int *p;
    int a[] = {1, 2, 3, 4, 5};
    
    // Passing the base address of array 'a'
    p = returnPointer(a); 
    
    printf("%d", *p);
    
    return 0;
}

int* returnPointer(int a[]) {
    // a[] is treated as a pointer variable *a
    a = a + 2;
    return a;
}

3. Visualizing Memory & Pointer Arithmetic

To understand what happens when a = a + 2 executes, we need to look at how the array is stored in memory and how the Stack works.

The Array in Memory

When int a[] = {1, 2, 3, 4, 5}; is declared in main, the computer allocates 5 contiguous blocks of memory. Assuming an integer takes 4 bytes, and the base address is 100, the memory looks like this:

Pointer Arithmetic: New Address = Base Address + (Increment * SizeOf(DataType))
New Address = 100 + (2 * 4 bytes) = 108

The Stack Frame Trace

When returnPointer(a) is called, a new frame is placed on the Stack. Let's trace the values.

The Call Stack
returnPointer() Frame
Parameter: int *a 100 108
* The local pointer variable 'a' receives 100.
* `a = a + 2` updates it to 108.
* Returns 108 to main.
main() Frame
Array: a Base: 100
Pointer: *p 108
* 'p' receives 108 from the function.
* `printf("%d", *p)` looks at 108 and prints 3.

4. The Big "Gotcha": Constant Pointers vs Pointer Variables

This is a crucial rule in C that confuses many students. The instructor poses a scenario: What if we try to do the pointer arithmetic directly inside main()?

⚠️ Compilation Error Scenario

If you write the following code inside main(), the compiler will throw an error: error: assignment to expression with array type

int main() {
    int a[] = {1, 2, 3, 4, 5};
    a = a + 2; // ILLEGAL! THIS WILL CRASH!
    printf("%d", *a);
}

Why does it fail in main() but work in the function?

  • Inside main(): a is the actual name of the array. The name of an array acts as a Constant Pointer to the base address (100). You cannot reassign or change the value of a constant pointer. It is locked to that array.
  • Inside returnPointer(): When you pass the array as an argument int a[], the compiler interprets it as a standard pointer variable int *a. A standard pointer variable is not constant. It is just a temporary variable holding an address, so a = a + 2 is perfectly legal there.

6. Summary & Pro-Tips

  • Asterisk is mandatory: To return a pointer, the function return type must have an asterisk (e.g., int *, float *, char *).
  • Arrays degrade to pointers: When passing an array to a function int func(int a[]), it is received as a pointer variable int *a.
  • Pointer Math depends on data type: Adding +2 to an integer pointer doesn't move it 2 bytes; it moves it 2 integers forward (which is 8 bytes if an int is 4 bytes).
  • Code Optimization: Instead of writing a = a + 2; return a;, you can simply write return a + 2; inside the function to save a line of code.
  • Safety First: In real-world code, it is good practice to pass the length of the array to the function as well (e.g., int returnPointer(int a[], int length)). This allows you to write if statements to prevent the pointer from moving out of bounds!
