Dynamic Memory Allocation (DMA) in C

Comprehensive Lecture Notes & Visualizations based on Jenny's Lectures

1. The Basics: SMA vs. DMA

In C programming, there are two primary ways to allocate memory to your program:

  • SMA (Static Memory Allocation): Memory requirement is fixed at compile time.
  • DMA (Dynamic Memory Allocation): Memory is allocated at run time.
⚠️ Crucial Concept Clarification: "Compile Time"
Even though we say SMA happens at "compile time," memory from your RAM is not actually given to the program during compilation. At compile time, your English-like C code is simply converted into an Object file (.obj / machine code).

Why do we say it happens at compile time? Because the amount of memory to be allocated is strictly fixed by the compiler. RAM is only physically assigned when the program is actually executed.

Example of Static Memory Allocation:

int main() {
    int a, b;  // Compiler fixes this to 8 bytes (4 bytes each)
    printf("Enter value for a and b:");
    scanf("%d %d", &a, &b);
    printf("a = %d b = %d", a, b);
    return 0;
}

The compiler sees int a, b; and decides, "I will need exactly 8 bytes." Once the program runs, you cannot change this fixed size.

2. The Drawbacks of Static Memory Allocation

To understand why we need Dynamic Memory Allocation, we must look at the flaws of SMA. There are three major drawbacks.

Drawback 1: Wastage of Memory

If you declare an array or string but don't use all of it, the unused memory sits empty and cannot be used by anything else.

Example 1: Array Wastage

int arr[5]; // Allocates 20 bytes (5 blocks of 4 bytes)

If you only input 3 values (e.g., 1, 3, -1), the remaining 2 slots are wasted.

Drawback 2: Shortage of Memory

If you created int arr[5]; but suddenly realize at runtime you need to store 7 values, you cannot do it. The size is fixed and will crash or overflow if exceeded.

Drawback 3: Memory Cannot be Freed or Reused

In SMA, memory remains occupied until the program completely finishes and exits. You cannot free up the 46 wasted bytes from the string example in the middle of your program to use it for something else.

3. Enter Dynamic Memory Allocation (DMA)

DMA solves all the drawbacks of SMA. Memory is allocated at runtime and is not fixed. You can modify (increase or decrease) the size of the memory based on your programming needs while the program is running.

C provides 4 built-in functions for DMA (we will learn the syntax in the next lecture):

  • malloc() - Memory Allocation
  • calloc() - Contiguous Allocation
  • realloc() - Re-allocation (resizing)
  • free() - De-allocation (deleting)

4. The C Program Memory Layout

To truly understand DMA, you must know how a C program uses your computer's RAM. The memory assigned to your program is divided into 4 main segments.

Stack Segment
Local variables (int a, b) & function calls (main, printf).
Has a limited size (e.g., 2MB).
Free RAM
Heap Segment
A massive pool of free memory. DMA exclusively uses the Heap.
Global / Static Segment
e.g., global int i = 10; or uninitialized int a;
Code / Text Segment
Where compiled machine instructions live.

Note: The Stack grows downwards, and the Heap grows upwards toward each other. All Dynamic Memory Allocation happens strictly inside the Heap.

5. How DMA Actually Works

Here is the core mechanic of Dynamic Memory Allocation. You cannot use DMA without Pointers. Pointers are absolutely mandatory.

The Allocation Process

When you use a DMA function like malloc(), you ask the computer for a specific amount of bytes. The block is created in the Heap, but it has no variable name. Therefore, a pointer living in the Stack must hold the base address of that Heap block to access it.

The Crucial Role of the free() Function

In SMA, memory is destroyed automatically when the program ends. In DMA, it is the programmer's responsibility to free the memory.

🚨 Danger: Memory Leaks
If you write p = NULL; before freeing the memory, the arrow breaks. The pointer no longer points to the Heap memory. However, that 8-byte block in the Heap is still reserved. You can no longer access it, but the computer can't use it either. Doing this repeatedly causes a Memory Leak, eventually exhausting the Heap and crashing the program.

To properly delete the memory so it can be reused by your program, you must use free(p);. You can call this anywhere in your program once you are done with the memory block.

6. Summary: The Golden Rule of Reusability

The entire concept boils down to resource management:

  • SMA: Memory size is fixed. It cannot be increased, decreased, or freed during program execution. It cannot be reused.
  • DMA: Memory size is flexible. It is allocated from the Heap. By using the free() function, you can release the memory back to the Heap during execution, allowing it to be reused later in the exact same program.
