C Programming: Returning Strings

Instructor Whiteboard Breakdown & Memory Visualizations

1. The Basics: Returning a String Literal

The instructor begins by trying to return a simple string literal ("Jenny") from a function to the main() function.

char* display() {
    return "Jenny";
}

void main() {
    char* str;               // Pointer to catch the address
    str = display();         
    printf("String is: %s", str);
}

The "Why": Strings in C are character arrays. You cannot return an entire array directly. Instead, you return the base address (a pointer to the first character, 'J'). This is why the return type must be char*.

Explanation: String literals are NOT stored in the stack frame. They are stored in a separate, Read-Only Data Section. When the display() stack frame is wiped out, the string remains safely in memory, and `main()` can still read it via the pointer!

3. The Interview Trap: Returning a Local Array

Next, the instructor shows the most common mistake students make: declaring the string as a local array.

char* display() {
    char str[] = "Jenny"; // Local array!
    return str;
}
⚠️ The Dangling Pointer Problem

By writing char str[], you allocate memory for the array inside the stack frame of display(). When the function returns, that stack frame is wiped out.

4. The Fix: Using the 'static' Keyword

How do we fix the local array trap? By using the static keyword.

char* display() {
    static char str[] = "Jenny"; // Added 'static'
    return str;
}

The static keyword removes the variable from the temporary Stack memory and places it in the Static memory section. Its lifetime is now the entire duration of the program.

5. Modifying Strings & The 'const' Keyword

What if you try to change the returned string inside main()?

void main() {
    char* str = display();
    str[0] = 'Z'; // Trying to change "Jenny" to "Zenny"
}

If you returned a string literal: The program will crash (Segmentation Fault). Why? Because string literals live in the Read-Only Data Section. You cannot write to it.

The Compilier Issue: The compiler won't stop you from writing str[0] = 'Z'. It will only fail at runtime.

✅ Best Practice: Use const

To protect your code, declare the return type as const char*. This tells the compiler the data is read-only. If you accidentally try to modify it, the compiler will throw an error immediately, saving you from a runtime crash.

const char* display() {          
    const char* str = "Jenny"; 
    return str;
}
