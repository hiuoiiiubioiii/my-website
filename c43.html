🚨 The Problem: Why Do We Need Nesting?

Before understanding what a nested loop is, we must understand why we use it. The instructor builds the logic through three scenarios:

Scenario 1: Print a single star.

printf("*");

Scenario 2: Print a line of 5 stars.

Hardcoding printf("*****"); is bad practice. If the user wants 500 stars, you can't type 500 stars. We use a single loop:

for(int i = 1; i <= 5; i++) {
    printf("*");
}

Scenario 3: Print a 6x5 Grid of Stars.

You cannot write 6 separate for loops sequentially. To repeat a process (printing a line) that is already a loop, you must put that loop inside another loop.

* * * * *
* * * * *
* * * * *
* * * * *
* * * * *
* * * * *

💻 The Solution: Syntax & Code

A nested loop is simply a loop placed inside the body of another loop. The outer loop controls the rows (vertical lines), and the inner loop controls the columns (items per line).

// General Syntax taught on the whiteboard
for(Initialization; Condition; Update) {    // OUTER LOOP
    
    for(Init; Condition; Update) {          // INNER LOOP
        // Inner Loop Statement Block
    }
    
    // Outer Loop Statement Block
}

Here is the exact code to generate the 6x5 star grid:

for(i = 1; i <= 6; i++) {          // Loops 6 times (Rows)
    for(j = 1; j <= 5; j++) {      // Loops 5 times per row (Cols)
        printf("* ");
    }
    printf("\n");                  // Moves cursor to next line
}

🧠 The Dry Run (How the Compiler Reads It)

The Golden Rule: For every single iteration of the Outer Loop, the Inner Loop runs to complete finish and resets.

Here is a trace of the variables executing in memory. Note how j resets to 1 every time i increases.

📝 Important Rules of Nesting

Before concluding, the instructor listed three critical rules about nesting loops in C that you must remember for exams:

  • 1. Infinite Depths: You are not restricted to just two levels (Inner and Outer). You can place a loop inside a loop inside a loop up to N levels. However, time complexity drastically increases.
  • 2. Mixing Loop Types: You do not have to put a for loop inside a for loop. You can legally nest a while loop inside a for loop, or a do-while inside a while.
  • 3. Sequential Inner Loops: It is totally valid to have an outer loop that contains multiple separate inner loops sequentially. For example: Outer loop starts ➔ Inner Loop A finishes ➔ Inner Loop B finishes ➔ Outer Loop updates and restarts.

💡 Pro-Tip (Dynamic Input): Avoid hardcoding numbers like 5 and 6. A better practice is to take input from the user (e.g., scanf("%d", &n);) and use i <= n; in your loop condition to make your program flexible!
