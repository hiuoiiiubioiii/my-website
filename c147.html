Question 1

Tracing auto vs. register

How do local block variables behave when a function is called multiple times?

int a, b, c = 0; // Global vars
void prtFun(void);

int main() {
    auto int a = 1;     // Local to main
    prtFun();
    a += 1;
    prtFun();
    printf("\n%d %d", a, b);
}

void prtFun(void) {
    register int a = 2; // Stored in CPU register
    int b = 1;          // Auto by default
    a += ++b;
    printf("\n %d %d", a, b);
}

🧠 Whiteboard Memory Trace

Global Scope (Initialized to 0)
a
0
b
0
c
0
main() Stack
a (auto)
2

*Updated from 1 to 2 via `a += 1`

prtFun() Stack (Destroyed on exit)
a (reg)
4
b (auto)
2
1. Enter main(): Memory allocated for local a. Set to 1.
2. First prtFun() Call: Local a=2, local b=1. Math: a = a + ++b. Pre-increment makes b=2. Then a = 2 + 2 = 4. Prints 4 2. Memory is destroyed.
3. Back in main(): a += 1 updates main's local a from 1 to 2.
4. Second prtFun() Call: Variables are recreated fresh! a is 2, b is 1. The exact same math happens. Prints 4 2 again.
5. Final Print in main(): Needs a and b. Local a is 2. main() has no local b, so it uses the Global b which is 0. Prints 2 0.
// Output:
4 2
4 2
2 0
Question 2

The extern Keyword & Linker Errors

What happens when we promise the compiler a variable exists, but it doesn't?

int i = 5;

int main() {
    extern int j;
    printf("%d %d", i, j);
    
    int j = 10;
    return 0;
}

🔎 Scope Search Visualizer

Compiler / Linker Process

extern int j; translates to:

  • "Hey Linker, don't allocate memory here."
  • "Look OUTSIDE this block for a global j."
Result: Global Search Failed! No global 'j' exists.
1. Global space: i is defined as 5.
2. extern int j: This is a declaration, not a definition. It forces the linker to search outside main() for a global variable named j.
3. The Conflict: We have int j = 10; defined inside main, after the print statement. But extern strictly searches outside the block.
4. The Crash: Because there is no global j, the linker throws an error.
// Output:
Linker Error: Undefined reference to 'j'
Question 3

Block Scope with extern

int main() {
    extern int i;
    printf("%d", i);
    
    {
        int i = 10;
        printf("%d", i);
    }
    
    return 0;
}
1. The Promise: extern int i; promises the compiler that a global i exists somewhere outside this main block.
2. The Trap: We declare int i = 10;, but it is trapped inside its own inner block { ... }.
3. The Rule: The scope of the inner i is strictly limited to those curly braces. The extern command outside cannot see it, and there is no global i.
// Output:
Linker Error: Undefined reference to 'i'
Question 4

Pointers and register Variables

int main() {
    register int Data = 10;
    int *piSumData = NULL;
    
    // Attempting to get address
    piSumData = &Data; 
    
    *piSumData = 5;
    printf("%d", *piSumData);
}

⚙️ Hardware Architecture Concept

RAM (Main Memory)
Has addresses (e.g., 0x7FFA...)
✅ `&` operator works
CPU Registers
No standard addresses
❌ `&` operator FAILS
1. register keyword: Data is stored directly in the CPU register for fast access.
2. The Pointer: We create a pointer piSumData.
3. The Crash Line: piSumData = &Data; attempts to fetch the memory address of a CPU register. Because registers do not have standard memory addresses in C, the compiler stops immediately.
// Output:
Compilation Error: Cannot take the address of a register variable.
Instructor Homework

Test Your Knowledge

Look at the code from Question 4. If we remove the word register, what will happen?

int main() {
    int Data = 10; // "register" removed, now standard auto variable
    int *piSumData = NULL;
    
    piSumData = &Data; 
    *piSumData = 5;
    
    printf("%d", *piSumData);
}
Student Note / Answer:

By removing register, the variable Data defaults to an auto storage class. This means it is successfully stored in standard RAM. Because it is in RAM, it has a valid memory address. The address-of operator (&) works perfectly. The pointer points to `Data`, changes its value to 5, and the printf will successfully output 5.
