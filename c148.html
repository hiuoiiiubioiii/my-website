Topic 1 The Basics: What is #define?

Instructor Audio Note: "Preprocessing is a step that happens before your code is actually compiled. You can use #define to create constants. The preprocessor will search your code and literally text-replace the name with the value before the compiler ever sees it."

#define A 10

Whenever you use A in your program, it gets replaced by 10.
Do NOT put a semicolon (;) at the end! 
#define A 10

int main() {
    int x = A; // Preprocessor changes this to: int x = 10;
    printf("%d", x);
}
💡 Instructor's Golden Rule

Always write macro names in UPPERCASE (e.g., A, PI, MSG). This is a standard coding convention so programmers instantly know it is a preprocessor macro and not a normal variable.

Topic 2 Defining Strings & Decimals

Instructor Audio Note: "The first part is the Macro Name, and the second part is the Character Sequence. You are not limited to just integers. You can define floating-point numbers or long text strings so you don't have to retype them repeatedly."

Macro Name | Character Sequence
#define PI     | 3.14159
#define MSG    | "Jenny's Lectures CS/IT NET&JRF" 
#define PI 3.14159
#define MSG "Jenny's Lectures CS/IT"

int main() {
    int r = 5;
    printf("Area: %f", PI * r * r); // Evaluates to 3.14159 * 5 * 5
    printf("%s", MSG); // Prints the long string
}

Topic 3 Function-Like Macros (Crucial Exam Topic)

Instructor Audio Note: "You can define macros that act like functions by passing arguments. This is faster than a normal C function because there is no 'function call overhead'. It is pure text substitution."

#define MUL(a, b) a * b

If we pass: MUL(2, 3)
Code becomes: 2 * 3 (Compiler calculates 6)
🚨 The Ultimate Exam "Gotcha"

Students get this wrong in exams and interviews all the time. What happens if we pass equations instead of single numbers?


Code: MUL(5-2, 7+4)


  • What you THINK happens: a becomes 3. b becomes 11. Answer = 33. (WRONG!)
  • What ACTUALLY happens: The preprocessor is "dumb." It doesn't do math; it only does copy and paste.
  • It substitutes text exactly: 5 - 2 * 7 + 4
  • The compiler applies BODMAS (Multiplication first): -2 * 7 = -14
  • Equation is now: 5 - 14 + 4
  • Final Answer: -5

Topic 4 Multi-line Macros using Backslash (\)

Instructor Audio Note: "Sometimes your macro logic is too long to fit on one line. If you hit 'Enter', C will throw an error. To fix this, use the backslash character (\) at the end of the line to tell the compiler 'this macro continues on the next line'."

// The backslash \ connects multiple lines into one macro
#define MAX(a, b) if(a > b) \
    printf("%d is max", a); \
else \
    printf("%d is max", b)

int main() {
    MAX(5, 6); // Will output "6 is max"
}
⚠️ String Substitution Note

If you write printf("a is max"); inside the macro, the compiler will literally print the letter "a". It will not replace the letter "a" inside double quotes with the number 5. You must use the %d format specifier to print variable values.

Topic 5 Destroying Macros with #undef

Instructor Audio Note: "If you are done using a macro and want to remove it so it cannot be used anymore in the rest of the file, you use #undef. It frees up the macro name to be reused, or simply prevents accidental usage down the line."

#define MAX(a,b) a > b ? a : b

printf("%d", MAX(10, 20)); // Works fine

#undef MAX // Macro is destroyed here!

printf("%d", MAX(5, 6)); // CRASH! Implicit declaration error. 
#define MAX(a,b) a > b ? a : b

int main() {
    printf("%d", MAX(10, 20)); 

    #undef MAX // Destroying the macro

    // If you uncomment the line below, the program will not compile
    // printf("%d", MAX(5, 6)); 
}
