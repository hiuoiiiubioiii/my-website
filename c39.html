The `do-while` Loop in C

Instructor: Jayanti Khatri Lamba | Complete Visual Study Guide

🧭 1. Core Concept: Entry vs. Exit Control

To understand the do-while loop, we must first compare it to the loops we already know:

The for and while loops are ENTRY-Controlled Loops.
They check the condition before entering the body of the loop. If the condition is false from the start, the loop body never runs.
The do-while loop is an EXIT-Controlled Loop.
It enters the loop body immediately, executes the code, and checks the condition at the exit.

💻 2. Syntax & The Semicolon Rule

Like all loops, it has three main factors: Initialization, a Condition, and an Update/Modifier. Here is exactly how to write it:

// Step 1: Initialization (done before the loop)
Initialization; 

do 
{
    // Step 2: Body of the loop (executes first!)
    Statement 1;
    Statement 2;
    
    // Step 3: Increment or Decrement (Modify expression)
    Modify / Update; 
} 
while (Condition); <-- Step 4: Condition check
⚠️ THE MOST COMMON MISTAKE:
You MUST put a semicolon (;) at the very end of while(Condition);. In a standard while loop, we don't do this. In a do-while loop, omitting this semicolon will cause a syntax error!

🛤️ 3. Visualizing the Flow (Flowchart)

Because it is an exit-controlled loop, the control flow looks like this:

Statement Block (Body)
Increment / Decrement
True
Condition
False
Exit Loop

🏆 4. The Golden Rule of do-while

Based on the flowchart above, the instructor emphasizes the most important rule for exams and interviews:

"The body of the do-while loop will be executed AT LEAST ONCE, irrespective of the condition. Even if the condition is false on the very first try."

⚖️ 5. Practical Proof: do-while vs while

The instructor writes two programs side-by-side to prove the Golden Rule. Let's do a dry run of both.

Program A: do-while

void main() {
    int i = 0;
    do {
        printf("Jenny\n");
    } while (i > 0);
    
    printf("Out of loop");
}

Dry Run: Control enters the do block instantly. "Jenny" prints. Then it checks 0 > 0 (False). It exits the loop.

> Jenny
> Out of loop

Program B: while

void main() {
    int i = 0;
    while (i > 0) {
        printf("Jenny\n");
    }
    
    printf("Out of loop");
}

Dry Run: It checks 0 > 0 immediately. It is False. The control never enters the loop body.

> Out of loop

📝 6. Student Assignment (Test Yourself!)

At the end of the video, the instructor adds one line of code (i++;) to the do-while program. What will the output be?

void main() {
    int i = 0;
    do {
        printf("Jenny\n");
        i++;  // <--- THIS WAS ADDED!
    } while (i > 0);
    
    printf("Out of loop");
}

Question: Will this be a finite loop or an infinite loop? How many times will "Jenny" print?

🧠 Answer Breakdown:

It creates an Infinite Loop! Here is the step-by-step trace:

  1. i starts at 0.
  2. Enters loop: Prints "Jenny".
  3. i++ executes: i becomes 1.
  4. Condition check: while (1 > 0). This is TRUE. Loop repeats.
  5. Enters loop: Prints "Jenny".
  6. i++ executes: i becomes 2.
  7. Condition check: while (2 > 0). This is TRUE. Loop repeats.

Because i keeps getting larger, it will always be greater than 0. "Jenny" will print endlessly!
