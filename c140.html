🔍 1. The Anatomy of a Variable

When you declare a variable in C, you are telling the compiler about its name, value, and data type. But there is a hidden property associated with every variable: The Storage Class.

int 1. Type
a 2. Name
= 10; 3. Value
auto 4. Storage Class (Hidden)

🛠️ 2. The 4 Properties Defined

If you don't assign a value to a variable, how does the compiler handle it? The Storage Class answers four crucial questions about your variable:

1. Default Value ❓

If uninitialized, what value does it hold? (e.g., 0 or a random "garbage" value).

2. Location 💾

Where is it stored in the hardware? (RAM/Main Memory or CPU Registers).

3. Scope 🌍

Where can you access this variable? (Inside a block, a function, or everywhere).

4. Lifetime ⏳

Till when is this variable alive before the memory is destroyed?

🗂️ 3. Types of Storage Classes

There are four storage class keywords in C. They are placed right before the data type.

auto int a;
register int b;
extern int c;
static int d;
Instructor Note: If you write int a; without a keyword, the compiler automatically assigns the auto storage class by default. Writing int a; is exactly the same as writing auto int a;.

🎯 4. Deep Dive into "Scope"

Scope defines the physical boundaries in your code where a variable is allowed to be accessed. The compiler resolves variables by looking from the inside out.

Program Scope (Global Variable)

Accessible anywhere in the file.

Function Scope

Accessible only inside this specific function.

Block Scope { }

Variable dies when the closing brace } is reached.

🖊️ 5. Whiteboard Code Dry Run

What happens when multiple variables have the same name but exist in different scopes? The compiler always looks for the closest, most local variable first.

#include <stdio.h>

int a = 50;             // GLOBAL SCOPE

void fun() {
    int a = 30;         // FUNCTION SCOPE (fun)
    printf("%d\n", a);
}

int main() {
    int a = 10;         // FUNCTION SCOPE (main)
    fun();              // Output: 30

    {                   // BLOCK 1 START
        int a = 20;     // BLOCK SCOPE
        printf("Inside Block %d\n", a); // Output: 20
    }                   // BLOCK 1 END (a=20 dies)

    {                   // BLOCK 2 START
        printf("%d\n", a++); // Looks outside! Output: 10
    }                   // BLOCK 2 END

    printf("%d\n", a);  // Output: 11 (main's 'a' was incremented)

    return 0;
}

💻 6. VS Code Practical Demo & Errors

In the IDE, the instructor proved the whiteboard theory. She also demonstrated what happens when you violate Scope boundaries.

If you declare int b = 89; inside fun1(), and try to print b down inside main(), the compiler will fail.

Terminal / Compiler Output
error: ‘b’ undeclared (first use in this function)
25 | printf("%d", b);

Conclusion: A variable's lifetime and scope are strictly bound to the braces { } it was created in. Once the function fun1() ends, b is destroyed. main() has no idea it ever existed.
