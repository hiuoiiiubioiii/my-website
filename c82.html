Dangling Pointers in C

Complete breakdown from Jenny's Lectures | Scroll to Study

Core Concept

What is a Dangling Pointer?

A dangling pointer is a pointer that points to a memory location that has been freed, deleted, or is no longer valid.

It is "hanging" or "dangling" because it holds an address that no longer belongs to the program. Dereferencing (using) it leads to undefined behavior, garbage values, or program crashes.

Jenny's Phone Analogy:
Imagine you have your friend's phone number saved. Your friend changes their number. You still have the old number saved (the pointer), but if you call it, you will either get a disconnected line or a total stranger (garbage value).
Cause #1

De-allocation of Dynamic Memory

This is the most common cause. It happens when you allocate memory dynamically on the Heap using malloc(), point to it, and then free() that memory without updating the pointer.

int *ptr = (int *)malloc(sizeof(int));
*ptr = 10;

// Memory is deleted here!
free(ptr);  

// ptr is now a DANGLING POINTER
printf("%d", *ptr); // Undefined behavior!

As Jenny explains: The memory block goes back to the free pool. But ptr still holds the address 100. It hasn't been erased.

The Fix

Immediately after freeing a pointer, set it to NULL.

free(ptr);
ptr = NULL; // It is no longer dangling; it is a Null Pointer.
Cause #2

Variable Going Out of Scope

Variables declared inside curly braces { } have "local scope." They are destroyed from the Stack as soon as the program exits those braces.

int main() {
    int *ptr; 
    
    { // Inner Block starts
        int a = 9;
        ptr = &a; 
    } // Block ends. 'a' is destroyed!
    
    // ptr is now a DANGLING POINTER
    printf("%d", *ptr); 
}

Practical Demo Note:
When Jenny ran this in VS Code, it actually still printed 9. Why? Because the OS hasn't overwritten that specific spot in memory yet. However, this is fundamentally unsafe and is technically a bug.
Cause #3

Returning Local Variables from Functions

This happens when a function returns the memory address of a variable created locally inside that function.

int* f() {
    int a = 10;
    return &a; // Returning address of local var!
}

int main() {
    int *ptr = f(); 
    // ptr is now a DANGLING POINTER
}
warning: function returns address of local variable [-Wreturn-local-addr]

When f() finishes executing, its entire Stack frame is destroyed. ptr in main is now pointing to a destroyed memory location.

The Fix

If you must return a local variable's address, declare the variable as static. Static variables have a global lifespan and survive until the program completely finishes.

int* f() {
    static int a = 10; // Survives function exit!
    return &a; 
}
Summary

Key Takeaways

  • Definition: Pointers pointing to freed or out-of-scope memory.
  • Risk: Causes undefined behavior, garbage values (e.g., 20448080), or crashes.
  • Prevention:
    • Set pointers to NULL immediately after using free().
    • Understand variable scope (lifespan of { }).
    • Use static variables if returning local addresses from functions.
