File Handling in C

Scroll to learn the core concepts, architecture, and terminology.

1

The Basics: RAM vs. Hard Disk

Before writing code, we must understand how a computer stores data. The instructor draws two memory blocks on the board to illustrate where programs live versus where files live.

Key Takeaways:

  • RAM (Volatile): When you run your C program, variables are stored here. They disappear when the program ends.
  • Hard Disk (Non-Volatile): To store data permanently (like output or logs), we must save it here.
  • What is a File? Simply a sequence of memory bytes stored permanently on the hard disk.
2

The Problem: Accessing the File

Your C program runs in RAM. It cannot directly reach into the Hard Disk to manipulate `abc.txt`. It needs a special pointer to track the file once it is loaded into memory.

int *ptr;
int a;
ptr = &a; // Valid: int pointer points to int

float *fptr;
fptr = &a; // INVALID! A float pointer cannot point to an int.

The Pointer Rule: Pointers are strictly typed. Since files are not integers or floats, we cannot use standard pointers to access them. We need a File Pointer.

3

The FILE Data Type

C provides a predefined data type just for this problem, called FILE (always written in all caps). It is defined inside the standard input/output header file.

#include <stdio.h>

// Inside stdio.h, FILE is defined like this:
typedef struct {
    // Memory Pointer
    // Buffer size
    // End of file indicator
    // etc...
} FILE;

Because it's already a `struct` defined in <stdio.h>, you don't write the struct yourself. You just include the header and use the FILE type directly.

4

Behind the Scenes: The Architecture

This is the most critical concept. When we create a FILE *fp; pointer, it does not point directly to the hard disk.

How it actually works:

  • Your pointer fp points to a buffer (the FILE structure) created inside the RAM.
  • When you read a file, a copy of the hard disk file is pulled into this RAM buffer.
  • When you write to a file, you write to this RAM buffer, which the OS later syncs to the physical Hard Disk.
5

Opening a File: fopen()

To create the RAM buffer and link it to the hard disk, we use the fopen() function.

FILE *fp;
fp = fopen("abc.txt", "r");

What does fopen() do?

  1. It creates the FILE structure buffer in RAM.
  2. It links that buffer to the file (e.g., "abc.txt") on the hard disk.
  3. It returns the memory address of the RAM buffer, which we store in fp.

The 6 File Modes

The second argument in double-quotes is the mode. It tells the OS what you intend to do with the file.

"r"Read
"w"Write
"a"Append
"r+"Read + Write
"w+"Write + Read
"a+"Append + Read
6

Closing a File: fclose()

When your program is finished reading or writing, it is your responsibility to close the file.

fclose(fp);

Why is this important?

  • It releases/frees the memory used by the buffer in RAM.
  • It breaks the link between your program and the physical file.
  • It ensures any pending data in the buffer is completely written to the hard disk, and makes the file available for other programs to use.
7

Preview: Data Operations

Once fp is successfully pointing to the opened file buffer, you can use these functions (covered in future lectures) to manipulate the data:

  • fprintf() / fscanf() - Read/Write formatted data
  • fputc() / fgetc() - Read/Write single characters
  • fputs() / fgets() - Read/Write entire strings
✓

Summary: The 4 Golden Steps

To successfully handle files in C, you must always perform these four steps in exact order:

  1. 1. Declare a file pointer: FILE *fp;
  2. 2. Open the file: fp = fopen("filename.txt", "mode");
  3. 3. Perform operations: fscanf(), fprintf(), etc.
  4. 4. Close the file: fclose(fp);
