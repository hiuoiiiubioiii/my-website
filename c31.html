Switch Statements in C

Complete Audio/Visual Breakdown • Continuous Study Feed

👋 1. What is a Switch Statement?

In C, the switch statement is a multi-way decision statement. It acts as an alternative to long, unreadable else-if ladders.

Why use it?

  • It makes your program shorter, more readable, and easier to understand.
  • It executes faster than a long if-else block because the compiler can often optimize the jump directly to the matching case.
💡 Instructor Note: "Whatever you do with an if-else block, you can do using a switch statement. It is a simplified version of a long if-else construct."

📐 2. The General Syntax

Here is the exact blueprint written on the whiteboard. Pay attention to the keywords, colons, and breaks.

syntax_blueprint.c 
switch (expression) {
    case value1:
        // Block of statements 1
        break; // Exits the switch
        
    case value2:
        // Block of statements 2
        break;
        
    // ... multiple cases allowed ...
        
    default:
        // Executes if no case matches
        // No break needed here at the end
}
// Statement-x (Rest of program)

⚙️ 3. Execution Walkthrough (Integer Example)

Let's see how control moves through the switch using a simple integer program.

integer_example.c 
int a;
printf("Enter a: ");
scanf("%d", &a);

switch(a) {
    case 1: 
        printf("JK\n");
        break;
    case 2: 
        printf("Jenny\n");
        break;
    default:
        printf("Bye\n");
}
printf("go to home\n");
Scenario 1 (User enters 1): Switch checks a. 1 == 1 is true. Jumps to case 1. Prints "JK". Hits break and exits. Finally prints "go to home".

Scenario 2 (User enters 3): Case 1 fails. Case 2 fails. Control drops to default. Prints "Bye". Exits and prints "go to home".

⚠️ 4. The 6 Golden Rules of Switch

The instructor spent heavy emphasis on these facts. These are common exam and interview questions!

1. Expressions are Allowed (Must be Integer/Char)

You can write switch(a * 2) or switch(a + b). As long as the math evaluates to a whole integer or a character, it is valid.

2. No Floating-Point Numbers ❌

You cannot use float or double in the expression or the case labels. switch(1.5) will cause a compiler error.

3. Case Order Does Not Matter

You do not have to put cases in sequential order (1, 2, 3). You can legally write case 10: followed by case 3:.

4. The Danger of "Fall-Through"

If you omit the break; statement, the code will execute the matched case AND all cases below it until it hits a break. This is called falling through.

5. Default is Optional (and Movable)

You don't have to include default:. If you do, it doesn't have to be at the bottom (though it is standard practice to put it there).

6. No Relational Operators in Cases ❌

A case checks for strict equality. You cannot write case marks > 80:. Case labels must be specific constants (like case 5: or case 'A':).

🔤 6. Characters & Intentional Fall-Through

You can evaluate characters instead of integers. By deliberately omitting break;, you can group cases together to execute the same block of code. Here is the whiteboard example testing for vowels.

vowel_checker.c 
char ch;
printf("Enter ch: ");
scanf("%c", &ch);

switch(ch) {
    // Notice the lack of breaks!
    case 'a':
    case 'e':
    case 'i':
    case 'o':
    case 'u':
        printf("Vowel\n");
        break; // Break triggers ONLY after printing
        
    default:
        printf("Not Vowel\n");
}

How it works: If you enter 'e', it matches case 'e':. Because there is no break, gravity pulls execution straight down through 'i', 'o', and 'u' until it hits the print statement and the break.
