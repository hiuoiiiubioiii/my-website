Mastering the for Loop in C

An in-depth, interactive study guide based on Jenny's Lectures.

1. Why Do We Need Loops?

Loops are used in programming when you want to execute a single statement, or a block of multiple statements, repeatedly.

The for loop is arguably the most popular loop in the C language. Programmers prefer it because it consolidates everything (initialization, condition, and updating) into a single, clean line of code.

Key Concept: "Entry-Controlled Loop"
The for loop is an entry-controlled loop. This means the compiler checks the test condition before entering the body of the loop. If the condition is false on the very first try, the loop body will not execute even once.

2. Anatomy of the for Loop

Here is the absolute standard syntax you must memorize:

for (initialization; condition; update/modify)
{
    // Body of the loop (statements to be executed)
}

Crucial Syntax Rules:

  • Semicolons are Mandatory: Notice the two semicolons ; separating the three expressions inside the parentheses. You will get a syntax error without them.
  • Curly Braces { }: If your loop body only has one statement, curly braces are optional. If it has multiple statements, you must enclose them in braces.

3. Step-by-Step Execution (Visualization)

Let's look at exactly how the C compiler reads a loop designed to print numbers from 1 to 3 (shortened from 10 for demonstration).

The 4-Step Flow:

  1. Initialization: Happens only once. Memory is allocated and set.
  2. Condition Check: If True, enter loop. If False, exit loop completely.
  3. Loop Body: Execute the statements inside.
  4. Update: Modify the variable, then jump straight back to Step 2.
int i;
for (i = 1; i <= 3; i++) {
printf("i = %d\n", i);
}

5. Tricky Facts & Syntax Variations

You don't always have to write loops exactly the same way. All the snippets below will execute the loop body exactly 10 times:

  • for (i = 1; i <= 10; i++) // Standard (1 to 10)
  • for (i = 0; i < 10; i++) // Zero-indexed (0 to 9)
  • for (i = 0; i <= 9; i++) // 0 to 9
  • for (i = 2; i <= 11; i++) // 2 to 11

Every Expression is Optional

One of the most interesting facts about the for loop is that the three inner expressions are completely optional. Only the semicolons are required!

// Valid: Initializing outside the loop
int i = 1;
for ( ; i <= 10; i++) { ... }

// Valid: The Infinite Loop
// If you omit the condition, C assumes it is always TRUE.
for ( ; ; ) { 
    printf("This runs forever!\n"); 
}

6. Check Your Understanding

The instructor left you with a tricky problem. What is the output of the following C code?

int i;
for (i = 10; i < 10; i++) 
{
    printf("%d", i);
}

Output: [Nothing]

Explanation: Let's trace the 4 steps:

  1. Init: i is set to 10.
  2. Condition: Is 10 strictly less than 10? No (False).
  3. Because this is an entry-controlled loop, it fails the very first test. Control immediately skips the body entirely. Nothing is printed!
