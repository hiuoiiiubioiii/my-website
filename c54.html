2D Arrays in C: Initialization

In the previous lesson, we learned the need for 2D arrays and how to declare them. Today, we learn how to put data into them. There are two main ways to initialize a 2D Array:

  • 1. At Compile Time (Hardcoding values directly in the code)
  • 2. At Run Time (Taking input from the user during execution)

Part 1: Compile-Time Initialization

Initializing at compile time means you give the array its values at the exact moment you declare it.

1. The Linear List Syntax

int a[2][3] = {0, 0, 0, 1, 1, 1};

Memory Math: Total elements = 2 rows × 3 cols = 6 elements.
Size = 6 elements × 4 bytes (size of int) = 24 bytes allocated.

2. The Grouped & Matrix Syntax (Better Readability)

To make the code easier for humans to read, group the rows using nested curly braces. You can even break it across multiple lines. The compiler treats it exactly the same.

int a[2][3] = {
    {0, 0, 0},  // Row 0
    {1, 1, 1}   // Row 1
};

⚠️ Crucial Rules & Padding Variations

What happens if we don't provide all the numbers? The C compiler has strict padding rules.

Variation A: Partial Grouped Initialization

int a[2][3] = {{0, 0}, {1, 1}};

We need 3 columns, but only gave 2 per row. Rule: The missing columns at the end of each row are automatically padded with 0.

Variation B: Omitting the Row Size

int a[][3] = {{0, 0, 0}, {1, 1, 1}};

Rule: You can leave the first bracket empty. The compiler counts the inner braces and knows there are 2 rows. Note: You CANNOT omit the column size. `a[2][]` causes an error!

Variation C: The "All Zeros" Trick

int a[2][3] = {0};

You only provided a single `0`. The compiler fills the remaining 5 elements with 0s, resulting in a fully empty grid.

📝 Instructor's Homework:
Q1: int a[][3] = {1, 1, 1}; → Result: A 1x3 array. Values: 1, 1, 1.
Q2: int a[][3] = {1, 1, 1, 2}; → Result: Compiler infers 2 rows. First row gets [1,1,1]. Second row gets [2,0,0].

Part 2: Run-Time Initialization

Initializing at run time uses scanf inside nested loops so the user can type values when the program runs.

  • The outer loop (i) controls the rows.
  • The inner loop (j) controls the columns.
int a[2][3]; // Uninitialized array has Garbage Values

for (int i = 0; i < 2; i++) {
    for (int j = 0; j < 3; j++) {
        scanf("%d", &a[i][j]); 
    }
}
