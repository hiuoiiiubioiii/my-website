🧵 Strings in C: Reading at Runtime

In-depth Study Notes & Memory Visualizations

🤔 Instructor's Prerequisite Question

Is 'string' a data type in C? Yes or No?

Answer: No! C does not have a primitive "string" data type. A string is simply an array of characters ending with a Null Character (\0).

1. Recap: Strings vs. Character Arrays

Before moving to runtime input, let's look at how strings are stored in memory when hardcoded (compile-time initialization).

char s1[10] = "Jenny";

The Visual Breakdown:

  • Because an array size of 10 is declared, the compiler allocates 10 bytes of memory.
  • The name of the array (s1) holds the base address (e.g., 1000).
  • The compiler automatically adds the Null Character (\0) at the end.
  • Without the \0, it is just a character array, not a string! The remaining blocks contain garbage values.

2. Reading Strings using scanf()

How do we read a string while the program is running (runtime) instead of hardcoding it?

char name[30]; 
printf("Enter name: ");
scanf("%s", name);

Instructor's Golden Rules for String input:

  • Format Specifier: We use %s to read strings.
  • NO Ampersand (&): Unlike scanf("%d", &age);, we do not write &name. Because name is an array, its name already points to the base memory address!
  • NO For-Loops: Unlike an integer array where you must loop to scan each number, %s reads the whole sequence automatically.

3. The Major Drawback of scanf()

What happens if the user types their full name with a space?

4. The Solution: Using gets()

To overcome the whitespace limitation, C provides the gets() function (Get String).

char name[30];
printf("Enter name: ");
gets(name);  // Replaces scanf
  • No format specifier (%s) is needed. Just pass the array name.
  • gets() reads the entire line, including spaces, and only stops when you press the ENTER key.
  • When you press Enter, it automatically appends \0 at the end.

🚨 5. The DANGER of both Functions: Buffer Overflow

This is a critical computer science concept. Both scanf and gets are "blind" functions. They do not perform bounds checking.

char name[5];  // Very small array! Size is 5.
gets(name);

User Input: Jayanti Khatri Lamba

We only asked for 5 bytes, but we typed a massive string. Because gets doesn't care about size, it will keep writing data, destroying whatever is sitting in the memory addresses *after* our array.

Danger! Overwriting unallocated memory!

Why is this bad? The memory after your array belongs to other variables, other programs, or your OS. Overwriting it causes software crashes, data corruption, and severe security vulnerabilities. (This is why gets() is considered highly unsafe in modern C).

6. A Hack to fix scanf() Overflow

While gets() cannot be fixed easily, there is a trick you can use with scanf() to prevent buffer overflows.

char name[5];
scanf("%4s", name);
  • By adding a number to the format specifier (%4s), you instruct scanf to read a maximum of 4 characters.
  • The (N-1) Rule: If your array size is 5, you must write %4s. Why? Because you MUST leave 1 byte of space for the compiler to add the Null Character (\0).
  • Drawback: Hardcoding numbers into format specifiers is generally considered bad practice.
