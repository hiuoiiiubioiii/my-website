Static Storage Class in C

Complete Whiteboard & IDE Breakdown

⚠️ 1. Introduction & Exam Importance

Out of the 4 storage classes in C (auto, register, static, extern), the instructor notes that static is the most important.

You will see it in 95% of competitive programming exams (GATE, NET, etc.). Students frequently fail these questions because they completely misunderstand the difference between the variable's Scope and its Lifetime.

✍️ 2. Syntax & Declaration

To declare a static variable, use the static keyword followed by the data type and variable name. It works with all data types.

static int y;
static char ch;
static char str[20];
static float x;

⚙️ 3. The 4 Properties of Static Variables

Every storage class defines 4 specific rules for a variable. Here is how the compiler treats static:

Property Behavior
1. Default Value 0 (int), 0.000000 (float), null (char). Unlike auto, there is NO garbage value.
2. Location Stored in Main Memory (RAM), not in CPU registers.
3. Scope Within Block: It can only be accessed inside the specific {} where it was created.
4. Lifetime Till end of program: It does NOT die when the function finishes running!
"Scope = Where can I see it? Lifetime = How long is it alive?"
A static local variable stays alive in memory, but you can only see/access it when you are inside its specific function.

🧠 4. The Whiteboard Dry-Run (Tracing Code)

The instructor writes this program to prove how "Lifetime" works. Let's trace it exactly as she does on the whiteboard.

void display(); // function prototype

void main() {
    display();  // First call
    display();  // Second call
}

void display() {
    static int x;    // Defaults to 0
    x += 10;         
    printf("\n x = %d", x);
}

Execution Visualization:

Step 1: First Call to display()

x
10

The compiler creates 'x', defaults it to 0, adds 10. Output: x = 10

Step 2: Second Call to display()

x
20

The line static int x; is SKIPPED. 'x' retains its old value of 10. We add 10 more. Output: x = 20

Instructor Quote: "It's like the birth of 'x'. Birth happens only once. Since it is still alive from the first call, it won't take birth again. The initialization line is skipped."

🗺️ 5. Memory Layout Explanation

Why does static remember its value, but auto suffers from amnesia? Because of where it lives in your RAM.

Heap segment
Global / Static Segment
(Where our 'x' lives permanently)
Stack Segment
(Where normal auto variables live and die)
Code / Instructions

Normal variables are created on the Stack. When a function ends, that part of the stack is wiped out (the variable dies). Static variables are stored in the Global/Static segment, which remains untouched until the program entirely finishes running.

Warning: Because static variables hold memory for the whole program, overuse can waste RAM.

💻 6. Practical IDE Demonstration Takeaways

The instructor jumps into VS Code and proves three crucial concepts:

  • The Split Test: She creates static int x = 10; and int y = 10; in the same function. Over two calls, x grows (20, 30) because it's static, while y constantly resets to its base calculation because it is an auto variable suffering from amnesia.
  • Proving Local Scope: She tries to write printf("%d", x); inside main(). The compiler throws an Error: 'x' undeclared. Even though 'x' is alive in the global memory segment, the main block is strictly forbidden from "seeing" it.
  • Default Nulls: She declares static char ch; globally. When printed, nothing shows up on the screen, proving the default value is a null character, not a random garbage symbol.
