The C Preprocessor

Complete Student Notes, Visualizations, and Breakdown

1. Introduction: What is a Preprocessor?

In every C program, you often see the # symbol (like #include <stdio.h>). This symbol represents a preprocessor directive.

  • "Program": The preprocessor is a separate piece of software/code.
  • Not a part of the compiler! This is crucial. The preprocessor finishes its job completely before the compiler even starts looking at your code.
  • "Text Substitution Tool": Its primary function. It simply replaces certain text (like macros) or pastes files into your code before compilation begins.

2. The Compilation Pipeline

When you click "compile," the compiler doesn't run immediately. The Preprocessor steps in first. It takes your raw C code and prepares it.

first.c
(Your raw code)
➔
Preprocessor
➔
first.i
(Expanded code)
➔
Compiler

Note: The preprocessor generates an intermediate file (often with a .i extension). This file contains your code, but all lines starting with # have been removed and resolved.

3. Preprocessor Directives (The '#' Commands)

Any line starting with # is a command directed only at the preprocessor. The actual C compiler does not understand lines starting with #.

#include <stdio.h>
#include <string.h>
#define PI 3.14

Because the compiler doesn't understand these, the preprocessor resolves them and deletes the # directives before handing the code over to the compiler.

4. Deep Dive: How #include Works (File Expansion)

Header files like stdio.h contain the actual code and definitions for standard functions like printf() and scanf().

When the preprocessor sees #include <stdio.h>, it literally copies the entire contents of that system file and pastes it into your program.

stdio.h
string.h
⬇
first.i (Intermediate File)

/* Hundreds of lines copied from stdio.h pasted here */
/* Lines copied from string.h pasted here */

int main() { ... } /* Your original code */

In this new expanded file, the #include lines no longer exist!

5. Deep Dive: How #define Works (Macro Substitution)

#define creates a Macro. The preprocessor acts like a "Find and Replace" tool.

If you write #define PI 3.14, the preprocessor searches your entire document. Everywhere it finds the word "PI", it deletes it and substitutes it with "3.14".

Area = PI * r * r;
➔
Area = 3.14 * r * r;

By the time the compiler reads the code, it doesn't see "PI * r * r"; it only sees the actual numbers.

6. List of Other Preprocessor Directives

Aside from include and define, there are several other directives you will encounter (to be covered in future lessons):

  • #if, #else, #elif : Used for conditional compilation.
  • #undef : Used to un-define a macro you previously created.
  • #pragma : Used to give special, compiler-specific instructions.

7. Crucial Distinction: < > vs. " " in #include

This is a very common exam question and essential coding concept.

#include <stdio.h> (Angle Brackets)
Tells the preprocessor to look for the file in the Standard System Directories (where the C language stores its official libraries).
#include "support.c" (Double Quotes)
Tells the preprocessor to look for the file in your Local/Current Directory first (the exact same folder where your main .c file is saved).

Note: You can also put a complete file path inside the double quotes (e.g., "C:/Users/Student/support.c") if the file is saved somewhere else on your computer.

8. Summary of the Process

  • Step 1: You write first.c (Contains `#` directives).
  • Step 2: Preprocessor runs. It expands included files and substitutes macro text.
  • Step 3: Preprocessor outputs first.i (Expanded source code, containing NO `#` symbols).
  • Step 4: Compiler takes first.i and turns it into object code (.obj).
  • Step 5: Assembler and Linker finish the job to create the final executable program.
