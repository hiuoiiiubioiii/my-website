1. What are Loops & Why do we need them?

Loops are fundamental programming constructs built for Iteration. They are used when you want to execute a specific statement (or a block of statements) repeatedly, multiple times in your program.

The Supermarket Analogy

To understand why iteration is necessary, the instructor used a real-life analogy about grocery shopping.

The Bad Way (No Loops)

Go to market -> Buy Mango -> Come home.
Mom says "Buy Apples" -> Go to market -> Buy Apples -> Come home.
Mom says "Buy Ladyfingers" -> Go to market...

Exhausting and repetitive!
The Good Way (With Loops)

Make a list of 15 items.
Go to market ONCE.
Iterate (loop) through the list, pick up items 1 through 15.
Come home when list is complete.

Efficient and saves time!

2. Translating to C Programming

Let's say you want to print the instructor's name, "Jenny", to the screen.

Printing it 1 time:

#include <stdio.h>
int main() {
    printf("Jenny\n");
    return 0;
}

The Problem: Printing it 100 times

Without loops, you would have to manually type the `printf` statement 100 times. Not only is this tedious, but it makes the code massive, hard to read, and prone to errors.

1. printf("Jenny\n");
2. printf("Jenny\n");
3. printf("Jenny\n");
...
100. printf("Jenny\n");

❌ We NEVER code like this!

The Solution: We write the `printf` statement exactly ONE time inside a loop body, and tell the loop to run 100 times.

3. The Anatomy of a Loop

For a loop to function properly, the computer needs rules. It needs to know exactly where to begin, what to execute, and exactly when to stop.

Every loop in C is built on Three Main Components:

i = 1
1. Initialization
You need a starting point. We create a counter variable (commonly i). You can start at 1, 0, or any number required by your logic.
i <= 100
2. Termination Condition
You must tell the loop when to stop. If you don't provide a condition, the loop will run forever (an Infinite Loop). Here, the loop stops as soon as i becomes 101.
i = i + 1
3. Update Expression
You must update the counter so it progresses toward the termination condition. You can increment (i++), decrement (i--), or use formulas like i = i + 2 or i = i * 2.

4. Types of Loops (Based on Condition Checking)

Loops are classified based on where the condition is checked in the process.

Type A: Entry-Controlled Loops

The condition is checked right at the beginning. If the condition is False initially, the loop body is never executed, not even once.

Analogy: Security checking you at the entry gate of MGF Mall. If you fail the check, you cannot enter.

Initialization (i = 1)
Condition Check
True
Body of Loop: printf("Jenny")
Update (i++)

If Condition is FALSE initially -> Exits immediately

Type B: Exit-Controlled Loops

The loop body executes first, without checking any conditions. The condition is checked at the very end of the loop.

Because of this structure, the loop body is guaranteed to run at least ONE time, even if the condition is completely False from the start.

Analogy: Shopping at "More" supermarket. You enter, put items in your basket, and your bill/items are checked at the exit counter.

Initialization (i = 1)
Body of Loop: printf("Jenny")
Update (i++)
Condition Check

Body executes AT LEAST ONCE before checking True/False

5. Advantages & Summary

By using loops in your C programs, you achieve:

  • Time Saving: Less code to write for repetitive tasks.
  • Code Readability: Your codebase remains small, clean, and professional.
  • Execution Speed: Compilers execute loops extremely efficiently.
