Types of Recursion in C

Complete In-Depth Video Breakdown & Whiteboard Tracing

1. Introduction & Overview

Recursion occurs when a function calls itself directly or indirectly. The instructor recaps the golden rule: understanding the execution flow is the secret to mastering recursion.

The Main Types of Recursion:

  • Direct Recursion (Covered below)
  • Indirect Recursion (Covered below)

  • Non-Tail Recursion (Next video)

2. Direct Recursion

Direct recursion occurs when a function calls itself directly within its own body.

Instructor's Analogy: "It's like I am calling myself directly. Jenny calling Jenny."
Jenny
Jenny

Code Structure:

void fun1() 
{
    // some code
    fun1(); // The function calls itself directly here
}

3. Indirect Recursion (Concepts)

Indirect recursion occurs when a function calls itself indirectly through another function (or a series of functions) forming a circular chain.

The Analogy (Whiteboard Visualized)

If Jenny calls Payal, and Payal calls Jenny back, that forms an indirect recursive loop.

Jenny
⇄
Payal

(2-Step Indirect Recursion)

It can also involve multiple people (functions). As long as the chain eventually loops back to the start, it is recursion.

Jenny
Payal
Jiya

(Jiya calls Jenny, completing the circle)

Conceptual Code Structure

void fun1() 
{
    fun2(); // fun1 calls fun2
}

void fun2() 
{
    fun1(); // fun2 calls fun1 back (Circular!)
}

4. The Code: Indirect Recursion in C

The instructor writes a complete C program to demonstrate indirect recursion. Note that when functions call each other, you must declare their prototypes at the top.

Fun Fact: The logic below calculates the Factorial of a number (5! = 120), just split across two functions.

#include <stdio.h>

// Function Prototypes declared first
int fun1(int n);
int fun2(int n);

void main() 
{
    printf("%d", fun1(5)); 
}

int fun1(int n) 
{
    if (n <= 1)
        return 1;   // Base condition
    else
        return n * fun2(n - 1); // fun1 calls fun2
}

int fun2(int n) 
{
    if (n <= 1)
        return 1;   // Base condition
    else
        return n * fun1(n - 1); // fun2 calls fun1
}

5. The Dry Run (Tracing the Code)

This is the most critical part of understanding recursion. You must trace the code forward until you hit the base case, and then carefully pass the values backward.

Phase 1: Moving Forward (Calling)

main() calls fun1(5)
↓
fun1(5): returns 5 * fun2(4)
(pauses here)
↓
fun2(4): returns 4 * fun1(3)
(pauses here)
↓
fun1(3): returns 3 * fun2(2)
(pauses here)
↓
fun2(2): returns 2 * fun1(1)
(pauses here)
↓
fun1(1): n <= 1 is True.
Returns 1 (Base Case)

Phase 2: Moving Backward (Returning)

fun1(1) passes 1 up to fun2(2)
↑
fun2(2) resolves: 2 * 1 = 2
↑
fun1(3) resolves: 3 * 2 = 6
↑
fun2(4) resolves: 4 * 6 = 24
↑
fun1(5) resolves: 5 * 24 = 120
↑
main() receives 120.
printf outputs 120.

6. Conclusion & The Golden Rule

The instructor concludes with a vital piece of advice for all students studying recursion:

The Golden Rule: "Moving forward is usually easy. Mainly, students get confused while moving backward. How you moved forward, you have to follow the exact same path backward to pass the return values up."
