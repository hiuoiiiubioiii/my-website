Declaring Variables / Objects of Structure

Instructor: Jayanti Khatri Lamba

1. Built-in vs. User-Defined Data Types

Before jumping into structures, let's look at how we normally declare variables.

  • Built-in Data Types: To create an integer variable, you simply write the data type followed by the variable name.
  • User-Defined Data Types: A structure is a custom data type you create. Just like int, you must declare variables of this new data type to actually use it in your program. These variables are also called Objects of the Structure.
// Built-in variable
int a; 

// User-defined variable (Structure Object)
struct student s;

2. The Core Example: Defining a Structure

Throughout this lecture, we use the following structure template as our main example:

struct student {
    int rollno;
    float marks;
    char name[20];
};
🚨 IMPORTANT: At this stage, NO MEMORY IS ALLOCATED! This code simply creates a "template". Memory is only allocated when you create a variable (object) of this type later.

3. Method 1: Declaring Variables Inside main()

This is the standard and recommended way to declare a structure variable.

Common Student Mistake: Many try to declare the variable by just writing struct s;. This is wrong because struct is just a keyword; the compiler doesn't know which structure you mean. The complete data type name is struct student.

void main() {
    // WRONG: struct s;
    
    // CORRECT:
    struct student s;  // Memory is allocated right here!
}
🧠 How Memory is Calculated:
The memory allocated is the sum of its members.
Memory Allocation for 's'
Object: s
int rollno 4 bytes
float marks 4 bytes
char name[20] 20 bytes
Total: 28 Bytes
💡 Pro-Tip: You can verify this in your IDE by using the sizeof operator:
printf("%d", sizeof(struct student)); or sizeof(s);

4. Method 2: Declaring at the End of Definition

You don't have to declare variables inside main(). You can declare them immediately after the closing bracket } of the structure definition, but before the semicolon ;.

struct student {
    int rollno;
    float marks;
    char name[20];
} s1, s2, s3;  // Declaring 3 objects immediately

Note: While valid, Method 1 is considered better practice. If you need 60 students, writing s1, s2... s60 is terrible code. Instead, you'll use an Array of Structures (covered in a future lecture).

5. Anonymous Structures (Missing Tags)

The word student in struct student is called the Structure Tag. Technically, it is optional.

The Drawback: If you omit the tag, you create an "Anonymous Structure." Because it has no name, it is impossible to declare more variables later inside your main() function!

// Anonymous Structure
struct {
    int rollno;
    float marks;
    char name[20];
} s1;

void main() {
    // Error! You have no structure name to use here.
    struct [ BLANK ] s2; 
}

6. Visualizing Multiple Objects

What happens in memory when we declare two objects: s1 and s2? Every time you declare a new object, it gets its own fresh, separate block of memory based on the template.

7. Common Mistake: Initialization in Template

You cannot store a value inside the structure definition block.

struct student {
    int rollno = 1;  // ❌ FATAL ERROR
    float marks;
    char name[20];
};
Why? Because the definition is just a template—no memory exists yet! You cannot store data where memory has not been allocated. You must initialize values using the objects (like s1) after they are declared in main.
