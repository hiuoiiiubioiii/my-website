Strings in C

Complete In-Depth Breakdown & Visualizations

1. What is a String?

In simple terms, C does not have a built-in data type for strings (like int or float). Instead, a string is a derived data type.

A string in C is simply an Array of Characters. Because of this, it can only store characters (A-Z, a-z, 0-9, spaces, and special symbols).

  • Data Type: Always char.
  • Format Specifier: %s is used to print or read strings.
  • Memory: Each character takes exactly 1 byte of memory.

2. The Null Character (\0)

This is the most crucial difference between a standard character array and a string. A string will always be terminated with a null character (\0).

The compiler uses this invisible character to know exactly where the string ends in memory. The ASCII value of the null character is 0.

3. Valid Ways to Initialize Strings

The instructor highlights several correct methods to initialize strings at compile time.

Method A: String Literal (Implicit Null)

char name[10] = "Jenny";

By using double quotes, the compiler automatically appends the \0 at the end.

Method B: Character Array List (Explicit Null)

char name[10] = {'J', 'e', 'n', 'n', 'y', '\0'};

When using curly braces and single quotes, the compiler does not add the null character. You must write it explicitly.

Method C: Implicit Sizing

char name[] = "Jenny";
// Size automatically becomes 6 (5 letters + 1 null)

char name2[] = {'J', 'e', 'n', 'n', 'y', '\0'};
// Size automatically becomes 6

Method D: Including Spaces and Numbers

char info[] = "Jenny Khatri 123";

Numbers and spaces inside double quotes are treated as standard characters and occupy 1 byte each.

4. INVALID Initialization (Common Errors) ❌

Avoid these syntax errors that will cause compiler failures or memory bugs.

  • Empty brackets with no initialization: 
    char name[]; // ERROR

    The compiler doesn't know how much memory to allocate. You must provide a size or initialize it immediately.

  • Array Out of Bounds: 
    char name[5] = "Jenny"; // ERROR / BUG

    "Jenny" requires 6 bytes (5 for letters + 1 for \0). You only gave it 5 bytes, causing an overflow.

  • Assigning literal after declaration: 
    char name[10];
name = "Jenny"; // ERROR

    The array name is a constant pointer to the base address. You cannot assign a new string literal to it after declaration. (You must use strcpy instead).

  • Assigning array to array: 
    char name[10] = "Jenny";
char s1[10];
s1 = name; // ERROR

    You cannot copy an array by assigning the base address of one variable to another.

5. Homework / Assignment 📝

The instructor left 5 scenarios on the whiteboard. Are they valid or invalid? If valid, what is the allocated size?

1. char S1[50] = "H No-126, Gurgaon";

✅ Valid. Plenty of space (50 bytes) for the characters, symbols, spaces, and the null character.

2. char S2[] = "12345678910";

✅ Valid. Implicit sizing is used. Size will be 12 (11 numbers + 1 null character).

3. char S3[5] = "Hello";

❌ Invalid / Overflow. "Hello" has 5 characters, but it needs a 6th byte for the \0. The size provided is only 5.

4. char S4[] = {'H', 'e', 'l', 'l', 'o', '\0'};

✅ Valid. Implicit sizing with an explicit null character. Size will be 6.

5. char S5[15] = "Jayanti Khatri";

✅ Valid. "Jayanti Khatri" is 14 characters (including the space). With the \0, it is exactly 15 bytes. Fits perfectly.
