Topic 1

Direct String Initialization

Before reading user input, we must know how to declare and initialize strings manually. The instructor shows two ways to do this in C.

Crucial Rule: Strings in C are just arrays of characters. Every valid string must end with a hidden Null Character \0.
  Method A & B 
char name[30] = "jenny"; // Method A: Compiler adds \0 automatically

char name[30] = {'j', 'e', 'n', 'n', 'y', '\0'}; // Method B: Manual

printf("%s", name); // %s tells compiler to expect a string

Topic 2

Reading Input with scanf()

Let's ask the user for their name. Notice that in scanf, we write name instead of &name. Because name is an array, the variable itself acts as a pointer to the memory address!

  main.c 
char name[30];
printf("enter name: ");
scanf("%s", name);
printf("%s", name);
Terminal Output
enter name: jenny khatri
jenny
Why did it ignore "khatri"? scanf automatically stops reading input the exact moment it hits a whitespace (space, tab, or Enter).
Topic 3

Reading Full Lines with gets()

To solve the space limitation of scanf, the instructor introduces gets() (Get String). It reads the whole line including spaces until you hit "Enter".

  main.c 
char name[30];
printf("enter name: ");
gets(name); 
printf("%s", name);
Terminal Output
enter name: jenny's lectures is best channel for cs/it
jenny's lectures is best channel for cs/it
WARNING

The Danger of gets(): Buffer Overflow

The instructor issues a massive warning: Do not use gets() in real-world programming!

Our array capacity is 30 characters. The sentence above is 42 characters. gets() does not check boundaries. It forces the extra 12 characters into memory spaces that don't belong to the array.

Memory overwritten! This causes program crashes and security vulnerabilities.

Topic 4

Printing Output with puts()

Just like gets replaces scanf for strings, puts() can replace printf. The main difference? puts() automatically presses "Enter" (appends a newline \n) after printing.

  main.c 
puts(name);
puts(name);
Terminal Output
jenny
jenny
Topic 5

Format Specifier Magic

You can manipulate exactly how many characters are read or printed by putting numbers inside the %s specifier.

Limit Output (%.5s): Prints only the first 5 characters.
Field Width (%10.5s): Allocates exactly 10 spaces on screen, then prints 5 characters pushed to the right.
  main.c 
// Inputting the word: Jayanti

printf("%.5s\n", name);
printf("%10.5s\n", name);
Terminal Output
Jayan
     Jayan

Safety Hack: You can also use scanf("%5s", name); to prevent buffer overflow. It forces the program to stop reading after 5 characters, ignoring the rest.

Topic 6

Pointer Offset Printing

Usually, printf("%s", name) starts printing from index 0. But what if we want to print starting from the middle of the string? We combine the address operator & with an array index.

  main.c 
// User inputs: Jayanti khatri
printf("%s", &name[2]);
Terminal Output
yanti khatri

How the compiler sees this:

"Go to the memory address of Index 2. Start printing from there until you hit the \0."
