File Handling in C: Reading Files

In-depth study notes based on Jenny's Lectures

1. Introduction & Context

In previous lessons, we covered writing to files using the "w" (write) mode. If a file doesn't exist in write mode, C creates a new one for you.

Reading is different. To read a file, we use the "r" (read) mode. The most crucial rule here is:

The file MUST already exist on your hard drive before you run the program. If it doesn't, the compiler will return an error (NULL).

Functions we will learn:

  • fgetc(): "File get character" - Reads a single character.
  • fgets(): "File get string" - Reads a complete string or line.

2. The Mental Model (How it works behind the scenes)

When you read a file, you are not deleting it from your computer. You are creating a bridge (a pointer) between your Hard Disk and your RAM, copying the data into your program's variables.

Hard Disk (H/D)

abc.txt

"Jenny"
⟵

RAM (Main Memory)

C Program

FILE *fp
char ch = 'J'

The FILE *fp pointer connects to the file in the hard disk. Using fgetc pulls data across the bridge into RAM.

3. Opening the File Safely

Because the file must exist, we must always check if the file opened successfully before trying to read from it.

FILE *fp = NULL;

// 1. Open in Read Mode
fp = fopen("abc.txt", "r");

// 2. Crucial Error Check
if (fp == NULL) {
    printf("Error: File does not exist.\n");
    exit(1); // Stop the program immediately
}

4. Reading Character by Character: fgetc()

The fgetc(file_pointer) function looks at where the pointer is currently located, grabs exactly one character, and automatically moves the internal pointer to the next character.

Reading one character:

char ch;
ch = fgetc(fp); 
// If file has "Jenny", 'ch' now holds 'J'.
// The internal pointer is now looking at 'e'.
printf("%c", ch);

Reading the ENTIRE file using a Loop

To read the whole file, we use a while loop alongside a special function called feof() (File End Of File).

  • feof(fp) returns 0 (False) if we are NOT at the end of the file.
  • feof(fp) returns 1 (True) if we HAVE reached the end.

Therefore, our loop condition is while(!feof(fp)), which translates to: "While we are NOT at the end of the file..."

while (!feof(fp)) {
    ch = fgetc(fp);   // Read current char
    printf("%c", ch); // Print it
}

5. Reading Strings/Chunks: fgets()

Instead of one character, fgets() reads chunks of text. To use it, we must first declare a character array (string) to hold the incoming data.

Syntax: fgets(variable_name, length, file_pointer);

The (N-1) Rule & Null Terminator

If you tell fgets the length is 5, it will actually only read 4 characters from the text file. It reserves the 5th spot to automatically insert a Null Terminator (\0), which C requires to know where a string ends.

Code: fgets(str, 5, fp); reading the word "Jenny"

J
e
n
n
\0

Notice the 'y' is left behind in the file because it hit the size limit!

⚠️ The 3 Stopping Conditions

The fgets function will stop reading and return what it has if it hits ANY of these three conditions first:

  1. It reads N - 1 characters (Size limit reached).
  2. It hits the End of File (EOF).
  3. Crucial: It encounters a Newline Character (\n). If your file has multiple lines, fgets will stop at the end of line 1, even if the size limit hasn't been reached.

Because of rule #3, fgets inside a loop is the perfect way to read a multi-line file line-by-line.

6. Final Practical Implementation

Here is the complete, compilable VS Code example demonstrating how to read a multi-line file perfectly using fgets in a loop.

#include <stdio.h>
#include <stdlib.h>

void main() {
    FILE *fp = NULL;
    char str[50]; // Buffer to hold our lines

    // 1. Open the file
    fp = fopen("abc.txt", "r");

    // 2. Error Check (CRITICAL)
    if (fp == NULL) {
        printf("Error: File not found.\n");
        exit(1);
    }

    // 3. Loop until End Of File
    while (!feof(fp)) {
        // Read up to 49 chars OR until a newline (\n) is hit
        fgets(str, 50, fp);
        
        // Print the string (it includes the newline if it read one)
        printf("%s", str);
    }

    // 4. Always close the file to free memory
    fclose(fp);
}
