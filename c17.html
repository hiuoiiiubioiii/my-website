1. Introduction to Logical Operators

While Relational Operators (like <, >, ==) are used to check a single condition, Logical Operators are used to test multiple conditions simultaneously and combine them into a single Boolean result.

🌟 The Golden Rule of C:
• False is ALWAYS represented by 0.
• True is represented by 1 (or strictly speaking, any non-zero number like 5, -10, etc).

The Three Operators we will cover:

  • && (Logical AND) - Binary operator
  • || (Logical OR) - Binary operator
  • ! (Logical NOT) - Unary operator
// Example: Combining two relational expressions
if (a > b && c < d) {
    // Both conditions must be met
}

2. Logical AND (&&) Binary

The Rule: For && to evaluate to True (1), ALL operands/conditions must be True. If even one is False (0), the entire expression is False.

Multiple Conditions Example:

// Assume a = 10, b = 5
result = a > b && b != 10 && b < 11 && a > 5;
// Result: 1 && 1 && 1 && 1  ➔ 1 (True)

⚠️ Critical Exam Concept: Short-Circuiting

If the compiler reads from left to right and finds a 0 (False), it stops immediately. It knows the whole && expression will fail, so it skips checking the rest!

3. Logical OR (||) Binary

The Rule: For || to evaluate to True (1), ANY ONE of the operands needs to be True. It only results in False if ALL operands are False.

4. Logical NOT (!) Unary

The Rule: This operator simply reverses/negates the Boolean state. True becomes False (0), and False becomes True (1).

!5     // 5 is non-zero (True). NOT True ➔ 0 (False).
!0     // 0 is False. NOT False ➔ 1 (True).

// Assume a = 10, b = 5
!(a > b) // (10 > 5) is 1. !1 ➔ 0

Precedence Note

The ! (NOT) operator has higher precedence than && and ||.

Example: 4 && !0 ➔ !0 becomes 1 first ➔ 4 && 1 ➔ 1.

6. Tracing with Post-Increment

How does short-circuiting interact with variable increments?

int a = 10, b = 5, result;

result = (a > b) && a++;
printf("%d %d", result, a);
  • 10 > 5 is True (1). The compiler checks the next part.
  • a++ is post-increment. It uses the current value (10, which is True) to evaluate the expression.
  • Result logic: 1 && 1 ➔ result = 1.
  • After evaluation, a is incremented to 11.
  • Output: 1 11

Question 1:

int a = 1, b = 6;
int result1 = a-- && ++b;
printf("%d %d %d", result1, a, b);
Output: 1 0 7
Trace: a-- is post-decrement. It uses the current value `1` (True) for the && check. Since it's True, compiler moves to the right. ++b pre-increments `b` to 7 (True).
Result: `1 && 1` = 1. Finally, `a` becomes 0 due to the post-decrement.

Question 2 (Using updated values):

// Currently: a = 0, b = 7
int result2 = --a && ++b;
printf("%d %d %d", result2, a, b);
Output: 1 -1 8
Trace: --a pre-decrements `a` to `-1`. Because `-1` is non-zero, it is True! The compiler moves to the right. ++b pre-increments `b` to 8 (True).
Result: `1 && 1` = 1.
