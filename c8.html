1. Why Do We Need Variables?

If you just want a program to print "Hello World", you don't need variables. However, if you want the computer to perform computational tasks—like finding the maximum of three numbers—you need a way to store data.

Humans use their brains to store numbers temporarily; a computer uses its memory. To store data in the computer's memory so it can be processed later, we use variables.

Computational Task Example

10, 20, 30

We must store these numbers in memory to find the maximum (30).

2. What is a Variable? (The Core Concept)

The computer has chunks of memory. When we want to store numbers, they are placed in random memory locations. Each location has a numeric memory address (like 1000 or 1024).

Because humans cannot easily memorize random memory addresses, we give these locations names (like a, b, c). These names allow us to easily fetch or change the data inside.

🔑 Key Definition: A variable is a named memory location. It is a data name used to store the value of data.

3. Best Practices: Meaningful Names

While you can name your variables a, b, and c, it is better programming practice to give them meaningful names so anyone reading your code understands what the data represents.

// Option 1: Bad Practice
int c = a + b; // Stores 40, but what does 'c' mean?

// Option 2: Best Practice
int sum = a + b; // Clearly indicates this variable holds a sum!

4. Variables "Vary" (They can change)

As the name suggests, a variable's value can change during the execution of a program. The old value gets overwritten by the new value.

Exception: The only time a variable cannot change is if you put the keyword const in front of it.

5. Declaration and Initialization

Before you can use a variable, you must tell the compiler two things:

  • What type of data it will hold (Datatype)
  • What its name is (Variable name)

This process is called Declaration. Giving it an actual value is called Initialization.

// 1. Declaration Syntax: Datatype variablename;
int a; 

// 2. Initialization Syntax:
a = 10;

// 3. Combined (Best Practice):
int a = 10;

⚠️ Note: Writing a = 10; before declaring int a; is INCORRECT and will cause a compiler error!

9:08 - 11:14

6. Different Data Types

Variables can hold different types of data. The datatype tells the computer how much memory space to reserve (e.g., an int usually takes 2 or 4 bytes depending on the compiler).

// Stores a decimal number
float a = 3.14;

// Stores a single character
char a = 'a'; 
/* Note: The variable name 'a' and character value 'a' are different. 
   The char 'a' is stored as its ASCII number: 97 */

// Declaring multiple variables at once
int a = 1, b = 2;

// ❌ INVALID: A variable cannot be of type void
void a; // ERROR

7. Rules for Constructing Variable Names

The compiler is very strict about naming. Here are the golden rules:

  • Allowed Characters: Only Letters (A-Z, a-z), Digits (0-9), and Underscores (_) are allowed. No spaces, commas, or special symbols ($, %, @).
  • Starting Character: MUST start with a letter or an underscore. It CANNOT start with a digit.
  • No Keywords: You cannot use the 32 reserved C keywords (e.g., float, if, break).
  • Case Sensitivity: C is highly case-sensitive. Num, num, and NUM are treated as completely different variables.
