C Programming: Strings

In-depth Lecture Notes: Using printf, puts, and Memory Layout

1. The Basic Setup: Read & Print

To use strings in C, we declare an array of characters. We can then ask the user for input and display it back on the screen.

char name[30];               // Declare array of size 30
printf("Enter name: ");      // Prompt the user
scanf("%s", name);           // Read the string
printf("%s", name);          // Print the string
💡 Instructor Note: Notice there is NO & symbol before name in the scanf function. Why? Because the name of an array acts as a base pointer to its very first memory address. It already provides the address!

3. Advanced Formatting Output (printf)

We don't always want to print the whole string. We can use specific format specifiers to control how the string looks on the screen.

Scenario A: Precision (Limiting Length)

Adding a decimal point and a number limits how many characters print.

printf("%.5s", name);
Jayan

Explanation: The compiler starts at index 0 and strictly cuts off the printing after 5 characters.

Scenario B: Field Width + Precision

We can reserve "space" on the screen (Field Width) and restrict characters (Precision).

printf("%10.5s", name);
Jayan

Explanation: The `10` creates a 10-character wide box. The `.5` grabs "Jayan". Because C right-aligns by default, it leaves 5 blank spaces at the beginning, followed by our 5 letters.

4. Introducing puts()

puts() is a predefined function inside the <stdio.h> header file used exclusively for printing strings. It is much simpler to type than printf.

puts(name);
🔥 The Golden Rule of puts():
puts() automatically adds a newline character (\n) at the end of your string. printf() does NOT do this.

Visualizing the Difference

5. Tricky Pointer Printing (Important for Exams)

The compiler determines exactly *what* to print based on the starting memory address you give it. Let's look at three trick questions.

Trick 1: Using the Address-of Operator

printf("%s", &name);

Output: Jayanti

Why? Because the address of the whole array (&name) resolves to the exact same memory location (2000) as the name of the array itself.

Trick 2: Shifting the Starting Address

printf("%s", &name[2]);

Output: yanti

Why? We are explicitly telling the compiler: "Start reading at the address of index 2 (which is 2002, where 'y' is)". The compiler starts at 'y' and keeps printing until it hits the \0 null character.

Trick 3: The Crash Scenario

printf("%s", name[2]);
⚠️ ERROR / CRASH
The %s format specifier expects a memory address (a pointer) to know where to start looking. name[2] does NOT provide an address; it provides the literal character 'y'. The compiler gets confused trying to read a character as a memory address, and the program will crash!
