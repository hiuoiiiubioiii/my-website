The char Data Type in C

Complete In-Depth Study Notes & Visualizations

1. Introduction to char

The char (character) data type is primarily used to store single characters—like letters, numbers, or symbols. Under the hood, computers only understand numbers, so every character is translated into a number.

Format Specifiers (How we print data)

%c Prints a Character
%d Prints a Signed Integer
%u Prints an Unsigned Integer

2. Memory Allocation & Ranges

When you declare a char, the computer allocates exactly 1 byte of memory. Because 1 byte equals 8 bits, there are 28 = 256 possible unique values.

The Two Modifiers

  • unsigned char: Only holds positive numbers. The range is exactly 0 to 255.
  • signed char (Default): Splits the 256 values in half to allow for negative numbers. 128 negative values (-1 to -128) and 128 positive values (0 to 127). The range is -128 to 127.

3. How Computers Understand Characters

Computers use an encoding system called ASCII (American Standard Code for Information Interchange) to map numbers to characters. When you type 'A', the computer stores it as binary representing 65.

Crucial ASCII Values to Memorize
Characters Decimal Range Example
Lowercase (a - z) 97 - 122 a = 97, b = 98
Uppercase (A - Z) 65 - 90 A = 65, B = 66
Numbers (0 - 9) 48 - 57 '0' = 48, '1' = 49

*Note: The character '0' is different from the integer 0.

4. The Circular Number Line (Overflows)

What happens if you try to put a number bigger than the maximum limit into a variable? It doesn't crash; it wraps around like a clock. The instructor drew these circles to visualize Overflow and Underflow.

Unsigned Circle

Range: 0 to 255

Signed Circle

Range: -128 to +127

The Golden Rule: If you add 1 to the max positive value, you jump across the gap at the bottom and land on the maximum negative value!

5. Dry Running Code Examples

Let's trace through the instructor's code examples exactly how she did on the whiteboard.


Example 1: Standard Print

char a = 'a';
printf("%c", a); 
printf("%d", a);
Behind the scenes:
Memory stores the binary for 97 (the ASCII for 'a').
%c prints the character: a
%d prints the signed integer: 97

Example 2: Positive Overflow (Signed)

char d = 130; 
printf("%c", d);
printf("%d", d);
Behind the scenes:
A normal char stops at 127. 130 is an Overflow.
Trace the Signed Circle: Start at 127. Step forward 1 is -128. Step 2 is -127. Step 3 (landing on 130) is -126.
%c prints: (Random garbage character)
%d prints: -126

Example 3: Negative Underflow (Unsigned)

unsigned char d = -130;
printf("%d", d);
Behind the scenes:
An unsigned char cannot hold negatives. Underflow!
Trace the Unsigned Circle: Start at 0 and move backward 130 spaces.
Math shortcut: 256 (total values) - 130 = 126.
Output: 126

6. Instructor's Homework

At the end of the video, she leaves this snippet for you to solve:

unsigned char d = -129;
printf("%d", d);
Output: 127

Logic: You are putting a negative number (-129) into an unsigned variable (range 0 to 255). You must travel backward (anti-clockwise) on the unsigned circle.
Start at 0. Step back 1 is 255. Step back 2 is 254...
Math Shortcut: 256 - 129 = 127.
