Program to Copy Content of One File into Another

1. The Conceptual Setup (Whiteboard Logic)

Before writing a single line of code, we must understand the logic. We have an existing file with text, and we want to transfer that text to a brand new file, character by character.

Key Concept: We cannot use a single file pointer for two files. We need exactly two file pointers. One will handle the "r" (Read) mode for the source, and the other will handle the "w" (Write) mode for the destination.

2. Essential C Functions Required

To achieve this, we will use the standard file handling functions provided by <stdio.h>.

  • fopen(): Used to open the files. We need "r" mode to read the source, and "w" mode to write to the destination.
  • fgetc(): File Get Character. This fetches a single character from our source file pointer.
  • fputc(): File Put Character. This places the fetched character into our destination file pointer.
  • fclose(): Used to close both files at the end of the program to free up memory.
  • exit(1): Located in <stdlib.h>. Used to immediately terminate the program if a file fails to open.

Instructor Note: If you want a file capable of both reading and writing simultaneously, you would use "w+" mode, but for a strict copy-paste operation, "r" and "w" are perfect.

3. Step-by-Step Logic Construction

Step 1: Pointers & Variables

Declare two file pointers and a character variable to hold the data temporarily.

FILE *fptr1, *fptr2;
char c;

Step 2: Open Source File & Error Check

Open abc.txt in Read mode. If the file doesn't exist, the pointer returns NULL.

fptr1 = fopen("abc.txt", "r");
if (fptr1 == NULL) {
    printf("Error opening source file.");
    exit(1);
}

Step 3: Open Destination File & Error Check

Open Destination.txt in Write mode. Magic trick: In "w" mode, if the file doesn't exist on your computer, C will automatically create it for you!

fptr2 = fopen("Destination.txt", "w");
if (fptr2 == NULL) {
    printf("Error creating destination file.");
    exit(1);
}

4. The Core Loop Mechanism

How do we know when to stop copying? We use a while loop that runs until it hits EOF (End Of File).

while ( (c = fgetc(fptr1)) != EOF )

What is happening inside this single line?

  • 1. fgetc(fptr1) grabs a character (e.g., 'H').
  • 2. c = ... assigns that 'H' to our variable c.
  • 3. != EOF checks: Is 'H' the End of File? No! So, enter the loop.

Inside the loop, we simply push that character into the second file:

fputc(c, fptr2);

5. The Complete Final Code

Here is the final, assembled code exactly as written on the whiteboard.

#include <stdio.h> #include <stdlib.h> // Required for exit(1) void main() { FILE *fptr1, *fptr2; char c; // 1. Open source file in Read mode fptr1 = fopen("abc.txt", "r"); if (fptr1 == NULL) { printf("Cannot open file\n"); exit(1); } // 2. Open destination file in Write mode fptr2 = fopen("Destination.txt", "w"); if (fptr2 == NULL) { printf("Cannot open file\n"); exit(1); } // 3. Read from fptr1 and Write to fptr2 until End Of File while((c = fgetc(fptr1)) != EOF) { fputc(c, fptr2); } // 4. Success message and cleanup printf("Successfully copied!"); fclose(fptr1); fclose(fptr2); }

6. Practical Execution (Laptop Demo)

During the final segment of the lecture, the instructor proves the logic works in VS Code:

  • Before Execution: The directory only contains abc.txt. There is no destination file.
  • Execution: The program is compiled and run. The terminal outputs "Successfully copied!".
  • After Execution: A brand new file named Destination.txt automatically appears in the VS Code file explorer. Clicking it reveals the exact text from abc.txt.
Instructor's Challenge: She mentions this is just one method (character-by-character). You can also achieve this by reading complete strings using different functions (like fgets and fputs). That is your assignment to try!
