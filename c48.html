Memory Representation & Accessing Arrays

Instructor: Jayanti Khatri Lamba | Topic: 1D Arrays in C

1. Declaration & Initialization

The instructor begins by declaring and initializing a 1-Dimensional (1D) array simultaneously. This is known as compile-time initialization.

int a[5] = {1, 10, 0, -1, 3};
  • int - The data type of the elements.
  • a - The name of the array.
  • [5] - The size of the array (it holds 5 elements).
  • {...} - The specific values assigned to the array instantly.

2. The Memory Tape & Total Size

Computer memory acts like a long tape made up of individual blocks. Each block represents 1 byte (8 bits).

To store our data, the computer needs to calculate how much memory to allocate. Here is how it calculates the required space:

Total Memory Calculation:

1 Integer = 4 bytes (assumed for this machine architecture)

Array Size = 5 elements

4 bytes × 5 = 20 bytes

The compiler will reserve exactly 20 bytes of space for this array.

3. The Golden Rule: Contiguous Allocation

The most important property of an array is that memory is allocated in a contiguous block. The 20 bytes are not scattered; they are placed side-by-side, one right after the other.

Visualizing the Memory Map

Let's assume the computer places the first element at hypothetical memory address 2000. Because an integer takes 4 bytes, the next element will naturally start at 2004.

4. The "Base Address" & Internal Pointers

The address of the very first element (2000) is known as the Base Address. This is the anchor point for the entire array.

Array Name Variable:

a [ 2000 ]

Crucial Concept: The name of the array (a) actually acts as an Internal Pointer. It is a variable that stores the base address of the array. When the computer sees a, it sees 2000.

5. Accessing Elements (Random Access)

Arrays allow for Random Access. If you want the 4th element (a[3]), the computer doesn't need to read the 1st, 2nd, and 3rd elements first. It jumps directly to it using a mathematical formula.

Address Calculation Formula:


Address = Base Address + ( Index × Size of Data Type )

The computer knows the Base Address (from a) and the data type size (4 bytes). When you provide an index, it calculates the exact memory location instantly.

6. Proving the Formula

Let's run the formula manually based on our memory map to prove it works.

Example A: Finding the 4th Element a[3]

  • Base Address = 2000
  • Target Index = 3
  • Data Size = 4
  • Math: 2000 + (3 × 4) ➔ 2000 + 12 = 2012

Result: Address 2012 correctly holds the value -1.

Example B: Finding the 1st Element a[0]

  • Math: 2000 + (0 × 4) ➔ 2000 + 0 = 2000

Result: Index 0 mathematically returns the base address itself!

Example C: Finding the 2nd Element a[1]

  • Math: 2000 + (1 × 4) ➔ 2000 + 4 = 2004

Result: Accurately points to the start of the second element.

7. Time Complexity & Summary O(1)

Because accessing an element relies entirely on a simple math formula rather than searching sequentially through memory blocks, accessing any element takes exactly the same amount of time.

In computer science, this constant speed is known as Big O of 1, or O(1) time complexity.
