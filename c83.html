Wild Pointers in C

In-depth lecture notes based on Jenny's Lectures. Scroll down to begin learning.

Prerequisites: Basic understanding of pointers. (Previously covered: Void, Null, and Dangling pointers).

1. What is a Wild Pointer?

A Wild Pointer is simply an uninitialized pointer. When you declare a pointer but do not assign it a valid memory address, it becomes "wild."

Because it is uninitialized, it contains a "garbage value." This means it is pointing to a completely random, unreserved location in your computer's memory.

// Declaration only. No address assigned.
int *ptr; // This is a Wild Pointer!

The Crash Scenario: If you try to dereference this pointer (e.g., printf("%d", *ptr);), your program will try to read or write to address 5146. Since your program hasn't reserved this space, the OS will intervene, resulting in a crash or undefined behavior.

3. Three Ways to "Tame" a Wild Pointer

To fix a wild pointer, you must assign it a valid state. There are three ways to do this.

Solution 1: Initialize to NULL

If you don't know what address the pointer should hold yet, assign it to NULL. It is no longer wild; it is a controlled Null Pointer.

int *ptr = NULL; // Safe. Not pointing to random memory.
ptr
NULL (0)

Solution 2: Assign a Variable's Address

Give the pointer the valid address of an existing, reserved variable.

int x = 5;
int *ptr = &x; // Points to valid memory
ptr
1000
Address: 2046
→
x
5
Address: 1000

Solution 3: Dynamic Memory Allocation

Allocate memory at runtime using malloc() and assign that legally reserved base address to the pointer.

// Allocates 4 bytes dynamically
int *ptr = (int *)malloc(sizeof(int));
*ptr = 67; // Valid operation!

4. VS Code Practical Demonstration

Here is exactly what happens when we run these scenarios in a C compiler.

Demo 1: The Crash (Wild Pointer)

#include <stdio.h>
void main() {
    int *ptr; // Uninitialized
    printf("%d\n", *ptr);

Demo 2: Tamed via Variable Address

#include <stdio.h>
void main() {
    int a = 9;
    int *ptr = &a; // Initialized properly
    printf("%d\n", *ptr); 
}
> gcc validpointer.c
> ./validpointer.exe
9

Demo 3: Tamed via Dynamic Memory

#include <stdio.h>
#include <stdlib.h>
void main() {
    int *ptr = (int *)malloc(sizeof(int));
    *ptr = 67;
    printf("%d\n", *ptr); 
}
> gcc dynamicpointer.c
> ./dynamicpointer.exe
67

5. Quick Review Flashcards

Q: What is the definition of a Wild Pointer?

A: A pointer in C that has been declared but not initialized. It holds a garbage value and points to an arbitrary, unreserved memory location.

Q: Why is a Wild Pointer dangerous?

A: Dereferencing it causes the program to read/write memory it does not own, resulting in unpredictable behavior or immediate program crashes (Segmentation Faults).

Q: What are the three standard ways to fix a Wild Pointer?

A: 1) Initialize it to NULL. 2) Assign it the address of a valid variable (&var). 3) Allocate memory dynamically (malloc).
