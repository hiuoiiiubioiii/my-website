C Programming: Introduction to Functions

Complete lecture breakdown, whiteboard visualizations, and code notes.

1. The Core Question: Why do we need functions?

Whenever you learn a new concept in programming, you must ask "Why?" If we can write all our code directly inside the main() function, why invent other functions?

The Problem: Repetitive Code

Imagine a simple block of code designed to add two numbers together:

int a, b, sum = 0;
printf("Enter two numbers: ");
scanf("%d %d", &a, &b);
sum = a + b;
printf("sum = %d", sum);

The Scenario: If you put this inside main(), it runs perfectly once. But what if your program requires you to calculate the sum of two numbers 5 separate times at different points in your program?

The Bad Solution: You would have to copy and paste those 5 lines of code five times. Your code becomes massive, highly repetitive, and incredibly difficult to read and debug.

2. The Solution: Definition & Calling

Instead of writing the code over and over, we wrap that block of logic inside a named container. This is a Function.

Whiteboard Visualization: Control Flow

// 1. Function Calling
void main() {
    sum(); ← Compiler jumps
    printf("Hello");
}
Control transfers
// 2. Function Definition
void sum() {
    int a, b, s;
    // ... logic ...
} ← Control returns

Function Definition: Tells the compiler what the function does and how it does it. It contains the actual logic (the curly braces with the math).

Function Calling: The compiler has no mind of its own. Inside main(), you must explicitly write sum(); to tell the compiler to pause, go execute the sum block, and then come back.

This is the most critical concept for understanding why functions make programs highly efficient.

Why is this important? If you copy-pasted the math logic 5 times directly inside main(), the variables (a, b, sum) would exist in RAM for the entire duration of the program. By using functions, variables are local. They take up memory only when sum() is running, and give that memory back immediately when finished.

4. Four Major Advantages of Functions

  • Reusability: Write the logic once (Definition) and call it as many times as you want (Calling). (e.g., 5 times sum, 3 times prime, 10 times even_odd).
  • Easy Debugging & Testing: If there is a math error, you only have to fix it in one place (the definition), rather than hunting down 5 copy-pasted blocks.
  • Better Memory Utilization: As shown in the RAM diagram, functions free up local memory the exact second they hit the closing } brace.
  • Modularity: You can break a massive, complicated software project down into smaller, manageable "modules" (sub-programs).

5. Types of Functions in C

Predefined Functions

Library functions already written by the creators of C. Their definitions live in header files (like <stdio.h>).

Examples: printf(), scanf(), clrscr(). You only call them.

User-Defined Functions

Functions where you (the programmer) write the custom logic and definition.

Examples: sum(), is_prime(), sub().

⚠️ The Golden Rule of C: A C program must have at least one function to run, and that function must be named main().

6. Practical Demonstration (VS Code)

Here is the exact code trace demonstrated in the editor. Note the crucial syntax rule at the bottom.

#include <stdio.h>

// 1. User-Defined Function
void sum() {
    int a, b, sum = 0;
    printf("Enter two numbers: ");
    scanf("%d %d", &a, &b);
    sum = a + b;
    printf("sum=%d\n", sum);
}

void main() {
    sum(); // Ask user for numbers & print
    printf("hello\n");
    sum(); // Ask user for numbers & print again
    sum(); // Ask user for numbers & print a 3rd time
}
🚫 Strict Syntax Rule Demonstrated: You can define functions above or below main(), but you cannot define a function INSIDE another function. Placing the void sub() { ... } logic inside the curly braces of main() will cause a compiler error.
