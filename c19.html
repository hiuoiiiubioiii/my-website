Example: To get 10, place a 1 under 8 and 2 (because 8 + 2 = 10). The rest are 0s. Binary: 00001010

1. Left Shift Operator <<

Shifts the bits of the first operand to the left by the number of positions specified by the second operand.

Syntax: variable << number_of_shifts

int a = 10;
int c = a << 2; // Shift bits of 'a' left by 2 places

🚀 The Mathematical Shortcut

You don't need to draw boxes in an exam! Left shifting is basically multiplying by powers of 2.

Result = Number × 2(shift_amount)
  • 10 << 2 → 10 × 22 → 10 × 4 = 40
  • 10 << 4 → 10 × 24 → 10 × 16 = 160

Shifts the bits of the first operand to the right by the number of positions specified by the second operand.

🚀 The Mathematical Shortcut

Right shifting is basically integer division by powers of 2. Drop any decimals!

Result = Number ÷ 2(shift_amount)
  • 10 >> 2 → 10 ÷ 22 → 10 ÷ 4 = 2.5 → Drop decimal → 2
  • 10 >> 4 → 10 ÷ 24 → 10 ÷ 16 = 0.625 → Drop decimal → 0

3. Bitwise NOT ~ (One's Complement)

A unary operator (takes only one number) that inverts every single bit. 0 becomes 1, and 1 becomes 0.

int a = 5;
int b = ~a; // Inverts all bits of 'a'

Visualizing: ~5 (Using 4-bits for simplicity)

Original (5):

Wait, isn't 1010 equal to 10?
In computers, if the left-most bit is a 1, the system treats it as a negative number stored in 2's complement form. Manually calculating this binary to a negative decimal is tedious.

🚀 The Mathematical Shortcut

To find the exact integer output of a Bitwise NOT instantly, use this algebra trick:

Result = -(Number + 1)
  • ~5 → -(5 + 1) = -6
  • ~10 → -(10 + 1) = -11
