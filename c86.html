Functions in C

Complete Visual Study Guide: Definition, Call, and Flow Control

🗺️

1. The Big Picture & Execution Flow

When you write a C program, the compiler always looks for the main() function first. When a function asks another function to run, control jumps to that function, executes it, and then returns.

// 1. Function Declaration (Prototype)
void sum();

void main() {       
    // 2. Calling Function
    sum();          // 3. Function Call
}

// 4. Function Definition
void sum() {        
    int a, b, total = 0;
    printf("Enter a & b:");
    scanf("%d %d", &a, &b);
    total = a + b;
    printf("Sum = %d", total);
}

Visualizing Program Control

Here is exactly how the compiler reads your code step-by-step:

Step 1: main() begins execution
Hits sum() call
Step 2: sum() is called
Compiler jumps to Definition
Step 3: void sum() { ... } executes
Asks for input, calculates, prints
Hits closing '}' and returns
Step 4: main() resumes and ends
🏗️

2. Anatomy of a Function Definition

Every function definition is split into two mandatory parts: the Header and the Body.

return_type function_name (arguments with data_type)
{
  // Local variables (optional)
  // Statements 1, 2, 3... (mandatory)
  // Return statement (optional if void)
}
  • Return Type: What data comes out? (e.g., int, float, void). Rule: If left blank, C assumes it returns an int.
  • Arguments: What data goes in? You MUST write both the data types and the variable names in the definition header.
  • Local Variables: Variables declared inside the {} only exist inside this function.
⚖️

3. Declaration vs Call vs Definition

The most common errors students make are mixing up semicolons, data types, and variable names across these three stages. Here are the golden rules:

1. Declaration

"Telling the compiler the function exists."

void sum(int, int);
  • Data types MUST be written.
  • Variable names are OPTIONAL.
  • MUST end with a ;

2. Call

"Executing the function in main()."

sum(5, 7);
sum(x, y);
  • Pass actual values/variables.
  • NEVER write data types here.
  • MUST end with a ;

3. Definition

"Writing the actual logic."

void sum(int a, int b) { ... }
  • MUST write Data types.
  • MUST write Variable names.
  • NO SEMICOLON after the header.
📖

4. Vocabulary: Parameters vs Arguments

These terms are often used interchangeably, but in technical interviews and exams, they have distinct meanings.

Actual Arguments: The actual data passed during the Function Call.
sum(5, 7);
Formal Parameters: The placeholders receiving the data in the Declaration and Definition.
void sum(int a, int b) { ... }

5. Instructor Challenge / Homework

Identify what each snippet is (Declaration, Call, or Definition) and spot any syntax errors based on the rules we just learned.

int fact(int a);
Valid: Function Declaration. Ends with a semicolon. The variable name 'a' is optional but completely legal. 
void multiply(int, int);
{
    Statements block
}
Error: Invalid Definition. Two mistakes: 1) You cannot put a semicolon after a definition header. 2) Variable names are missing (you must provide names in a definition, e.g., int a, int b). 
Sum(x, y);
Valid: Function Call. Passing actual arguments (variables). No data types are written, ends with a semicolon. 
divide(m+5, n+5);
Valid: Function Call. You can pass expressions as actual arguments. The math will calculate first, then the result is passed. 
void isPrime(int),
Error: Invalid Declaration. It ends with a comma , instead of a semicolon ;. 
void sum(int a, float b)
{
    printf("Sum = %f", a+b);
    return;
}
Valid: Function Definition. No semicolon after the header, both data types and variable names are present. Using return; is optional but valid in a void function.
