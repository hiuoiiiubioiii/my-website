🎙️ 1. What is the `typedef` Keyword?

typedef is a built-in keyword in C programming. Its entire purpose is to create an alias (a custom name, nickname, or synonym) for an existing data type.

🧠 The Instructor's Analogy

Official Name:
"Jayanti"
🤝
Nickname:
"Jenny"

If you call her "Jayanti" or "Jenny", she responds to both because they represent the same person. In C, `typedef` lets you give a nickname to a data type, and the compiler will treat both names as the exact same thing.

✍️ Syntax & Whiteboard Example

Formula: typedef [old_data_type] [new_alias_name];

typedef int integer; 

void main() {
    // The standard way:
    int a = 10; 

    // The new way using typedef:
    integer b = 15; 
}

🏢 2. The Problem with Vanilla Structures

Normally in C, every time you want to declare a variable for a structure, you are forced to type out the long phrase struct StructureName.

The "Old" Way (Without typedef)

struct Student {
    int rollno;
    char name[20];
    float marks;
};

void main() {
    // ❌ Annoying: You MUST write "struct" every single time!
    struct Student s = {10, "Jenny", 89}; 
    
    printf("%d %s %f", s.rollno, s.name, s.marks);
}

If you use a structure 20 times in a large program, typing struct Student over and over becomes tedious and clutters your code.

💡 3. The Solution: Typedefined Structures

By combining typedef with a structure, you can create a short, single-word alias for that entire structure block.

The "New" Way (With typedef)

Watch how the instructor modified the code:

  1. Added the word typedef to the very beginning.
  2. Added a new name (alias) right before the final semicolon.
// 1. Add typedef here
typedef struct Student {
    int rollno;
    char name[20];
    float marks;
} student;  // 2. Add alias name here

void main() {
    // ✅ Much cleaner! We completely dropped the word "struct"
    student s = {10, "Jenny", 89}; 
    
    printf("%d %s %f", s.rollno, s.name, s.marks);
}
🚨 Crucial Warning!
The alias name you put at the very end of the `typedef struct` block (like student) is NOT a variable. It is the new name of your custom data type. It consumes zero memory until you actually declare a variable with it inside a function.

🌍 4. Rules of Scope

Aliases have boundaries. Where you place the typedef statement determines where you are allowed to use it.

✅ Global Scope

Placed outside of all functions (usually at the top under your #include statements).

typedef int integer;

void main() {
    integer a = 10; // Works!
}

void sum() {
    integer b = 20; // Works!
}

❌ Local Scope

Placed inside a specific function. Only that specific function knows the nickname.

void sum() {
    typedef int integer;
    integer a = 10; // Works here!
}

void main() {
    // ERROR! main() doesn't 
    // know what "integer" is.
    integer b = 20; 
}

📝 Video Summary Checklist

  • ✅ What is it? typedef provides an alias (nickname) for existing data types.
  • ✅ Why use it with Structs? It saves you from repeatedly typing the struct keyword.
  • ✅ Syntax to remember: typedef struct OldName { ... } NewName;
  • ✅ Memory: typedef creates a type, not a variable. It takes up zero memory.
  • ✅ Scope: Obeys standard C scope rules depending on where you write it (Global vs Local).
