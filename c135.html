1. What is realloc()?

realloc stands for reallocation of memory. It is a built-in function defined in the <stdlib.h> library.

Purpose: It is used to resize a memory block that was previously allocated dynamically (using malloc() or calloc()).

  • Increase Size: Expand a block to store more data.
  • Decrease Size: Shrink a block to free up unused space.
⚠️ Crucial Rule

You can only use realloc() on memory that was allocated dynamically. If you try to use it on memory allocated statically (like a standard array int arr[10];), it will result in undefined behavior.

2. Syntax & Arguments

How do we write the function? Let's break down the signature.

void* realloc (void *pointer, size_t size);
  • Argument 1 (void *pointer): The pointer pointing to the base address of the previously allocated memory block.
  • Argument 2 (size_t size): The new total size you want for the memory block in bytes. (Note: It is NOT the amount of memory you want to add, it is the new total).
  • Return Type (void*): It returns a generic void pointer to the new memory block. You must typecast it to match your data type.
Syntax Mapping Example

Resize an existing integer pointer to hold 400 bytes (100 integers):

ptr = (int*) realloc( ptr , 400 );

3. Behind the Scenes: Expanding Memory

The absolute best feature of realloc() is that it resizes memory without losing your previous content/data. When you ask for a larger size, the system attempts to do one of two things automatically.

Case 1: Contiguous Space is Available (In-Place Expansion)

If there is free space immediately following your current memory block in the heap, realloc() will simply extend the existing block.

  • The base address remains exactly the same.
  • Your old data stays exactly where it is.

Case 2: Contiguous Space is NOT Available (New Block Allocation)

If the space immediately after your block is occupied by other program data, realloc() cannot just stretch the block. It performs a 4-step process behind the scenes:

📝 Interview Focus: The 4 Steps of Reallocation
  1. Searches the heap for a brand new block large enough for the new requested size.
  2. Copies all your existing data into the new block.
  3. Automatically frees (deallocates) the old memory block.
  4. Returns the base address of the new block.

Note: If the heap is totally full and realloc cannot find enough space anywhere, it will return NULL. Your original data and memory block will remain untouched and safe.

4. Behind the Scenes: Shrinking Memory

If you use realloc() to decrease the size (e.g., from 5 ints down to 3 ints), the block is shrunk. The base address remains the same, but data in the cut-off portion will be lost forever.

5. Crucial Edge Cases (Cheat Codes)

There are two special scenarios where realloc() perfectly mimics other memory functions. Write these down!

realloc(NULL, size) = malloc(size)

If the pointer passed is NULL, it just creates a brand new memory block.

realloc(ptr, 0) = free(ptr)

If the requested size is 0, it deallocates the memory and returns NULL.

6. Practical Code Demonstration

Here is the full C program demonstrated by the instructor on her laptop. It proves whether the memory expands in-place or jumps to a new address.

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr, *ptr1, n, i;
    
    printf("Enter initial number of values (n): ");
    scanf("%d", &n);

    // Initial allocation using calloc
    ptr = (int*) calloc(n, sizeof(int)); 

    printf("Enter %d values:\n", n);
    for(i=0; i<n; i++) {
        scanf("%d", (ptr+i));
    }

    printf("Enter updated size of n: ");
    scanf("%d", &n);

    // Reallocation - storing in a new pointer ptr1
    ptr1 = (int*) realloc(ptr, n * sizeof(int));

    // Checking if the address changed
    printf("Previous addr: %d, new address: %d\n", ptr, ptr1);

    printf("Values are:\n");
    for(i=0; i<n; i++) {
        printf("%d\t", *(ptr1+i));
    }

    // Always free the final dynamically allocated pointer
    free(ptr1); 
    
    return 0;
}

✅ Summary Cheat Sheet

  • Include: <stdlib.h>
  • Primary Use: Resizing existing dynamically allocated memory.
  • Data Safety: Old data is preserved when expanding. Data is truncated (lost) when shrinking.
  • Return Value: Returns a void* pointer (requires typecasting) to the newly allocated space, or NULL if allocation fails.
