Function Pointers in C

Mastering Arrays of Function Pointers to build Menu-Driven Programs

🚨 The Problem: Bulky Switch Cases

In the previous lecture, we learned about Callbacks. Now, we are exploring the second major application of function pointers: Replacing Switch Statements.

When building menu-driven programs (like a calculator), students traditionally rely on long, repetitive switch-case blocks. Every time a new feature is added, the code grows longer and harder to read.

// The Old Way: Bulky and hard to scale
switch(ch) {
    case 0: add(a, b); break;
    case 1: sub(a, b); break;
    case 2: mult(a, b); break;
    case 3: div(a, b); break;
}
💡
The Solution: We can replace that entire switch block with one single line of code using an Array of Function Pointers!

🛠️ Step 1: Defining the Functions

To use an array of function pointers, all functions must have the exact same signature (same return type and same parameters). Let's define our four calculator operations.

void add(int a, int b) { 
    printf("%d", a + b); 
}

void sub(int a, int b) { 
    printf("%d", a - b); 
}

void mult(int a, int b) { 
    printf("%d", a * b); 
}

void div(int a, int b) { 
    printf("%d", a / b); 
}

We also need our main() function setup to prompt the user for their choice and inputs.

void main() {
    int ch, a, b;
    
    printf("Enter choice (0:Add, 1:Sub, 2:Mult, 3:Div): ");
    scanf("%d", &ch);
    
    printf("Enter values of a and b: ");
    scanf("%d %d", &a, &b);

🧠 Step 2: The Array of Function Pointers

Just like you can have an array of integers, you can declare an array that holds memory addresses of functions. Here is the golden line of code that replaces the switch case:

    // Declaring and initializing the array of function pointers
    void (*fptr[10])(int, int) = {add, sub, mult, div};

Syntax Breakdown:

  • void: The return type of the functions this array will point to.
  • (*fptr[10]): Declares `fptr` as an array of size 10 that holds pointers.
  • (int, int): The parameter types the functions accept.
  • = {add, sub...}: Initialization. Remember, just writing the name of a function without parentheses acts as its memory address!

⚡ Step 3: Executing the Function Call

Now, instead of writing multiple case statements, we write one single line to call the correct function dynamically based on the user's input ch.

    // Call the function dynamically based on user choice
    (*fptr[ch])(a, b);
} // End of main()

How the computer reads this execution trace:

  1. User Input: User enters 0 for ch, and 2 and 3 for a and b.
  2. Array Lookup: The program evaluates fptr[ch], which becomes fptr[0].
  3. Dereference: It looks inside fptr[0], finds memory address 100 (the add function), and dereferences it using the *.
  4. Execution: It passes (2, 3) to that memory address. The add function runs and prints 5.

💻 Live IDE Demo & Debugging

During the VS Code demonstration, the instructor compiles the code and encounters a specific warning message. This is a very common student mistake!

⚠️
Warning: "initialization from incompatible pointer type"
Why? The instructor accidentally typed int (*fptr[10])(int, int) instead of void. Because the actual functions (`add`, `sub`) return void, the pointer array must also be declared with void.

Testing the fixed code:

  • User enters choice 1 (Subtraction), and inputs 5 and 1. The program correctly jumps to fptr[1] and outputs 4.
  • User enters choice 3 (Division), and inputs 4 and 2. The program jumps to fptr[3] and outputs 2.

🎯 Key Takeaways

  • Function Signatures Must Match: To group functions inside an array of function pointers, they must all have the exact same return type and the exact same parameter list.
  • Function Names are Addresses: You do not need the & symbol when assigning a function to a pointer. fptr[0] = add; works perfectly because the compiler treats the function name itself as a base memory address.
  • Incredible Scalability: If you wanted to add a "Modulo" function later, using a switch case means writing another whole case 4: ... break; block. With function pointers, you just add the word mod to the array initialization list: {add, sub, mult, div, mod}. The execution code (*fptr[ch])(a,b) never has to change!
