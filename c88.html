Topic: Classification of Functions

Based on Instructor's Whiteboard & VS Code Lecture

1️⃣ The 4 Categories of Functions

Any user-defined function you write in C (or other languages like Java) will fall into one of four categories based on how it handles arguments (inputs) and return types (outputs).

  • 👉 Category 1: No Argument without Return Type (Focus of this lecture)
  • Category 2: No Argument with Return Type
  • Category 3: With Argument No Return Type
  • Category 4: With Argument with Return Type

2️⃣ Deep Dive: No Argument & No Return Type

This type of function takes zero input parameters from the calling function, and it gives zero values back. All logic, calculations, and printing happen completely inside the function itself.

1. The Declaration (Prototype)

Placed at the top of your file, before main().

void fun(void);

Audio Context: void means "nothing". The first void means it returns nothing. The void in the parentheses means it accepts no arguments. (You can leave it empty like (), but (void) is best practice).

2. The Definition (The Code)

void fun() {
    // Write your statements here
    // Do NOT write a 'return value;' statement here
}

3. The Call

Inside main(), simply call the function by its name with empty parentheses.

fun();

3️⃣ The Whiteboard Example: The `sum` Function

Here is the exact code the instructor wrote on the whiteboard to demonstrate adding two numbers.

#include <stdio.h>

void sum(void);     // Declaration

void main() {
    sum();          // Function Call
}

void sum() {        // Function Definition
    int a=5, b=7, sum=0;
    sum = a + b;
    printf("sum = %d", sum);
}

4️⃣ Practical VS Code Experiments

The instructor switched to VS Code to show what happens when you make deliberate mistakes. These are highly common interview/exam questions.

✅ Experiment 1: The Perfect Run

Running the exact code from the whiteboard.

Result: Compiles perfectly. Output is sum = 12.

❌ Experiment 2: Passing Arguments Illegally

Changing the call in main to: sum(5, 4);

Result: Compiler Error: too many arguments to function 'sum'.

Lesson: Because you declared the function as (void), the compiler actively blocks you from pushing inputs into it.

⚠️ Experiment 3: Deleting the Declaration

Deleting void sum(void); from the top of the file entirely.

Result: Compiler Warning: conflicting types for 'sum'.

💡 Why does this happen? (Crucial Concept)
In older C standards, if the compiler sees a function call before it has seen a declaration, it makes a blind guess: It assumes the function returns an int.

When it finally reaches the actual definition at the bottom and sees it returns void, it panics because its implicit guess (int) conflicts with reality (void). Always write your declarations!
