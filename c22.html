Common Beginner Mistakes in C Programming

Instructor: Jayanti Khatri Lamba

Visual Study Notes & Troubleshooting Guide

Introduction: When beginners start writing programs in C, they often encounter errors that prevent their code from running. Most of the time, these aren't complex logic failures—they are simple "silly mistakes" like typos or missing punctuation. Scroll down to see the most common mistakes, what the compiler error looks like, and exactly how to fix them.

1

Spelling the Standard Header File Incorrectly

Beginners see "stdio" and their brain automatically autocorrects it to "studio". However, stdio actually stands for Standard Input Output.

Incorrect #include <studio.h>
Correct #include <stdio.h>
Unable to open file 'studio.h'
2

Placing a Semicolon After the Main Function

You should never put a semicolon immediately after declaring your main function. A semicolon tells the compiler a statement has ended, but main() is just beginning its block of code.

Incorrect void main(); {
   // code
}
Correct void main() {
   // code
}
Declaration terminated incorrectly

📌 The Perfect Reference Program

For the rest of the mistakes, we will reference this perfectly written, simple program designed to add two numbers.

#include <stdio.h> #include <conio.h> void main() { int a, b, sum; clrscr(); printf("Enter two numbers:"); scanf("%d %d", &a, &b); sum = a + b; printf("Sum is = %d", sum); getch(); }
3

Missing Semicolons at the End of Statements

Inside the body of your function, every single executable statement must end with a semicolon ;. This tells the compiler the line of instruction is finished. You will develop muscle memory for this over time.

Incorrect sum = a + b
getch()
Correct sum = a + b;
getch();
Statement missing ;
4

Function Spelling Errors

Misspelling built-in functions is incredibly common. For example, writing clsrcr() instead of the correct clrscr() (Clear Screen).

Undefined symbol/function 'clsrcr'
5

Semicolon Inside scanf Format String

Putting a semicolon inside the quotation marks confuses the compiler. This creates a Logical Error. The program will run, but it will only accept the first input. The second input will turn into a "garbage value".

Incorrect scanf("%d; %d", &a, &b);
Correct scanf("%d %d", &a, &b);
6

Missing Comma in scanf or printf

You must separate the quotation marks (the format string) from the variables using a comma. If you forget it, the compiler doesn't know where the string ends and the variables begin.

Incorrect printf("Sum is = %d" sum);
Correct printf("Sum is = %d", sum);
Function call missing ,
7

Placing Quotation Marks Incorrectly

If you close the quotation mark at the very end of the parentheses, wrapping the variables inside the string, the program treats your variable names as plain text. This is a Logical Error.

Incorrect scanf("%d %d, &a, &b");
Correct scanf("%d %d", &a, &b);
8

Forgetting the Format Specifier %d in printf

If you forget %d, you will not get a compile-time error. However, at runtime, if the user enters 2 and 3, the screen will just output "Sum is =". It will not print 5, because the compiler wasn't given the command to fetch an integer.

Incorrect Output Sum is =
Correct Output Sum is = 5
9

Using the "Address Of" Operator (&) in printf

In scanf, you MUST use &a to tell the computer: "Store user input AT this memory address."

In printf, you MUST NOT use &. You just want the value inside the box, not the address of the box itself. If you use & in printf, it will print a random memory address number instead of your sum.

Incorrect printf("Sum is = %d", &sum);
// Output: Sum is = 1072 (Memory Address)
Correct printf("Sum is = %d", sum);
// Output: Sum is = 5 (Actual Value)
10

Missing Closing Braces }

In simple programs, it's easy to remember. But as you move into Advanced C (using if/else statements, for/while loops), you will have many opening braces {. Beginners frequently forget to match every single opening brace with a corresponding closing brace } at the end of their program.

Compound statement missing }

🎓 Final Summary: The Two Error Types

1. Syntax Errors (Compile-time): These stop your program from running entirely. Examples include spelling "stdio" wrong, missing commas, or missing semicolons. The compiler will usually point you near the line where the error occurred.

2. Logical Errors (Run-time): These are dangerous because the program will run, but it will behave incorrectly. Examples include forgetting %d in printf, or putting & in printf. The compiler cannot help you catch these; you must read your code carefully.
