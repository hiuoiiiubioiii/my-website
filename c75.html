Pointer Arithmetic in C

Scroll down to learn how pointer addition works under the hood.

Rule #1

Adding Two Pointers is ILLEGAL

The very first rule of pointer arithmetic: You cannot add two pointer variables together.

If pointer p holds memory address 1000, and pointer q holds 1000, adding them together yields 2000. This makes absolutely no logical sense in computer memory.

int a = 10;
int *p = &a;
int *q = &a;

p + q;  // ❌ ERROR: invalid operands to binary +
Rule #2

Adding an Integer to a Pointer is VALID

While adding two pointers is forbidden, you can add an integer to a pointer. This simply tells the compiler to jump to the next block of memory for that data type.

p + 1; // ✅ Valid
p + 5; // ✅ Valid
p + 0; // ✅ Valid
The Math

How the Math Actually Works

A common student mistake is treating pointer math like standard math.

int a = 10;  // Let's say 'a' is stored at address 1000
int *p = &a; // p holds 1000
p = p + 2;   // Where does p point now?

❌ The Misconception: Students think 1000 + 2 = 1002.
✅ The Reality: An integer takes up 4 bytes (addresses 1000, 1001, 1002, 1003). If p points to 1002, it points to the middle of the variable, giving garbage data!

The Golden Formula: New Address = p + (n * sizeof(data_type))

Applying the formula to p = p + 2:

  • Base address (p) = 1000
  • Value to add (n) = 2
  • Size of int = 4 bytes
  • Math: 1000 + (2 × 4) = 1008

The True Power: Pointers & Arrays

Pointer arithmetic is designed for Arrays because array elements are stored contiguously (side-by-side) in memory.

int a[5] = {0, 1, -1, 10, 11};
int *p = &a[0]; // Pointer starts at index 0

If we execute p = p + 2;, the pointer calculates 1000 + (2 * 4) = 1008. It safely lands exactly on the starting byte of a[2].

The Array Name is a Constant Pointer

In C, the array name a acts as a pointer to the first element (1000).

  • *(a + 2) -> Valid! Safely peeks at value -1.
  • a = a + 2; -> Invalid! You cannot permanently move a because it is a Constant Pointer.
Code Demo

VS Code Practical Proof

Let's print the actual memory addresses to prove the formula works.

int a[5] = {1, 4, 3, -8, 0};
int *p = &a[0];

printf("Address is: %u\n", p);  
// Output: Address is: 6422200

p = p + 2;

printf("Address is: %u\n", p);  
// Output: Address is: 6422208 (Exactly 8 bytes higher!)
Homework

Pop Quiz!

Look at the following code. What will be the final output?

int a[5] = {1, 4, 3, -8, 0};
int *p = &a[0];

p = p + 1;
*p = 2;

printf("value is: %d", *p);
Tap to Reveal Answer

The output is: 2

  1. p = p + 1; moves the pointer to index 1 (which currently holds the value 4).
  2. *p = 2; uses the dereference operator to overwrite the 4 with a 2.
  3. The print statement outputs the newly assigned value.
