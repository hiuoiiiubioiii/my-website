C Programming: Call by Value vs Reference

In-depth breakdown of the GATE 2016 Question

1. The Problem Setup

The instructor introduces a previous year GATE exam question. It tests your understanding of what happens when you mix "Call by Value" and "Call by Reference" in a single function call.

void f(int *p, int m) {
    m = m + 5;
    *p = *p + m;
    return;
}

void main() {
    int i = 5, j = 10;
    f(&i, j);
    printf("%d", i + j);
}

Rule of thumb: Execution of a C program always begins at the main() function. Let's trace it step-by-step.

2. Initializing Variables in main()

The execution starts in `main()`. Memory is allocated for the `main` function. Two integer variables, i and j, are declared and initialized. The instructor assigns hypothetical memory addresses (1000 and 2000) to visualize how pointers will work.

3. The Function Call: f(&i, j)

We are calling function f. Look closely at the arguments being passed:

  • &i (Call by Reference): We pass the address of `i` (1000) into pointer *p.
  • j (Call by Value): We pass the normal value of `j` (10) into variable m.

A new memory block is created for function f.

4. Executing inside f()

Code Line 1: m = m + 5;
We update the local variable `m`. The current value is 10. 10 + 5 = 15. The 10 is crossed out and becomes 15.

Code Line 2: *p = *p + m;
Always evaluate the right side of the equals sign first!
- *p means "Value at address". Pointer `p` holds 1000. Look back at `main`—the value at address 1000 is 5.
- m is our local variable, currently 15.
- Calculation: 5 + 15 = 20.
We assign this 20 to the left side (*p). We reach up into `main` and overwrite `i` with 20.

5. Return to main() & Final Output

The return; statement executes. Function f finishes and its memory block is theoretically destroyed. Control jumps back to `main`.

Code Line: printf("%d", i + j);

We look solely at our `main` memory block now.
Value of i is 20 (permanently altered by the pointer).
Value of j is 10 (untouched because it was passed by value).

20 + 10 = 30

The instructor runs the code live in Visual Studio Code at the end of the video, confirming the terminal output is exactly 30.

🎓 Key Student Takeaways

  • Draw Memory Blocks: When tracing C code in exams, always draw separate boxes for different functions. Variables inside main are isolated from variables inside custom functions unless pointers are used.
  • Call by Value: Passing just the variable name (e.g., j). It creates an isolated copy in the receiving function. Changing the copy does not affect the original.
  • Call by Reference: Passing the address of a variable (e.g., &i) into a pointer (*p). Using the * (dereference) operator allows the secondary function to reach back into main and permanently alter the original variable.
