The Golden Rule: Pointers vs. Array Names

You can write p = a; but you cannot write a = p;

Why? An array name (a) is a Constant Pointer. Its address is fixed in memory during compilation. You cannot modify a, a++, or a--. You can do these things to p.

2. Basic Pointer Increment (p++)

What actually happens when you increment a pointer?

  • The Math: p++ is evaluated as p = p + 1.
  • Pointer Rule: It does not just add the integer 1 to the address 1000. It adds 1 multiplied by the size of the data type.
  • Calculation:
    New p = 1000 + (1 * sizeof(int))
    New p = 1000 + (1 * 4)
    New p = 1004

Result: p now points to index 1 (the value 2).

3. Combining Dereference * and Increment ++

This is where most students get confused. Pay close attention to Pre vs. Post operations. (Assume p starts at 1000 for A and B)

A) Post-Increment

*p++

Rule: Use the value first, THEN increment.

  1. Read Original Address: p is 1000.
  2. Dereference: What is *1000? It is 3.
  3. Print: The function prints 3.
  4. Increment: Now increment p. Address becomes 1004.
End State: Printed 3 • p is now at 1004 (pointing to 2)

B) Pre-Increment

*++p

Rule: Increment first, THEN use the value.

(Associativity is Right-to-Left. It reads ++ before *)

  1. Increment: Increase p immediately. Address goes from 1000 to 1004.
  2. Dereference: What is *1004? It is 2.
  3. Print: The function prints 2.
End State: Printed 2 • p is now at 1004 (pointing to 2)
🔄

Reset for C & D: Assume we set p = &a[3];.
p now starts at address 1012, pointing to the value 0.

C) Post-Decrement

*p--

Rule: Use value first, THEN decrement.

  1. Read Original Address: p is 1012.
  2. Dereference: What is *1012? It is 0.
  3. Print: Prints 0.
  4. Decrement: Address becomes 1012 - 4 = 1008.
End State: Printed 0 • p is now at 1008 (pointing to 67)

D) Pre-Decrement

*--p

Rule: Decrement first, THEN use the value.

  1. Decrement: Decrease p immediately. Goes from 1012 to 1008.
  2. Dereference: What is *1008? It is 67.
  3. Print: Prints 67.
End State: Printed 67 • p is now at 1008 (pointing to 67)

4. The Tricky Stuff: Multiple Operations

What happens when you pass multiple pointer operations into a single function like printf?

p = a; // Reset p back to 1000
printf("%d %d", *p++, *p++);

The "Right-to-Left" Argument Rule

In C compilers like GCC, arguments passed to a function are evaluated from Right to Left before they are printed.

1. Evaluate Right Argument First: *p++
  • Current p = 1000.
  • Dereference 1000 = 3 (Saved for the second %d slot).
  • Increment p = 1004.
2. Evaluate Left Argument Next: *p++
  • Current p = 1004.
  • Dereference 1004 = 2 (Saved for the first %d slot).
  • Increment p = 1008.

5. Homework & Tricky Combinations

Dry-run these on paper first. Draw the memory blocks, use a pencil to track the address in p, and remember right-to-left evaluation!

printf("%d %d %d", *--p, *--p, *--p);

Evaluates right to left. Subtracts 4 from address, reads value, saves it. Does it three times.

--(*p)

Decrements the VALUE the pointer is looking at, not the address. Modifies the array itself.

(*p)++

Increments the VALUE (post-increment) the pointer is looking at. Array element increases by 1.

*p++  vs  *(p++)

These do the exact same thing. Post-increment pointer, evaluate original address.
