Low-Level vs. High-Level Programming Languages

An in-depth breakdown of how humans communicate with hardware, visualised step-by-step.

1. The Native Language of Computers

Computers are general-purpose machines capable of performing any computational task, but they are "nothing without programs." To make a computer work, we must submit a set of instructions (a program).

"The language of the computer is Binary Language. A computer can only understand 0s and 1s."

No matter what language you write your program in (C, C++, Java), the CPU processor eventually needs to execute it in pure 0s and 1s. This reality splits programming languages into two main levels based on how close they are to the hardware.

2. Low-Level: Machine Language

Machine language (or Machine Code) is purely **Machine Oriented**. It operates directly with the hardware.

  • Format: Written completely in 0s and 1s.
  • Difficulty: Requires extensive knowledge of the system configuration, memory management, and internal CPU architecture.

The Portability Problem (Hardware Dependency)

Machine code is heavily dependent on the specific architecture of the CPU. The instructor uses the example of instruction formatting:

CPU 1 (20-bit architecture for adding 1+2)

Opcode (4-bit)
0001 (+)
Operand 1 (8-bit)
00000001 (1)
Operand 2 (8-bit)
00000010 (2)

CPU 2 (32-bit architecture)

Opcode (6-bit)
000001
Operand 1 (13-bit)
Operand 2 (13-bit)

Because CPU 1 and CPU 2 expect bits in entirely different formats, a machine code program written for CPU 1 will crash on CPU 2. Machine language is NOT portable.

Advantage: Fastest execution. It's already in 0s and 1s, so the computer requires zero translation time.

Disadvantage: Unreadable for humans. "Hello World" looks like a random block of 01001000 01100101...

3. Low-Level: Assembly Language

To overcome the nightmare of writing in pure binary, Assembly Language was created. It uses English-like symbols and numbers called Mnemonics.

Important Note: Assembly is still considered "Low-Level". Why? Because there is a strict 1-to-1 correspondence with machine code. You still need to know how many bits your CPU uses and manage its registers manually. Therefore, it is still Machine Dependent and not portable.

4. High-Level Languages

Languages like C, C++, Java, Python, and PHP were designed with higher abstractions. They are completely separated from the hardware details.

  • Syntax: Uses mathematical notations (+, -), English words (if, while, print), variables, and loops.
  • No Hardware Knowledge Required: You don't need to know if the CPU uses 20-bit or 32-bit architecture. You just write c = a + b;
  • Portability: A program written on a Windows machine can be run on a Mac or Linux machine without rewriting the logic.

Translation: Compilers & Interpreters

Because High-Level languages are so far from binary, they require heavy translation back down to machine code.

(High-Level)

Object Code
(Binary)
  • Compiler: Takes the entire source code, translates it all at once into an object file (machine code), and then the CPU executes it.
  • Interpreter: Translates and executes the program line by line. It does not generate a separate compiled object file.
Advantage: Highly usable, readable, and portable.

Disadvantage: Execution is slower than low-level languages because of the heavy translation process required before the CPU can understand it.
