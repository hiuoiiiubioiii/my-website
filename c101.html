Function Pointers in C

Comprehensive Lecture Notes & Visualizations

1. The Basic Concept

You are likely familiar with standard pointers. An integer pointer holds the memory address of an integer variable. A float pointer holds the address of a float.

int *ptr;   // Pointer to an integer
float *fptr; // Pointer to a float

Similarly, a Function Pointer is a special variable that holds the memory address of a function. Just like data is stored in memory, executable code is also stored in memory, and we can point to it!

2. Syntax & Declaration

To declare a function pointer, you must exactly match the "signature" (prototype) of the function you want it to point to. You need to know two things:

  1. The function's return type.
  2. The data types of the arguments it accepts.
return_type Match function return
(*pointer_name) Must be in parentheses!
(arg_types) Match function params

Example:

If we have a function that adds two numbers:

int sum(int a, int b) {
    return a + b;
}

The pointer for this specific function is declared as:

int (*ptr)(int, int);

3. ⚠️ CRUCIAL: Operator Precedence

A very common exam and interview question is: Why do we need the parentheses around (*ptr)? Let's look at what happens if we forget them.

int (*ptr)(int, int);

Correct!

Parentheses force (*ptr) to evaluate first. The compiler sees: "A pointer named ptr that points to a function taking two ints and returning a normal int."

int *ptr(int, int);

Wrong!

Function call () has higher priority than *. The compiler reads ptr() first and thinks ptr is a normal function that returns an Integer Pointer (int *).

Interview Tip

Always remember: The asterisk and pointer name MUST be wrapped in parentheses to overcome the higher precedence of the function parameter parentheses.

Normal pointers point to data. Function pointers point to executable code. When we execute ptr(2,3), the CPU jumps to memory address 1000 and runs those instructions!

5. Initialization & Execution

Now that the pointer is declared, we assign it the address of our sum function and use it to execute the code.

    int s = 0;

    // 2. Call the function by dereferencing
    s = (*ptr)(2, 3); 
    
    printf("%d", s); // Prints 5
}

6. Practical Syntax Shortcuts

At the end of the video, the instructor demonstrates standard C compiler shortcuts. Because the name of a function automatically decays into a pointer (its base address), the & and * are actually optional!

Strict Syntax:
int (*ptr)(int, int) = ∑
s = (*ptr)(2, 3);

Modern/Shortcut Syntax:
int (*ptr)(int, int) = sum;
s = ptr(2, 3);

Both will compile correctly and output the exact same result. Most modern C programmers use the shortcut syntax for cleaner readability.

7. Complete Source Code

Here is the final, complete code written in VS Code by the instructor at the end of the lecture. Copy and paste this into your own IDE to test it!

#include <stdio.h>

// Function Definition
int sum(int a, int b) {
    return a + b;
}

void main() {
    int s = 0;
    
    // Function Pointer Declaration & Assignment
    // (Using the shortcut syntax)
    int (*ptr)(int, int) = sum;
    
    // Executing the function via pointer
    s = ptr(1, 2);
    
    printf("sum=%d\n", s); // Outputs: sum=3
}
