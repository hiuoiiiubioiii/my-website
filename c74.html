Pointer to Pointer (Double Pointer)

Complete visual breakdown & study notes based on the instructor's masterclass.

1. Recap: Single Pointers & Memory

Before jumping to double pointers, we need to establish a rock-solid foundation of how single pointers map to memory.

int a = 10;
int *p = &a;
  • int a = 10; reserves memory (e.g., at address 1000) and stores 10.
  • int *p tells the compiler p is a pointer variable. It gets its own memory (e.g., at address 850).
  • p = &a; stores the base address of a inside p.
  • Crucial Rule: The data type in int *p dictates what type of variable it is allowed to point to (only integers!).

10
Addr: 1000
int *p
1000
Addr: 850

2. The Core Concept: Double Pointers

What if we want a pointer to point to variable p? Because p is already a pointer, a normal pointer cannot hold its address. We need a Pointer to a Pointer (Double Pointer).

int **q;
q = &p;

Breaking down int **q:

  • The * closest to q tells the compiler: "Hey, q is a pointer."
  • The remaining part, int *, tells the compiler the data type of the variable q is pointing to. Because p is an int *, q must be an int **.
  • Warning: int **q = &a; is illegal. A double pointer MUST contain the address of a pointer variable.

3. Accessing Values (The Dereference Trace)

How do we print the original value 10 using our new double pointer q? We use the indirection (dereferencing) operator * sequentially.

Syntax Meaning Result
q Value stored directly inside q 850 (Address of p)
*q Value at address(850) 1000 (Address of a)
**q Value at address(1000) -> read as *(*q) 10 (Original Value)

4. Expanding the Chain: Triple Pointers

You can keep chaining these pointers! Let's create a pointer that points to q. Since q is a level-2 pointer, we need a level-3 pointer (***).

int ***r = &q;

To print 10 using r, you match the levels: printf("%d", ***r);

5. Modifying Values via Pointers

Because pointers map directly back to the original memory block, modifying data using a fully dereferenced double/triple pointer alters the core variable.

The "Illegal" Move: What happens if you write *q = 25;?

Answer: *q represents the box p. You would overwrite the memory address inside p (850) with the number 25. Because 25 is not a valid memory address you own, your program will crash!

6. Live Coding Proof (VS Code)

Here is the complete C program the instructor runs to prove these concepts. Notice how `%x` (or `%p`) is used to print addresses.

#include <stdio.h>

void main() {
    int a = 10;
    int *p = &a;      // Single pointer
    int **q = &p;     // Double pointer
    int ***r = &q;    // Triple pointer

    // 1. Printing the Value (All output "10")
    printf("a = %d\n", a);
    printf("a = %d\n", *p);
    printf("a = %d\n", **q);
    printf("a = %d\n", ***r);

    // 2. Printing Addresses
    printf("address of q = %x %x\n", r, &q); 
    printf("address of p = %x %x\n", q, &p); 

    // 3. Modifying original value
    **q = 17;
    printf("a = %d\n", a);  // Output: 17
}

🌟 Key Takeaways

  • Level Matching: A pointer must match the level of the address it is storing. (Address of int * requires int **).
  • Dereference Formula: To get the original value, the number of asterisks applied to the variable must match its declared level.
  • Memory Editing: Updating data using a correctly dereferenced pointer chain (e.g., ***r = 50;) directly rewrites the data in the original variable's block.
