C Pointers: & vs *

Complete visual study notes from the lecture

1 Introduction & Pointer Sizes

Pointers cause massive confusion because of two specific operators. Our goal is to master them today:

  • & (Ampersand): Address of operator (Referencing).
  • * (Asterisk): Indirection operator (Dereferencing).

Note on Pointer Sizes: The size a pointer takes up in memory isn't fixed—it depends on your computer's architecture (compiler/machine).

  • 16-bit machine: Size of pointer = 2 bytes
  • 32-bit machine: Size of pointer = 4 bytes

2 Visualizing Memory & Variables

Let's look at what happens behind the scenes when we declare variables and empty pointers.

int a = 10, b = 9;
int *p, *q;

The computer sets aside "boxes" in memory. Variables hold standard data. Pointer variables hold memory addresses. Since p and q haven't been assigned anything yet, they hold "garbage" values.

Variable 'a'
10
Address: 1000
Variable 'b'
9
Address: 2000
Pointer '*p'
Garbage
Address: 3046
Pointer '*q'
Garbage
Address: 2046

(Note: We use easy numbers like 1000 for the "Base Address", which is the starting byte of the 4 bytes an integer actually takes up).

3 Initializing Pointers with '&'

We use the & (Address of) operator to give our pointers a destination.

p = &a; // "p gets the Address of a"
q = &b; // "q gets the Address of b"

Now, look at how the memory changes. The pointers now store the exact physical memory addresses of our integer variables.

Variable 'a'
10
Address: 1000
Pointer 'p'
1000
Address: 3046
↳ Points to 'a'
Variable 'b'
9
Address: 2000
Pointer 'q'
2000
Address: 2046
↳ Points to 'b'

4 Tricky Exam Trap: Comma vs Assignment

What happens if you write multiple assignments on one line? This is a common exam question.

Scenario 1: No Brackets

p = &a, &b;

The Assignment Operator (=) has a higher precedence than the Comma Operator (,). So, p = &a executes first. The &b is discarded. Result: p points to a.

Scenario 2: With Brackets

p = (&a, &b);

Brackets have highest precedence. The Comma Operator evaluates left-to-right, discarding the left and returning the right. It evaluates &a, tosses it, evaluates &b and returns it. Result: p points to b.

5 The Indirection (*) Operator

How do we use a pointer to actually see the data it is pointing to? We use the * (Indirection/Dereferencing) operator. It literally translates to "Value at Address..."

The Ankit & Pulkit Analogy:
Think of Variable a as Ankit (Value 10). You want to visit Ankit, but don't know his address.

Pointer p is Pulkit. Pulkit DOES know Ankit's address.
You go to Pulkit (p), get the address, and use it to find Ankit. This is Indirect Access. 
printf("Direct: %d", a);  // Output: 10
printf("Indirect: %d", *p); // Output: 10

Mathematical breakdown: When the compiler reads *p, it translates it to *(&a). It means: "Give me the Value at the Address of a".

6 Printing Memory Addresses

You cannot use %d to print a memory address, it will just show garbage integers. You must use %x (Hexadecimal) or %p (Pointer formatting).

// Printing the Address of variable 'a'
printf("%x", &a); 

// Printing the value stored INSIDE 'p' 
// (which is the address of 'a')
printf("%x", p);  

// Printing the physical address where pointer 'p' itself lives
printf("%x", &p);

Note: &a and p will print the exact same hexadecimal string (e.g., 61fe14).

7 The Whiteboard Code Trace Challenge

Try to trace this code step-by-step as written on the board:

int a=10, b=9;
int *p, *q, c;

p = &a;
q = &b;

c = *q;  // Step 1
*p = 20; // Step 2

Instructor Breakdown:

  1. c = *q; -> q holds the address of b. The * means "value at". So c gets the value at b. Variable c is now 9.

  2. *p = 20; -> The *p means "the value stored at the address inside p". It goes to address 1000 (where a lives) and overwrites the 10. Variable a is now permanently changed to 20!

8 VS Code Practical Demo Output

When executing this in real life, the computer generates real RAM addresses instead of "1000". Note how pointers are dynamic and can be reassigned.

void main() {
    int a=10, b=9;
    int *p = &a, *q = &b;

    printf("Value of a = %d \n", a);
    printf("Value of a = %d \n", *p);
    
    printf("Address of a = %x \n", &a);
    printf("Address of a = %p \n", p);

    // Changing pointer destination
    q = &a; 
    printf("Value of *q = %d \n", *q); 
}
> Value of a = 10
> Value of a = 10
> Address of a = 61fe14
> Address of a = 000000000061FE14
> Value of *q = 10
