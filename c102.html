Callback Functions in C

In-Depth Study Notes & Memory Visualizations

📘 1. Prerequisite & Introduction

Before diving into callback functions, you must understand the basics of Pointers in C:

  • Normal Pointer: Contains the memory address of data/variables (e.g., an int or char).
  • Function Pointer: Contains the base address of a function's code in memory.

What is a Callback Function?

A callback function is an application of function pointers. Simply put, it is a function passed into another function as an argument, which is then invoked inside the outer function to complete some kind of routine or action.

💡 2. The Core Concept (The "Why")

The instructor sets up a scenario to explain why we use this mechanism:

  • Imagine two simple functions: Sum() and Sub().
  • You have a third function called display().
  • Usually, you pass variables to functions. But here, we want display() to execute either Sum or Sub dynamically at runtime.
  • To do this, we pass the address of the Sum or Sub function to display().
  • display() receives this address using a Function Pointer, and then "calls back" the function passed to it.

✍️ 3. Whiteboard Code Breakdown

Here is the step-by-step code the instructor wrote on the whiteboard.

Step 1: Defining the Callback Functions

First, define two standard functions. Note that their signatures (return types and parameter types) are identical.

void Sum(int a, int b) {
    printf("sum=%d\n", a+b);
}

void Sub(int a, int b) {
    printf("sub=%d\n", a-b);
}

Step 2: Defining the Receiving Function

The display function needs to accept the address of another function. Its parameter must be a function pointer that perfectly matches the signature of Sum and Sub.

// The parameter: void (*fptr)(int, int) 
// This reads as: fptr is a pointer to a function that takes two ints and returns void.

void display(void (*fptr)(int, int)) {
    // Dereferencing the pointer to call the function
    // Passing hardcoded arguments 5 and 1
    (*fptr)(5, 1); 
    
    // Note: The instructor mentions you can also just write: fptr(5, 1);
}

Step 3: The Main Function

void main() {
    // Pass the base address of the Sum function
    display(&Sum);  // or simply display(Sum);

    // Pass the base address of the Sub function
    display(Sub);
}

Execution Trace:

  1. main calls display(Sum). The base address 1000 is passed.
  2. Inside display, the pointer fptr receives the value 1000.
  3. The code runs (*fptr)(5, 1). Because fptr points to 1000, it executes the code for Sum(5, 1). It prints 6.
  4. Control returns to main. main calls display(Sub). The base address 1046 is passed.
  5. fptr is overwritten with 1046.
  6. The code runs (*fptr)(5, 1), which executes Sub(5, 1). It prints 4.

⚠️ 5. Crucial Rules & Pitfalls

Signature Matching is Mandatory!
The instructor stressed that if you change the return type of Sum to float, it will no longer match the function pointer parameter void (*fptr)(int, int).

If the signatures (return type and number/type of arguments) do not match perfectly, the function pointer cannot safely point to your callback function. This will result in compiler warnings, errors, or unpredictable runtime behavior (segmentation faults).
