1. Formatted vs. Unformatted Functions

Before writing code, the instructor establishes the context of input functions in C.

  • Formatted Input/Output: Functions like scanf() and printf(). They require Format Specifiers (e.g., %d for integers, %f for floats, %c for chars). You can format exactly how the data looks and mix data types.
  • Unformatted Input/Output: Functions that do not use format specifiers during input. They are rigid, primarily working with character data types (a single character or a string).

The instructor lists 4 unformatted input functions on the board, which we will deeply analyze below:

getchar(), getch(), getche(), and gets().

2. getchar() <stdio.h>

What it does: Reads a single character from the standard input device (keyboard).

  • Waits for Enter: It will buffer your input and will not process it until you press the "Enter" key.
  • Echoes: The character you type will be visible on the output screen.
  • Buffering behavior: If you type multiple characters (e.g., "JKLMNO") and hit Enter, it only reads and stores the very first character ("J").

Instructor's Code:

#include <stdio.h>

void main() {
    char ch;
    ch = getchar();        // Takes input, no arguments needed inside ()
    printf("ch=%c", ch);   // Prints the input to verify
}

Visual Output Simulation:

Console - getchar.exe
J // User types 'J' (Echoed on screen)
// User presses [ENTER] key
ch=J // Program prints the stored character
_

3. getch() <conio.h>

What it does: Reads a single alphanumeric character from the keyboard.

  • Immediate Execution: As soon as you press a key, the function executes instantly. It does NOT wait for the Enter key.
  • NO Echo: The character you type is hidden. It will NOT be displayed on the screen.
  • Common Use Cases: Taking passwords securely, or placing it at the very end of main() to "hold the output screen" until the user presses any key to exit.

Instructor's Code:

#include <conio.h>

void main() {
    char ch;
    ch = getch();          // Takes input instantly
    printf("ch=%c", ch);   // Prints the input to verify
}

Visual Output Simulation:

Console - getch.exe
// User presses 'a' key
// Notice: 'a' is NOT shown on screen (No Echo)
// Program immediately jumps to next line without waiting for Enter
ch=a
_

4. getche() <conio.h>

What it does: Exactly the same as getch(), but the 'e' stands for "Echo".

  • Immediate Execution: Executes as soon as you press a key (No Enter needed).
  • Echoes: The character you type WILL be displayed on the screen instantly.

Instructor's Code:

#include <conio.h>

void main() {
    char ch;
    ch = getche();         // Takes input instantly AND echoes it
    printf("ch=%c", ch);
}

Visual Output Simulation:

Console - getche.exe
a // User presses 'a' (Echoes on screen instantly)
ch=a // Program immediately executes print statement right next to it
_

5. gets() <stdio.h>

What it does: Reads a complete string (sequence of characters) from standard input until the Enter key is pressed.

  • Why use it over scanf? If you use scanf("%s"), it stops reading as soon as it encounters a space. gets() reads the entire line, including spaces.
  • Requires a character array (string declaration) rather than a single character variable.

Instructor's Code:

#include <stdio.h>

void main() {
    char ch[30];           // Declares a string (char array) to hold the words
    gets(ch);              // Reads string WITH spaces. Pass variable name inside ()
    printf("ch=%s", ch);   // Uses %s to print the string
}

Visual Output Simulation:

Console - gets.exe
Jenny's lectures // User types string with a space
// User presses [ENTER] key
ch=Jenny's lectures // Whole string is captured and printed
_
