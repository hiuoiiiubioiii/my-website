Recursion & Static Variables (GATE 2016 Question Breakdown)

↓ Scroll down to trace the code step-by-step ↓

1. The Problem Statement

This exact question appeared in the GATE 2016 examination. Take a moment to read the code before we break it down.

#include <stdio.h>

void count(int n) {
    static int d = 1;
    
    printf("%d ", n);
    printf("%d ", d);
    
    d++;
    
    if (n > 1) {
        count(n - 1);
    }
    
    printf("%d ", d);
}

void main() {
    count(3);
}

2. Instructor's Pre-Analysis

Before executing the code, the instructor identifies two crucial characteristics of this function:


Direct Recursion The function count() calls itself directly inside its own body.

Non-Tail Recursion The recursive call count(n - 1) is not the last statement to execute. Because there is a printf statement after it, the function must pause, execute the child function, and then return back to finish the final print statement.

3. ⚠️ The Trap: The `static` Keyword

Many students get this question wrong because they misunderstand how static int d = 1; works in recursion.

  • Initialization happens ONLY ONCE: The variable d is set to 1 only during the very first time the function is called. Future recursive calls skip this line entirely.
  • Global Lifetime, Local Scope: It is stored in a separate memory location, not on the local function stack. It retains its updated value across all function calls.
Global/Static Memory Space
Shared by all recursive calls
d = 1

4. The Dry Run: Moving Forward

We start in main() and trace the execution moving down the recursion tree.

Step 1: count(3) is called

Local n: 3

Static d: 1 (Initialized for the first and only time)

  • Prints n (3)
  • Prints d (1)
  • d++ happens. Static Memory Box d becomes 2.
  • Condition (3 > 1) is True. Calls count(2).
  • Note: The final printf(d) is paused and waiting in memory!
Output so far: 3 1
Step 2: count(2) is called

Local n: 2

Static d: 2 (Retains value from Step 1)

  • Prints n (2)
  • Prints d (2)
  • d++ happens. Static Memory Box d becomes 3.
  • Condition (2 > 1) is True. Calls count(1).
  • Note: Another printf(d) is paused and waiting!
Output so far: 3 1 2 2
Step 3: count(1) is called

Local n: 1

Static d: 3 (Retains value from Step 2)

  • Prints n (1)
  • Prints d (3)
  • d++ happens. Static Memory Box d becomes 4.
  • Condition (1 > 1) is False! Recursion stops moving forward.
Output so far: 3 1 2 2 1 3

Here is what the instructor draws on the whiteboard. Notice how the final statements are "paused" waiting for the recursive calls to finish.

6. The Unwinding Phase: Moving Backward

Because this is Non-Tail Recursion, we must travel backward up the tree to execute those paused printf("%d ", d); statements.

Current Global State
d = 4
Step 4: Finishing count(1)

We skip the if block and execute the final statement: printf(d).

Because d is static and shared, we look at the Global Memory Box. d is currently 4.

Prints: 4
Step 5: Finishing count(2)

We return to the paused state of count(2) and execute its final statement: printf(d).

The global memory box still says d is 4.

Prints: 4
Step 6: Finishing count(3)

We return to the paused state of count(3). Execute final printf(d).

The global memory box still says d is 4.

Prints: 4

7. Final Output & Takeaways

Combining the forward execution phase and the backward unwinding phase, we get the final answer to the GATE 2016 question:

3 1 2 2 1 3 4 4 4

📝 Pro-Tips for Exams:

  • Always draw a recursion tree: Write out a visual tree just like the instructor did to keep track of paused function calls.
  • Separate Local vs. Global/Static variables: Keep a designated spot on your scratch paper for static variables. Do not tie them to a specific branch of the tree!
  • Remember the Unwinding Phase: If there is code after the recursive call, you are not done when the base case is hit. You must walk back up the tree.
