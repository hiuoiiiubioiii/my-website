The If-Else Statement

1. Introduction to If-Else

In the previous lecture, we covered the simple if statement. A simple if statement only executes a block of code if a condition is true. If false, it does nothing.

The if-else statement is an extension of this. It provides two distinct blocks of code. The program will NEVER execute both blocks; it chooses one path based on the condition.

🛒 Real-Life Analogy: The Supermarket Task: You are sent to the supermarket to buy fruit.
Logic: If oranges are available, buy oranges. Else (otherwise), buy mosambis (sweet limes). If neither are available, just come home.

2. General Syntax

Here is how you structure an if-else statement in C. Pay close attention to the curly braces { }.

if (Condition)
{
    // True-block statements
    // Executes ONLY if condition is 1 (True)
}
else
{
    // False-block statements
    // Executes ONLY if condition is 0 (False)
}

// Statements here run after the if-else block finishes
⚠️ The Curly Brace Rule
  • Multiple Statements: If you have more than one statement inside your if or else block, you must use curly braces.
  • Single Statement: If you only have one statement, curly braces are optional (though using them is highly recommended for readability).

4. Live Code Example & Tracing

Let's look at the complete program written by the instructor and trace how the computer executes it based on different user inputs.

void main()
{
    int age;
    printf("enter age: ");
    scanf("%d", &age);
    
    if (age > 25 && age < 30)
    {
        printf("age : %d \n", age);
        printf("Coffee with me \n");
    }
    else
    {
        printf("age : %d \n", age);
        printf("Go home \n");
    }
    
    printf("Out of if-else");
}
Scenario A: User Inputs 28

Check Condition: (28 > 25 && 28 < 30)

Both sides of the Logical AND (&&) evaluate to True (1). The program enters the if block.

The else block is completely skipped.

enter age: 28
age : 28
Coffee with me
Out of if-else
Scenario B: User Inputs 20

Check Condition: (20 > 25 && 20 < 30)

The first part (20 > 25) is False (0). Due to short-circuiting in C, it stops checking. The whole condition is False.

The if block is skipped. It jumps to else.

enter age: 20
age : 20
Go home
Out of if-else

5. Exam Pitfalls & Compiler Errors

These are the exact tricks examiners use to test your understanding of C syntax rules. Read carefully!

Trick 1: Semicolon after the Condition

if (age > 25 && age < 30);  // <-- Notice the semicolon!
{
    printf("Coffee");
}
else 
{
    printf("Go home");
}
Result: Compiler Error ("Misplaced else") The semicolon immediately terminates the if statement. The block below it becomes a disconnected block of code. When the compiler hits the word else, it crashes because an else MUST immediately follow an if block.

Trick 2: Missing Curly Braces

if (age > 25 && age < 30)
    printf("age : %d \n", age);
    printf("Coffee with me \n");  // <-- This breaks the flow!
else
    printf("Go home");
Result: Compiler Error ("Misplaced else") Without curly braces, the if statement only "owns" the very first line beneath it. The compiler considers the "Coffee with me" line as normal code happening after the if statement. Because of this intervening line of code, the else is no longer attached to the if, causing a syntax error.
