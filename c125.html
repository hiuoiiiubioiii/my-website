1. What is `w+` Mode?

The w+ mode is a file handling mode in C that opens a file for both reading and writing. While it shares similarities with the r+ mode, its fundamental behavior regarding file creation and existing data is entirely different.

Rule 1 (File Doesn't Exist): The compiler will automatically create a brand new, empty file.
Rule 2 (File Exists): If the file already exists, it will instantly truncate the file length to zero. This means all previous content is completely erased the moment the file is opened.

2. The Big Difference: `r+` vs `w+`

Because both modes allow reading and writing, students often get confused. The instructor emphasizes this crucial comparison:

Feature r+ (Read/Write) w+ (Write/Read)
Primary Purpose Updating existing files. Creating new files / Overwriting entirely.
If file doesn't exist Returns NULL (Fails). Creates a new empty file.
If file DOES exist Keeps previous content safe. Erases everything instantly.
Cursor start position Beginning of existing text. Beginning of the empty file.

4. The Code & The "Gotcha"

The instructor writes the following code to demonstrate a common trap students fall into when trying to write, and then immediately read a file.

#include <stdio.h>
#include <stdlib.h>

void main() {
    FILE *fp = NULL;
    char ch;

    // 1. Open file. "Jenny" is erased here.
    fp = fopen("abc.txt", "w+");

    if (fp == NULL) {
        printf("Cannot open file");
        exit(1);
    }

    // 2. Write new content. Cursor moves to the end.
    fputs("Khatri", fp);

    // 3. Attempt to read the file to the console
    while (!feof(fp)) {
        ch = fgetc(fp);
        printf("%c", ch);
    }

    fclose(fp);
}

🛑 Why doesn't this print "Khatri"?

If you run this code, the screen will be completely blank! Why?

After fputs finishes writing "Khatri", the file pointer (cursor) is sitting at the very end of the file. When the while loop starts, it checks feof(fp) (End of File).

  • Because the cursor is already at the end, feof(fp) returns 1 (True).
  • The condition is !feof(fp). The NOT operator (!) turns that 1 into a 0 (False).
  • Because the while condition is False, the loop breaks immediately. fgetc is never executed.

5. The Solution: `rewind()`

To fix this issue, you must move the file pointer from the end of the text back to the very beginning before you start reading.

C provides a special function for this: rewind()

fputs("Khatri", fp);  // Writes text, cursor is at end

rewind(fp);           // Moves cursor to index 0

while (!feof(fp)) {    // Now this loop works perfectly!
    ch = fgetc(fp);
    printf("%c", ch);
}

6. IDE Practical Demonstration

In VS Code, the instructor proved this concept with a practical run:

  1. She created an abc.txt file containing: "This is awesome".
  2. She ran the program using w+ mode and wrote "Jenny".
  3. Result without rewind: Blank output screen. abc.txt changed to just "Jenny".
  4. Result with rewind: The output screen successfully printed "Jenny".
