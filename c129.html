The rewind() Function

In-depth study notes from the C File Handling lecture. Keep scrolling to learn the concept, see the code, and interact with the file pointer.

1. What is rewind()?

In file handling, whenever you read or write data, an internal "cursor" (called the file pointer) moves forward. If you reach the middle or the end of the file and realize you need to read the file from the very beginning again, you have two choices:

  • The hard way: Close the file using fclose() and reopen it using fopen().
  • The smart way: Use the rewind() function.
Analogy: Just like rewinding an old VHS tape or cassette, rewind() instantly rolls the file pointer back to the starting index 0 of the file.

Syntax

void rewind(FILE *fp);
  • Argument: It takes exactly one argument, which is the file pointer (fp) of the file you want to rewind.
  • Return Type: It returns void (it does not return any value).

Setup & Opening the File: She chooses the "r+" mode (read/update). She specifically notes that if you use "w+", it will erase the existing content of the file immediately.

Moving the Cursor manually: Using fseek, she moves the cursor 6 bytes forward from the beginning (SEEK_SET).

3

First Reading Loop: She uses a standard loop to read from the current cursor position to the end of the file.

while (!feof(fp)) {
    ch = fgetc(fp);
    printf("%c", ch);
}
// OUTPUT: "s lectures is great channel for CS/IT"
// Cursor is now at the END of the file.
4

The Rewind: She wants to print the entire file from the start. Instead of reopening the file, she simply rewinds it.

rewind(fp); 
// Cursor snaps back to index 0.
5

Second Reading Loop: The exact same while loop is run again.

while (!feof(fp)) {
    ch = fgetc(fp);
    printf("%c", ch);
}
// OUTPUT: "Jenny's lectures is great channel for CS/IT"

fclose(fp);

5. Key Takeaways 🎯

  • rewind(fp) is the dedicated function to reset the file pointer to the beginning of the file (index 0).
  • It saves processing time and code lines because you don't have to fclose() and fopen() again.
  • Writing fseek(fp, 0, SEEK_SET); essentially does the exact same thing under the hood. rewind() is just a cleaner, more readable shorthand for this.
  • Always be mindful of your opening modes ("r+" vs "w+"). Rewinding a file opened in "w+" after reading won't give you old data, because "w+" erased it upon opening. Write first, then rewind, then read.
