📝 Program to Count Lines in a File

In-depth Study Notes & Code Visualizations

1. The Core Logic: How Computers See Lines

When you look at a text file, you see distinct lines of text. But how does a computer actually know when one line ends and another begins?

Whenever you press the Enter key, the computer inserts a hidden character called the New Line Operator, represented in C as \n.

Hi
Hello
Hi\n
Hello
The Mathematical Formula:

If we simply count the \n characters, we will find exactly 1 in the example above. However, there are visually 2 lines. Therefore, the logic rule is:

Total Lines = (Count of \n) + 1

2. Visualizing Edge Cases

Before writing code, we must ensure our logic holds true for weird scenarios (Edge Cases). Let's test two scenarios from the whiteboard.

Hi\n
Hello\n
\n
Jenny

Test 1: Multiple Lines

In the first box, we have 3 \n characters. Using our formula (3 + 1), we get 4 lines. If you count the lines (Hi, Hello, Empty Line, Jenny), there are exactly 4 lines. The logic works!

Test 2: Empty File

In the second box, the file is completely empty. There are 0 \n characters. Using our formula (0 + 1), we get 1 line. Even an empty file technically has one line where the cursor sits. The logic works here too!

⚠️ Crucial Coding Rule: Because of this logic, when we initialize our counting variable in C, we must start it at 1, not 0.
int count = 1;

3. The C Program Implementation

Here is the complete C program exactly as written on the whiteboard, fully commented for understanding.

#include <stdio.h>
#include <stdlib.h>

void main() {
    // 1. Create a file pointer
    FILE *fp = NULL;

    // 2. Initialize our counter to 1 (Based on our logic)
    int count = 1;

    // 3. Variable to hold each character we read
    char ch;

    // 4. Open the file in "r" (Read) mode
    fp = fopen("abc.txt", "r");

    // 5. Safety Check: Did the file open successfully?
    if (fp == NULL) {
        printf("Error: Cannot open file");
        exit(1); // Exit program if file doesn't exist
    }

    // 6. The Core Loop: Read character by character
    while ((ch = fgetc(fp)) != EOF) {
        
        // 7. Check if the character is a New Line
        if (ch == '\n') {
            count++; // Or: count = count + 1;
        }
    }

    // 8. Close the file to free memory
    fclose(fp);

    // 9. Print the final result
    printf("Number of lines are: %d\n", count);
}

Key Functions Used:

  • fopen(filename, "r"): Opens the file in read mode.
  • fgetc(fp): Fetches a single character from the file pointer and automatically moves the cursor to the next character.
  • EOF: Stands for "End Of File." It is a special macro in C that signals there are no more characters to read.

5. Alternative Approach (The VS Code Method)

During the practical demonstration, the instructor showed a slightly different way to write the exact same logic using a for loop instead of a while loop. This is a great way to write cleaner, more compact code.

Instead of the while loop:

while ((ch = fgetc(fp)) != EOF) {
    if (ch == '\n') {
        count++;
    }
}

You can write this for loop:

// for (initialization; condition; increment)
for (ch = fgetc(fp); ch != EOF; ch = fgetc(fp)) {
    if (ch == '\n') {
        count++;
    }
}

Why this works: fgetc(fp) is used in the initialization to grab the very first character. It is also used in the increment step to grab the next character at the end of every loop cycle!
