Structure Packing in C

A complete video breakdown & visualization of memory management

1. Recap: What is Structure Padding?

Before understanding packing, we must review padding. By default, the compiler inserts empty, wasted bytes (padding) into your structures to align data in memory. This helps the CPU read the data faster.

struct abc {
    char a;  // 1 byte
    int b;   // 4 bytes
};

void main() {
    struct abc s;
    printf("%d", sizeof(s)); // Output: 8
}

The Instructor's Whiteboard Math: Intuitively, 1 byte + 4 bytes = 5 bytes. But the actual output is 8 bytes. Why? The compiler added 3 padding bytes after the char so the int aligns cleanly on a 4-byte boundary.

2. Enter Structure Packing

What if you don't want to waste those 3 bytes? What if memory space is strictly limited? This is where Structure Packing comes in. We can force the compiler to pack everything tightly together.

To do this, the instructor adds a preprocessor directive at the very top of the program:

#pragma pack(1)

struct abc {
    char a;
    int b;
};

Instructor's Verbal Note: The #pragma pack(1) directive instructs the compiler to align structural members on a 1-byte boundary. In simple terms: turn off padding.

4. The Ultimate Trade-Off

If packing saves so much memory, why isn't it the default? The instructor emphasizes a critical programming concept: The Trade-off between Memory and Performance.

✅ Pros of Packing

Saves Memory: Highly efficient space usage. Crucial for embedded systems with tiny RAM, or when sending large amounts of structured data over a network.

❌ Cons of Packing

Performance Penalty: It takes the CPU extra cycles to read an unaligned integer. The CPU has to perform multiple read operations and stitch the data together, slowing down execution.

5. Instructor Pro-Tip: Smart Ordering

During her VS Code demonstration, she reveals a "smart way" to save memory without taking the CPU performance hit of #pragma pack(1). You can achieve this by simply grouping variables of the same size together.

// Notice the order: char, char, int
struct abc {
    char a;  // 1 byte
    char c;  // 1 byte
    int b;   // 4 bytes
};
