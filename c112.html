📚 Array of Structures in C

Instructor: Jayanti Khatri Lamba | Comprehensive Study Feed

1. Recap: Structures & Structure Variables

Before diving into arrays, let's establish the foundation. A structure is a user-defined data type that groups different data types together.

struct Student {
    int rollno;
    char name[20];
    float marks;
};
💡 Instructor Note: The items inside the curly braces (rollno, name, marks) are called Structure Members. They are not variables, and they do not occupy any memory yet.

To actually allocate memory and hold data, you must declare a Structure Variable (also known as an object):

struct Student s1, s2; // Variables for 2 students

2. The Problem: Why an Array of Structures?

Declaring s1, s2 is fine for two students. But what if you need to store data for 60 students (or 1000 employees)?

  • ❌ The Bad Way: Declaring s1, s2, s3... s60. This creates a massive, unreadable program and makes it impossible to remember variable names.
  • ✅ The Solution: Use an Array of Structures. This allows you to store a large collection of objects under one single name, using an index to access each one.

3. Declaring the Array

To solve the 60-student problem, we declare an array of type struct Student.

void main() {
    // Creating an array of 60 student objects
    struct Student S[60]; 
}

What happens here? S is an array containing 60 elements. Each element is a complete struct Student object, meaning each element gets its own individual rollno, name, and marks.

To make the concept easy to visualize, let's shrink the array size from 60 down to 3: struct Student S[3];. Here is exactly how memory is allocated internally.

Deep Dive into Memory Math: Look at S[0]. The base address is assumed to be 1000.

  • rollno is an integer (4 bytes). It occupies addresses 1000 to 1003.
  • name is a char array of size 20 (20 bytes). It starts directly after at 1004 and goes to 1023.
  • marks is a float (4 bytes). It starts at 1024.

5. Accessing Array Members

To insert or read data from these specific memory blocks, you combine the Array Index with the Dot (.) Operator.

If you want to access the data for the first student (Index 0), you write:

  • S[0].rollno
  • S[0].name
  • S[0].marks

For the second student (Index 1):

  • S[1].rollno, S[1].name, S[1].marks

6. Writing the Code: User Input

Because this is an array, we must use loops to input the data. We do not write 60 individual scanf statements. We loop through the indices 0, 1, and 2.

int i;

for(i = 0; i < 3; i++) {
    // Taking input for student 'i'
    scanf("%d %s %f", &S[i].rollno, S[i].name, &S[i].marks);
}
⚠️ CRITICAL RULE for `scanf`: Watch your `&` operators!

You must use the `&` (address of) operator for integers and floats: &S[i].rollno and &S[i].marks.

You must NOT use the `&` operator for the string: S[i].name. Why? Because name is a character array, and the name of an array inherently acts as its base address!

7. Writing the Code: Printing Output

To print the data back to the screen, we use a nearly identical loop, replacing scanf with printf.

for(i = 0; i < 3; i++) {
    // Printing output for student 'i'
    printf("%d %s %f\n", S[i].rollno, S[i].name, S[i].marks);
}

Note: printf only needs the values, not the memory addresses. Therefore, the & operator is never used here for any data type.
