2. Theory Review

Rules of the Switch Statement

The instructor verbally emphasizes two crucial rules to remember when using the switch statement:

  • Rule 1: Unique Cases. Two cases cannot have the same value. If you write case 2: and then write another case 2: later in the block, the compiler will throw a "duplicate case" error.
  • Rule 2: Character Constants. Because our mathematical operators are characters (symbols), they must be wrapped in single quotes inside the case statement (e.g., case '+':). This tells the C compiler to treat it as a character constant based on its ASCII value.
3. The Code

Complete Whiteboard Program

Here is the exact C program written by the instructor on the whiteboard. Take note of the variables declared at the top and the structure of the switch block.

#include <stdio.h>
#include <conio.h>

void main() {
    int a, b, sum, sub, div, mul;
    char operator;

    // 1. Ask for Operator
    printf("Enter operator: ");
    scanf("%c", &operator);

    // 2. Ask for Numbers
    printf("Enter two operands: ");
    scanf("%d %d", &a, &b);

    // 3. Switch Logic
    switch(operator) {
        case '+':
            sum = a + b;
            printf("Sum = %d", sum);
            break;
        case '-':
            sub = a - b;
            printf("Sub = %d", sub);
            break;
        case '*':
            mul = a * b;
            printf("Mul = %d", mul);
            break;
        case '/':
            div = a / b;
            printf("Div = %d", div);
            break;
        default:
            printf("Enter valid operator");
    }
    getch();
}
4. Code Breakdown

Why was it written this way?

Let's combine the instructor's audio explanations with the code:

  • The scanf formats: For the operator, she uses %c to grab a single character. For the numbers, she grabs both at once using "%d %d". Remember to use the ampersand (&) to store inputs at specific memory addresses!
  • Printing results: She verbally reminds the class that you do not use an & inside a printf statement, only in scanf.
  • The break keyword: Crucial step! The instructor explains that break forces the program to exit the switch block entirely. If you omit it, the program will "fall through" and execute the code in the cases below it.
  • The default net: This catches errors. If a user types the letter 'X' instead of a math symbol, the program jumps to default and prints an error message.
