📝 The ftell() Function in C

Comprehensive Lecture Notes & Code Visualizations

🧠 1. Introduction to Random File Access

In C programming, when dealing with File Handling, we sometimes need to read or write data from specific, random locations in a file rather than just reading it sequentially from beginning to end.

  • Previous Knowledge: The fseek() function is used to move the file pointer to a specific location.
  • Today's Topic: The ftell() function is used to find out exactly where that file pointer currently is.

👁️ 2. Core Concept: What does ftell() do?

Definition: As the name suggests, it "tells" you the current position of the file pointer. It returns the current position as the number of bytes (or characters) calculated from the starting position of the file.

Scenarios based on the whiteboard:

  • Scenario 1: When you first open the file in read mode ("r"), the pointer is at the very beginning ('J').
    ➔ ftell() will return: 0
  • Scenario 2: If you use fseek() to move the pointer to the letter 'k'.
    ➔ ftell() will return: 6
💡 Major Application: Because ftell() counts bytes from the very start of the file, its most common practical application is finding the total length (size) of a file.

📜 3. Syntax and Return Type

Here is the formal syntax the instructor writes on the board:

long int ftell(FILE *pointer);
  • Return Type (long int): It returns an integer representing the position. It uses long int because files can be very large, and the byte count might exceed the limit of a standard int.
  • Argument (FILE *pointer): It takes exactly one argument—the file pointer of the file you are currently working with.

💻 4. Step-by-Step Code Breakdown

The instructor writes a complete C program to demonstrate how ftell() reacts as the file pointer moves. Here is the synthesized code:

ftell_demo.c
#include <stdio.h>
#include <stdlib.h>

void main() {
    FILE *fp = NULL;
    int pos;         // To store location from ftell
    char ch;         // To read a single character
    char str[50];    // To read a string

    // 1. Open the file
    fp = fopen("abc.txt", "r");
    if (fp == NULL) {
        printf("Can't open file");
        exit(1);
    }

    // 2. Initial Position (Pointer is at index 0)
    pos = ftell(fp); 
    printf("%d\n", pos);  // Output: 0

    // 3. Move the pointer using fseek (5 bytes forward)
    fseek(fp, 5, SEEK_SET); 
    printf("%d\n", ftell(fp)); // Output: 5 (At space after "Jenny")

    // 4. Read character (Moves pointer forward by 1 byte)
    ch = fgetc(fp); 
    printf("%c\n", ch);        // Output: (prints blank space)
    printf("%d\n", ftell(fp)); // Output: 6 (Moved past space to 'k')

    // 5. Read string (Reads until space)
    fscanf(fp, "%s", str);     // Reads "khatri" (6 letters)
    printf("%s\n", str);       // Output: khatri
    printf("%d\n", ftell(fp)); // Output: 12 (6 + 6 letters read)

    fclose(fp);
}

📏 5. Finding the Length of a File

The instructor highlights this as a crucial exam and interview concept. How do you find the exact size of a file using ftell?

The Logic:
1. Move the file pointer to the absolute end of the file using fseek().
2. Ask ftell() where the pointer is. Because it counts from 0, the position at the end of the file IS the total byte count! 
// Move pointer to the end (0 offset from SEEK_END)
fseek(fp, 0, SEEK_END); 

// Print the current location
printf("Total file length: %d\n", ftell(fp));

Ultimate Takeaways:

• Remember that reading operations (fgetc, fscanf) consume bytes and push the pointer forward automatically!
