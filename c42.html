Continue Statement in C

In-depth Visual Study Notes & Whiteboard Breakdown

1. Context & Recap: How break works

Before understanding `continue`, we must recall how the break statement behaves. They are both control statements used within loops, but they do very different things.

The `break` Behavior

When a `break` statement is encountered inside a loop, it immediately terminates the loop completely. The program control is thrown completely out of the loop, and no further iterations occur.

for (initialization; condition; update) {
    S1;
    S2;
    if (condition) {
        break; // TERMINATES LOOP
    }
    S3; // Skipped forever
    S4; // Skipped forever
}
// Control jumps down here immediately ↘️

Visual Flow of Break:

[Loop Body] ➔ 💥 BREAK ➔ ⤓ Jumps OUT of loop entirely

2. Core Concept: How continue works

Unlike `break`, `continue` does not throw you out of the loop entirely. Instead, it is used to skip specific statements for the current iteration only.

When `continue` is triggered, the program skips the remaining code below it and jumps straight back to the update expression (e.g., `i++`) to start the next iteration.

for (initialization; condition; update (i++)) {
    S1;
    S2;
    if (condition) {
        continue; // JUMPS UP TO UPDATE ⤴️
    }
    S3; // Skipped for THIS iteration only
    S4; // Skipped for THIS iteration only
}

Visual Flow of Continue:

[Loop Body] ➔ ⤴️ CONTINUE ➔ ↺ Jumps UP to Loop Update (i++)

S3 and S4 get crossed out (X) for that cycle!

3. The Real-Life Analogy: "Jenny's DSA Classes"

Imagine there are 100 Data Structures and Algorithms (DSA) classes. Most students must attend all 100 classes.

However, a student named "Jenny" already knows DSA perfectly. If the student checking in is Jenny, she gets to skip today's classes, but the loop continues for the next student in line.

for (int class = 1; class <= 100; class++) {
    if (student_name == "Jenny") {
        continue; // Jenny skips the class
    }
    // Other students attend the class
    Attend_Class(); 
}
  • If name == "Jenny": The `continue` triggers. The `Attend_Class()` step is skipped. Loop goes to the next student.
  • If name != "Jenny": The student attends the class normally.

5. Real C Program & Memory Trace

The Goal: Ask the user to input 5 numbers and calculate their sum. However, if the user enters a negative number, the program should ignore it (skip it), NOT add it to the sum, and ask for the next number.

#include <stdio.h>

void main() {
    int i, a, sum = 0;

    for(i = 1; i <= 5; i++) {
        printf("Enter an Integer: ");
        scanf("%d", &a);

        if (a < 0) {
            continue; // Skips the sum addition below
        }

        sum = sum + a; // Statement to skip if a < 0
    }
    printf("Sum = %d", sum);
}
