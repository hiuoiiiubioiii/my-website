Topic: Passing Strings as Arguments to Functions

1 The Golden Rule of Strings

Before writing any functions, you must remember one absolute rule in C:

A string is simply a character array ending with a Null Character (\0).

Therefore, all the rules we learned about passing arrays to functions apply exactly the same way to strings. When you pass a string to a function, you are not passing a copy of the text. You are passing the Base Address (Call by Reference).

2 Example 1: The Simple Display Function

Let's write a function that takes a string from main and prints it to the screen.

void display(char str[]); // Prototype

void main() {
    char str[] = "Jenny";
    display(str);         // Function Call
}

void display(char str[]) {
    printf("String is %s", str);
}

3 Example 2: Finding Length & Modifying

Now we will pass two strings. We will calculate the length of the first string, and modify a character in the second string.

void modify(char str1[], char str2[]) {
    int i, length = 0;
    
    // Logic 1: Find length of str1 ("Jenny")
    for(i = 0; str1[i] != '\0'; i++) {
        length = length + 1;
    }
    
    // Logic 2: Modify str2 ("khatri")
    str2[1] = 'z'; 
    
    printf("%s, %s", str1, str2);
}
  • Finding Length: We loop through str1 until we hit the Null Character \0. That signals the end of the string.
  • Modifying: Because we passed by reference, str2[1] = 'z' permanently alters the string in the main function's memory.

4 Alternate Pointer Syntax

During the practical IDE coding, the instructor showed that the compiler treats arrays and pointers synonymously in function arguments.

These two function headers mean the exact same thing to the compiler:

// Syntax 1: Array Notation
void modify(char str1[], char str2[]) { ... }

// Syntax 2: Pointer Notation
void modify(char *str1, char *str2) { ... }

You can use whichever you feel more comfortable reading, but know that under the hood, C is converting the array notation into a pointer anyway.

5 ⚠️ The Segmentation Fault Bug

When typing the code in VS Code, the instructor demonstrated a critical, common error that crashes the program. If you initialize your strings in `main` like this:

char *str1 = "Jenny";
char *str2 = "khatri";
Program Crashes! (Segmentation Fault)
When you declare a string using a pointer like char *str = "Text", C stores that string literal in a Read-Only section of memory.

When our modify function tries to do str2[1] = 'z', the OS blocks it and kills the program because you aren't allowed to write to read-only memory.
The Fix: Use Array Notation
Change the initialization to: char str2[] = "khatri";
This forces the compiler to create a fully modifiable copy of the string on the Stack memory, which our function is allowed to alter safely.

📝 Lecture Cheat Sheet

  • Passing: Just pass the name of the string (e.g., display(str)). It passes the base address.
  • Receiving: Catch it with char str[] or char *str.
  • Modification: Any changes made inside the function apply directly to the original string.
  • End of String: Always loop until \0.
  • Safety: Never try to modify a string declared as char *str = "literal". Always use char str[] = "literal" if you intend to change its contents.
