Identifiers & Keywords in C

Lecture breakdown by Jayanti Khatri Lamba • Standard: ANSI C (1989)

1. What are Keywords?

Keywords are reserved words or predefined words in the C language. Their meanings are already built into the C compiler, and you absolutely cannot change them.

💡 Instructor's Analogy: Just like printf is a predefined function in the <stdio.h> library whose meaning you cannot alter, keywords are basic building blocks with fixed instructions.

Key Facts:

  • There are exactly 32 keywords in the ANSI C standard.
  • They form the sequence of instructions used to build a program.
  • Crucial Rule: All keywords must be written in exactly lowercase letters.
Examples of Keywords on the Board:
int, float, break, goto, if, for, continue, while, do, switch, auto, double...

Case Sensitivity Warning ⚠️

C is highly case-sensitive. The instructor visually demonstrates that capital letters break keywords.

int a = 5; // ✅ VALID: 'int' is lowercase
Int a = 5; // ❌ INVALID: Compiler does not recognize 'Int'

2. What are Identifiers?

Identifiers are the user-defined names given to various program elements. You use them to name your variables, functions, arrays, structures, and unions.

👤 Instructor's Analogy: To identify the instructor, you use her identifier (name): Jayanti. In code, an identifier is the name you give a piece of data so you can "identify" and use it later.

Because these are created by the user (you), the compiler needs strict rules to understand what is a valid name and what isn't.

3. The 6 Rules for Identifiers

Before writing code, you must memorize these rules for naming things in C:

  • 1. Allowed Characters: You can only use letters (a-z, A-Z), numbers (0-9), and the underscore ( _ ).
  • 2. No Special Characters: Symbols like @, $, %, -, ., spaces, or parentheses are strictly forbidden.
  • 3. Starting Character: An identifier must always begin with a letter or an underscore. It cannot start with a number.
  • 4. No Keywords: You cannot use any of the 32 reserved keywords (like int, while) as an identifier name.
  • 5. Length Limit: According to ANSI C, only the first 31 characters of an identifier are significant to the compiler.
  • 6. Case Sensitivity: Uppercase and lowercase letters are treated as completely different characters.

5. Testing Keywords & Case Sensitivity

The Keyword Rule

int float; // ❌ INVALID

You cannot use a keyword as an identifier. float is one of the 32 reserved keywords, so you cannot name an integer variable "float".

int a; // ✅ VALID

a is not a keyword. It is a perfectly valid identifier.

Are these the same?
1. Sum
2. sum
3. SUM

Answer: NO. Because C is case-sensitive, the compiler treats uppercase and lowercase letters uniquely. The compiler sees these as three completely different, separate variables.

6. Identifiers aren't just for variables!

The instructor points out that identifiers are also used to name Functions.

void Sum();

In the whiteboard example above:

  • void is a Keyword (it tells the compiler the function returns nothing).
  • Sum is the Identifier (the user-defined name of the function).
