The Code Problem

The instructor emphasizes: Never just guess the output. You must draw memory blocks and trace the variable values step-by-step (a "dry run") with pen and paper.

What will the following code print?

#include <stdio.h>

int jumble(int x, int y) 
{
    x = 2 * x + y;
    return x;
}

int main() 
{
    int x = 2, y = 5;
    
    y = jumble(y, x);  // First Call
    x = jumble(y, x);  // Second Call
    
    printf("%d\n", x);
    return 0;
}

Instructor's Core Concepts

Before we run the code, keep these rules taught in the lecture in mind:

  • Pass by Value: When main calls jumble, it only sends copies of the numbers. Modifying variables inside jumble does not alter the variables inside main directly.
  • Variable Scope: The x and y in main are entirely separate from the x and y in jumble. They live in different memory blocks.
⚠️ The Trap (Positional Argument Matching): Functions map incoming values by their position, not their name! If a function expects (int x, int y) and you call it via jumble(y, x), the first item passed gets assigned to the first parameter.

Step 1 Main Function Starts

Execution always begins at main(). The computer allocates a block of memory for main and initializes our two local variables.

Step 2 First Function Call

Code: y = jumble(y, x);

Using the current values in main, we are calling jumble(5, 2). A new memory block is created for jumble.

  • The first parameter x receives the first argument: 5.
  • The second parameter y receives the second argument: 2.
Memory: jumble() (Call 1)

y
2

Inside jumble:

x = 2 * x + y;
x = 2 * 5 + 2;
x = 10 + 2 = 12;

The local x inside jumble updates to 12. The function executes return x;, sending 12 back to main.

Step 3 Back to Main (Update Y)

The jumble() memory block from Step 2 is destroyed (erased from the whiteboard). Back in main, the returned value (12) is assigned to y.

Step 4 Second Function Call

Code: x = jumble(y, x);

Using the updated values in main, we are calling jumble(12, 2). A brand new memory block is created.

  • The first parameter x receives the first argument: 12.
  • The second parameter y receives the second argument: 2.

Inside jumble:

x = 2 * 12 + 2;
x = 24 + 2 = 26;

The local x inside jumble updates to 26. The function executes return x;, sending 26 back to main.

The second jumble() memory block is destroyed. Back in main, the returned value (26) is assigned to x.

🏁 Final Output

Code: printf("%d\n", x);

Looking at our final main() memory block, the value of x is 26.
