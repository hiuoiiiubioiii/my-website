C Programming File Handling

Understanding r+ Mode

A complete visual breakdown of reading, writing, and the file pointer. Scroll down to begin the lesson.

1. Whiteboard Concept: How r+ Compares

Before diving into code, the instructor compares r+ mode to the basic modes we already know to establish a baseline.

  • r (Read Mode): Used only to read. If the file doesn't exist, it returns NULL (error).
  • w (Write Mode): Used to write. If the file doesn't exist, it creates a new one. If it does exist, it deletes the old content entirely.

The r+ Mode (Read + Write):
The file is opened for BOTH reading and writing. However, its primary purpose is reading. Therefore, if the file does NOT exist, it will NOT create a new file. It will simply return NULL.

2. Safely Opening a File

Because r+ will not create a missing file, you must always check for a NULL pointer to prevent your program from crashing.

FILE *fp = NULL;
fp = fopen("abc.txt", "r+");

if(fp == NULL) {
    printf("error");
    exit(1);
}

If abc.txt exists, the file pointer fp is successfully created and points to the very beginning (index 0) of the file.

3. Whiteboard Visualization: Writing with r+

The true power of r+ mode is modification. It doesn't append to the end, and it doesn't wipe the file clean. It writes over existing characters one-by-one, exactly where the pointer is sitting.

Action 1: Overwriting a single character

Let's use fputc('Z', fp); right after opening the file.

Hi, this is awesome
➡️
Zi, this is awesome

The pointer started at 'H', overwrote it with 'Z', and automatically moved forward one space to rest on the 'i'.

Action 2: Overwriting a string

Now, let's execute fputs("jenny", fp);.

This is awesome
➡️
jennyis awesome

The word "jenny" has 5 characters. It overwrote the first 5 characters (T, h, i, s, [space]). The rest of the file remains completely untouched!

5. The Golden Rules & Assignment

If you get an exam or interview question about r+ mode, remember this checklist:

  • ✅ Dual Functionality: Opens an existing file for BOTH reading and writing.
  • ✅ No File Creation: If the file is missing, it returns NULL. It will never create a blank file for you.
  • ✅ Cursor at the Start: It does not erase the file upon opening. The cursor starts at the very beginning.
  • ✅ 1-for-1 Overwrite: Any writing overwrites existing characters one-for-one. It leaves the remainder of the file intact.
