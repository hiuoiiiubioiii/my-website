Pointers in C

Practice Problems, Arithmetic & Memory Visualizations

1. Constant Variables & Constant Pointers

Let's look at how the const keyword interacts with variables and pointers. First, let's visualize a standard integer and pointer assignment.

void main() {
    int a = -11;
    int *p = &a;
    *p = 10;
    printf("%d", a); // Prints 10
}
Pointer (p)
1000
addr: 100
⟶
Variable (a)
-11
→ 10
addr: 1000

The indirection operator *p accesses the memory location 1000 and overwrites -11 with 10.

Introducing the const Keyword

Now, watch what happens when we modify the code to make a a constant variable.

void main() {
    const int a = -11;
    const int *p = &a; // Pointer must match data type
    *p = 10; // ⚠️ Error occurs here
    printf("%d", a);
}
Compilation Error: "assignment of read-only location" Because a is declared with const, its memory location becomes strictly read-only. Attempting to modify that location using the pointer (*p = 10;) is illegal and will cause the program to crash upon compilation. You can only read the value, not change it.

2. Pointer Arithmetic with Strings

This problem tests how pointers navigate through a character array (string) using increment and decrement operators combined with ASCII math.

void main() {
    char str[] = "Welcome to Jenny's lectures";
    char *ptr = str;

    printf("%c\n", *ptr);
    printf("%c\n", *(ptr++ + 1));
    printf("%c\n", *((ptr-- + 5) - 1) + 3);
    printf("%c\n", *(++ptr + 10) - 32);
    printf("%c %c %c\n", *ptr, *++ptr, *--ptr);
}

Visualizing the String in Memory

Because it's a char array, each element takes exactly 1 byte. We will assume the base address (the address of the first character 'W') starts at 100.

char *ptr = str; assigns the base address (100) to ptr.

Step-by-Step Breakdown

printf("%c\n", *ptr);
Action: Dereference the current value of ptr.
Math: *100 → The value at address 100.
State: ptr remains at 100.
Output: W
printf("%c\n", *(ptr++ + 1));
Rule: Parentheses evaluate first. Contains a post-increment (ptr++). Use the current value in the expression first, then increment the pointer.
Math: 1. Use current ptr (100): *(100 + 1) → *101
2. Increment ptr: ptr becomes 101
3. Dereference: Value at 101 is 'e'.
State: ptr is now 101.
Output: e
printf("%c\n", *((ptr-- + 5) - 1) + 3);
Rule: Contains a post-decrement (ptr--). Use the current value first, then decrement. Afterwards, perform ASCII addition.
Math: 1. Use current ptr (101): (101 + 5) - 1 → 106 - 1 → 105
2. Decrement ptr: ptr becomes 100
3. Dereference: *105 is 'm'.
4. ASCII Addition: 'm' + 3. Moving 3 letters forward in the alphabet ('m' → 'n' → 'o' → 'p').
State: ptr is back to 100.
Output: p
printf("%c\n", *(++ptr + 10) - 32);
Rule: Contains a pre-increment (++ptr). Increment the pointer first, then use the new value in the expression.
Math: 1. Increment ptr: ptr goes from 100 to 101.
2. Use new ptr (101): (101 + 10) → 111.
3. Dereference: *111 is 'j' (from "Jenny's").
4. ASCII Subtraction: 'j' - 32. Subtracting 32 from a lowercase letter converts it to Uppercase.
State: ptr is now 101.
Output: J
printf("%c %c %c\n", *ptr, *++ptr, *--ptr);
⚠️ Crucial Rule for Multiple Expressions When a single printf contains multiple expressions separated by commas, they are evaluated from Right to Left, but printed from Left to Right.
Evaluation: (From Right to Left)
1. Rightmost (*--ptr): Pre-decrement ptr (101 → 100). Dereference *100 → Value is 'W'.
2. Middle (*++ptr): Pre-increment ptr (100 → 101). Dereference *101 → Value is 'e'.
3. Leftmost (*ptr): Use current ptr (101). Dereference *101 → Value is 'e'.
Printing: Print the evaluated values from Left to Right.
Output: e e W
