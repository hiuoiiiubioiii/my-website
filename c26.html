Unformatted Output Functions: putchar(), putch(), and puts()

1. Introduction & Context

In the previous lesson, we learned about Unformatted Input Functions (getch(), getche(), getchar(), gets()). Now, we are looking at how to output that data.

What does "Unformatted" mean?
Unlike printf(), where you can format text using format specifiers like %d, %c, or %s, unformatted functions only output the raw data. You cannot mix them with custom strings like "The character is = ".

Whiteboard Overview (High-Level)
  • putchar() → Prints a Single Character
  • putch() → Prints a Single Character
  • puts() → Prints a String (Complete lines)

Note: putc() and getc() exist but are used for file/stream handling, which is a separate topic.

2. Deep Dive: putchar()

The putchar() function is used to print exactly one character to the screen. It is defined inside the <stdio.h> header file.

The Transition from printf()

Using Formatted (printf):

char ch = 'J';
printf("Character is = %c", ch);

Using Unformatted (putchar):

char ch = 'J';
putchar(ch); // No extra text allowed

Visualizing the Output

If we run the unformatted program where the user types 'J', the screen looks like this:

Instructor's Tricks:
You can print hardcoded characters without variables by using single quotes.
• putchar('\n'); prints a new line.
• putchar('K'); prints the letter K.

3. Deep Dive: putch()

The putch() function does the exact same job as putchar(). It outputs a single character to the screen.

So, what is the difference?
The difference is entirely in where the function comes from:

  • putchar() belongs to <stdio.h> (Standard Input/Output)
  • putch() belongs to <conio.h> (Console Input/Output)

Just like putchar, you can write putch(ch);, putch('\n');, or putch('K');, and the output screen will behave exactly the same way.

4. Deep Dive: puts()

Because putchar and putch can only handle one character at a time (unless you manually put them inside a for loop), C provides puts() to handle whole strings (character arrays). It lives in <stdio.h>.

Syntax & Usage

First, we declare a character array and take unformatted input using gets():

char ch[20];
puts("Enter a string:"); // Hardcoding a string
gets(ch);                // Taking input
puts(ch);                // Printing the string variable

⭐ The Most Important Feature of puts():
Did you notice the cursor on the screen above moved to a new line automatically? puts() has a built-in newline operator. When you use puts(ch);, it prints the string and automatically moves the cursor to the next line. You do NOT need to add \n.

5. Instructor Assignments & Trick Question

Assignment 1 (Conceptual)

"Let me know in the comments: What do you mean by formatted input/output vs unformatted input/output?"

Study Guide Answer: Formatted functions (printf/scanf) allow developers to combine variables and text freely using format specifiers (%d, %s). Unformatted functions (puts, putchar, gets) only deal directly with raw memory arrays or characters, with no formatting allowed.

Assignment 2 (The Trick Question)

The instructor writes this code on the board and asks what the output will be:

puts('A');

Study Guide Breakdown:
Notice she used single quotes ('A'). Single quotes denote a character. However, puts() expects a string pointer (which uses double quotes "A").

If you run this in a real compiler, it will cause an error or crash! The program will try to read the integer ASCII value of 'A' (which is 65) as a memory address, leading to a segmentation fault. To fix it, you must write puts("A"); or use putchar('A');.
