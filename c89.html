Lecture Breakdown

Classification of Functions:
No Argument, With Return Type

In this session, we transition from strictly void functions to functions that calculate a value and send it back to main().

1. Concept & Syntax

The Anatomy of the Function

Because the function takes no inputs from `main()`, we put void inside the parentheses. Because the function calculates something and sends a value back to `main()`, it must have a data type before its name.

Syntax Breakdown

int fun (void)

Rule of thumb: The data type on the left (`int`, `float`, `char`) dictates the type of data the `return` statement is allowed to spit out.

2. The Complete Program

Building the sum() Function

Let's build a program to calculate the sum of two numbers to demonstrate this concept. Pay close attention to how `main()` catches the returned value.

/* Step 1: Function Declaration */
int sum(void);

void main() {
    // We need a variable to "catch" the returned value
    int s; 
    
    // Jump to sum(), get the result, store in 's'
    s = sum(); 
    
    printf("sum = %d", s);
}

/* Step 3: Function Definition */
int sum() {
    int a, b, sum = 0;
    
    printf("Enter a & b: ");
    scanf("%d %d", &a, &b);
    
    sum = a + b;
    
    // Send the answer back to main()
    return sum;
}
Why ask for input inside the function?

Because the function has no arguments! main() is not sending it any numbers. Therefore, the sum() function must ask the user for the numbers internally using scanf.

3. Memory & Execution Flow

How Variables Scoping Works

A crucial concept taught on the whiteboard: The variables a, b, and sum only exist inside the function block. Once the function hits return, those variables are destroyed!

You cannot print the variable sum directly inside main(). This is why main() creates its own variable s to "catch" the answer.

4. Whiteboard Deep Dive

Two Crucial 'Return' Rules

1. Unreachable Code

What happens if you write code after the return statement?

return sum;
// Everything below is DEAD code
printf("Inside sum function");

As soon as the compiler executes a return statement, it immediately exits the function and goes back to main(). The printf line is unreachable and will never execute.

2. The Comma Operator

What happens if you try to return multiple variables?

return ( sum, a, b );

The compiler evaluates from left to right. It looks at sum, discards it. Looks at a, discards it. Looks at b, and returns it. It always returns the right-most value.

5. VS Code Execution

Practical Demonstrations

During the practical segment, the instructor verified the rules by running different scenarios:

  • Proving Unreachable Code: She typed printf("hi"); immediately after the return statement. Output was just the sum. "hi" never printed.
  • Proving Comma Operator: She ran return sum, a, b; with inputs 5 and 7. The output printed was 7 (the value of 'b').
  • Proving the 'Void' Rule: She tried to call the function with arguments: sum(4, 6);. The compiler threw a strict error: "Too many arguments in function call" because the declaration strictly stated void.
  • Different Data Types: She successfully changed the return type to float sum(void) (outputting 12.000000) and char sum(void) (outputting the letter 'a').
