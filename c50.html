C Arrays: Sum & Average

Scroll down to review the complete lesson, whiteboard notes, and dry runs.

1. Problem Statement & Setup

Goal: Write a program (WAP) to read marks of 5 students, store them in an array, and calculate the sum and average.

We start inside the main() function by declaring the necessary variables:

void main() {
    int marks[5];
    int sum = 0, avg;
    int i;
}
⚠️ Instructor's Warning - The "Garbage Value" Trap:
You must initialize sum = 0;. If you don't, C assigns a random "garbage value" to the memory block. When you try to add the students' marks to it, your final math will be completely ruined.

We also declare an array marks[5] to hold our 5 integers, and an iterator i for our upcoming loops.

Instructor Notes:
• The array name marks points to the Base Address (100).
• Because an integer takes 4 bytes of memory, the addresses jump by 4 (100 → 104 → 108...).
• The formula used behind the scenes is: Base Address + (i * size_of(int))

3. Reading Input from User

We need to fill the 5 blocks of memory shown above. Instead of writing scanf 5 times, we use a for loop.

for (i = 0; i < 5; i++) {
    scanf("%d", &marks[i]);
}
Crucial Syntax: Notice the ampersand & in &marks[i]. This tells the computer the exact memory address (e.g., 100, 104) where the typed number should be stored.

4. Calculating the Sum (Whiteboard Dry Run)

Now that the array is full, we use a second loop to pull out each number and add it to our sum variable.

for (i = 0; i < 5; i++) {
    sum = sum + marks[i];
}

The instructor performs a detailed "dry run" (tracing the loop manually). Let's visualize how the sum variable changes in memory step-by-step using the numbers from her example (10, 20, 15, 25, 7):

Once i becomes 5, the condition i < 5 is false, and the loop breaks. Our final sum is 77.

5. Calculating Average & "The Float Trap"

To get the average, we divide the total sum by the number of students (5).

avg = sum / 5;

The Whiteboard Discussion:
If we leave sum and avg as int data types, 77 / 5 will result in exactly 15. Integers chop off (truncate) the decimal values.

To get an accurate average (like 15.4), the instructor explains that it is better to declare our math variables as float. This requires us to use the %f format specifier when printing.

6. Live Coding Session (VS Code)

The instructor moves to her laptop to write the final, robust version of the code, implementing the float fix.

*Note: During the recording, she gets a compilation error because she initially forgot to declare int i; inside the loop. She quickly fixes this at the top of the function. Here is the final working code she writes:

#include <stdio.h>

void main() {
    int i;
    int marks[5];
    
    // Changing to float to prevent decimal truncation!
    float sum = 0, avg; 

    printf("Enter marks of 5 students:\n");
    
    // 1. Read values into array
    for(i = 0; i < 5; i++) {
        // Using %f because we are allowing floating-point marks now
        scanf("%f", &marks[i]); 
    }

    // 2. Calculate Sum
    for(i = 0; i < 5; i++) {
        sum = sum + marks[i];
    }

    // 3. Calculate Average
    avg = sum / 5;

    // 4. Print Outputs (using %f for floats)
    printf("sum = %f\n", sum);
    printf("average = %f\n", avg);
}
