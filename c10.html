1. Why Do We Need Data Types?

To understand data types, think about real-world storage:

  • 👕 The Wardrobe Analogy: You have different spaces for different things. Large shelves are for clothes, while small locked drawers are for cash or jewelry. You wouldn't put a single ring in a massive clothing shelf, nor stuff clothes into a tiny drawer.
  • 🥛 The Kitchen Analogy: You use a glass to drink water and a bowl to eat vegetables. The container matches the contents.

Application to Computers:
When you write a program, you ask the computer for memory space. Just like the analogies above, the computer needs to know:

  • What type of data are you storing? (A number? A letter? A decimal?)
  • How much space do you need?

Definition: A Data Type defines the type of data a variable will hold and dictates exactly how much storage (memory) should be allocated to it.

2. How Memory Allocation Works

Let's visualize what happens inside the computer when you write a simple line of C code:

int a = 12;
Address: 1000
a
12
Type: int (Integer)

What the compiler does step-by-step:

  1. It sees int and allocates a specific block of memory.
  2. It assigns the name a to that block so you can call it easily.
  3. It assigns a background memory address (e.g., 1000).
  4. It stores the value 12 inside that block.

4A. Deep Dive: Integers (int)

Used to store whole numbers without decimals.

  • Format Specifier: %d
  • Memory Size: Depends on your architecture (2 Bytes for 16-bit machines, 4 Bytes for 32/64-bit machines).

Modifiers

You can modify integers to optimize memory and functionality:

  • Size Modifiers: short int (less memory) and long int (more memory).
  • Sign Modifiers: signed (allows positive & negative) and unsigned (allows only positive numbers, starting from 0).

Ranges on a 16-bit Machine (2 Bytes)

  • Signed int (Default): -32,768 to +32,767
  • Unsigned int: 0 to 65,535

Memory Math Note: 1 Byte = 8 bits. Therefore, a 2-byte integer uses 16 bits of memory. A 2GB RAM can store 2 * 1024 * 1024 * 1024 * 8 bits!

4B. Characters (char)

Used to store a single character.

  • Memory Size: Always 1 Byte (8 bits).
  • Format Specifier: %c
  • Ranges: signed char (-128 to 127), unsigned char (0 to 255).

Crucial Concept: ASCII Values

Characters are actually stored as numbers (ASCII codes) in the computer's background. Look at this code trick:

printf("%c", 98); // Outputs: b

Why? Even though 98 is a number, because you used the %c (character) format specifier, the computer prints b, because 98 is the standard ASCII code for lowercase 'b'.

4C. Floats, Doubles & Void

Floating Point Numbers

Used to store real numbers featuring decimal points (e.g., 10.1, 3.14). There are three levels of precision:

  • float: 4 Bytes | Format: %f | Precision: Up to 6 decimal places.
    (e.g., 10.1 prints as 10.100000)
  • double: 8 Bytes | Format: %lf | Precision: Up to 14 decimal places. Used for higher math accuracy.
  • long double: 10 Bytes | Format: %Lf | Used for massive decimal calculations.

Void (void)

Means "Empty" or "Nothing". You cannot create a variable with void (e.g., void a; will crash). It is used primarily with functions to tell the compiler: "This function will not return any value."
