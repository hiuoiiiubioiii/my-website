Function with Argument, Without Return Type

This is the 3rd classification of user-defined functions in C programming. We will learn how to send data into a function, process it, and print the result directly inside the function without returning anything back to the caller.

1. General Syntax & Blueprint

Before writing a full program, you must understand the three pillars of a function: Declaration, Definition, and Calling.

1. Function Declaration (The Prototype)

Tells the compiler the function exists, what it takes, and what it returns. Must be placed before main().

void fun(int, int); // Declaration
  • void: The function returns nothing.
  • (int, int): The function expects two arguments. Note: You only need to specify the data types here, not the variable names!

2. Function Definition (The Logic)

This is where the actual code of the function lives.

void fun(int x, int y) // x and y are "Formal Parameters"
{
    // logic goes here
}
  • The return type and argument types must exactly match the declaration.
  • Here, you must provide variable names (like x and y) so you can use them inside the logic.

3. Function Calling

This is how you trigger the function from inside main().

fun(5, 7); // 5 and 7 are "Actual Parameters (Arguments)"
// OR
fun(a, b);
Crucial Rule: Do not write data types in the function call! Never write fun(int a, int b);. Only pass the values or the variable names.

2. The "Sum" Example (Whiteboard Code)

Let's write a complete C program that takes two float numbers from the user, passes them to a function, and prints the sum.

#include <stdio.h>

// 1. Declaration
void sum(float, float); 

void main() 
{
    float a, b;
    
    printf("Enter a & b: ");
    scanf("%f %f", &a, &b);
    
    // 2. Function Call (Passing Actual Parameters)
    sum(a, b); 
}

// 3. Function Definition (Receiving Formal Parameters)
void sum(float x, float y) 
{
    float s = 0; // Local variable
    s = x + y;   // Processing
    printf("Sum = %f", s); // Printing here because return type is void
}

🏠 The "Neighbor's House" Analogy (Understanding Scope)

Variables a and b belong exclusively to the main() function's house. Variables x and y belong exclusively to the sum() function's house.

If you wanted, you could name the variables inside sum a and b as well. It won't cause an error because they live in different houses. It's like two neighbors both having a child named Rohan; they are completely different kids in different houses.

4. VS Code Practical Execution Rules

During the laptop demonstration, Jenny showed what happens if you break the rules of arguments. Here are the compiler errors you need to know:

  • Static Input Works: You can skip scanf and just type sum(3.1, 4.1);. The program calculates perfectly because actual parameters can be fixed numbers.
  • Error 1: Too Many Arguments: If you write sum(x, y, 8.1);, the compiler throws an error:
    error: too many arguments to function 'sum'. The definition only has two slots!
  • Error 2: Too Few Arguments: If you write sum(x);, the compiler throws an error:
    error: too few arguments to function 'sum'. You left a parameter empty!
The Golden Rule: The number of arguments passed in the function call must exactly match the number of arguments established in the declaration and definition.
