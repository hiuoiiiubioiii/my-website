1. Documentation Section

Status: Optional

This is the very first section, also known as the comment section. It is used to provide information about the program to anyone reading the code in the future.

// Author: Jayanti Khatri // Date: 18 Dec 2020 // Program for addition of two numbers /* Program for addition of two numbers. */
👩‍🏫 Instructor Explanation:

The compiler completely ignores this section. It does not convert these lines into object code. It is strictly for human understanding.

  • // is used for single-line comments.
  • /* ... */ is used for multi-line comments.

Even though it is optional, she highly recommends making it a habit so people (including your future self) understand what the code is for.

2. Link Section

Status: Optional (but usually required in practice)

This section links your program to Header Files (libraries) that contain predefined functions.

#include <stdio.h> #include <conio.h> #include <math.h> #include <string.h>
👩‍🏫 Instructor Explanation:

The compiler doesn't have a "brain" of its own. If you type printf to output text, the compiler will say "What does this mean?".

  • # is a Preprocessor Directive.
  • stdio.h (Standard Input Output): Contains the logic for printf() and scanf().
  • conio.h (Console Input Output): Contains logic for functions like getch() (which holds the output screen).
  • math.h / string.h: Used if you need math formulas (like sqrt, pow) or string logic (like strlen).

3. Definition Section

Status: Optional

This section is used to define Symbolic Constants or Macros.

#define PI 3.14159 #define MAX 100
👩‍🏫 Instructor Explanation:

Imagine you have a program with 1,000 lines of code, and you use the maximum student value of 100 in twenty different places. If the school expands to 200 students, finding and changing every '100' to '200' is a nightmare.

Instead, use #define MAX 100. When you compile, the compiler automatically replaces the word MAX with 100 everywhere. If you ever need to update it, you only change it once right here at the top.

4. Global Declaration Section

Status: Optional

If you need a variable or a custom function to be accessed by every part of your program, you declare it here.

// Global variable declaration int a = 50; // User-defined function prototypes void display(); void sum();
👩‍🏫 Local vs. Global Variables:
  • Local: If you declare int a; inside main(), only main() can use it. Other functions can't see it.
  • Global: Declared here (outside of all functions). Now, main(), display(), and sum() can all share and modify a.

5. Main Section

Status: Compulsory

This is the heart of the C program. Every C program must have exactly one main function. This is where execution begins.

void main() { // 1. Declaration Part int sum; // 2. Executable Part sum = 10 + 20; printf("Sum is %d", sum); }
👩‍🏫 Instructor Explanation:

When you hit "Run", the control goes directly to the very first opening brace { of the main() function.

It consists of two parts:

  1. Declaration Part: All variables you plan to use inside main must be declared first.
  2. Executable Part: The actual logic (math, calling other functions, printing outputs). There must be at least one executable statement.

6. Subprogram Section

Status: Optional

This section contains the actual code (definitions) for the user-defined functions you named earlier.

void display() { printf("Jenny's Lectures"); }
👩‍🏫 Instructor Explanation:

These functions perform specific tasks. They don't run on their own; they only run when the main() function specifically "calls" their name.

7. Putting It All Together & Execution Flow

Here is the final code the instructor writes on the board, demonstrating how all the sections stack together to create a working program.

// 1. Documentation // Author: Jayanti Khatri // 2. Link #include <stdio.h> #include <conio.h> // 3. Definition #define MAX 100 // 4. Global Declaration int a = 50; void display(); // 5. Main void main() { printf("Hello\n"); display(); getch(); } // 6. Subprogram void display() { printf("Jenny's Lectures"); }

How the Computer Reads This (Execution Flow)

1. Linking: Compiler loads stdio.h & conio.h so it understands the code. It replaces any MAX with 100.
2. Starts at Main: Execution jumps directly into void main().
3. Print Hello: The computer executes printf("Hello"). "Hello" appears on the screen.
4. Function Call: The computer sees display();. It stops what it is doing in main, and jumps down to the subprogram section.
5. Executes Subprogram: Inside display(), it reads printf("Jenny's Lectures"). Prints it to screen.
6. Returns to Main: Subprogram is done. Control jumps back up to main to execute getch();, holding the screen so the user can see the final output.
