2D Arrays in C

Continuous Study Feed • Conceptual Breakdowns • Visual Memory Models

1. The Need for 2D Arrays (The "Why")

Instructor Audio

Before understanding 2D arrays, let's look at 1D arrays to establish a baseline. If you want to store the total marks for 60 students, rather than creating 60 individual variables, you create a single 1D array.

On Whiteboard 
int marks[60]; // Stores 60 items in a single row

Mental Model of 1D Array:

0
1
2
...
58
59

The Problem: What if you want to store the marks of 60 students, but across 5 different subjects? Using 1D arrays, you have to write tedious code:

int s1[60]; // Subject 1
int s2[60]; // Subject 2
int s3[60]; // Subject 3
int s4[60]; // Subject 4
int s5[60]; // Subject 5

The Solution: To avoid declaring multiple arrays, we store all this data under a single variable name using a 2-Dimensional Array.

int marks[5][60]; // 5 subjects, 60 students each

Note on Scaling: If a university has 5 departments, 5 subjects, and 60 students, you just add another dimension to make a 3D array: int marks[5][5][60];

2. Declaration and Syntax (The "How")

On Whiteboard

The general syntax for declaring a 2D array requires two sets of brackets:

Datatype name_of_Array [row_size] [col_size];
Instructor Audio

The first subscript [] always represents the number of Rows. The second subscript [] always represents the number of Columns.

Practice Scenario: Declare an array to store the marks of 5 students across 10 subjects.

int marks[5][10];
Crucial Rule

When simply declaring a 2D array, you must provide the size inside both brackets. If left empty, the compiler throws an error. (Exception: The row bracket can be empty if you initialize the array at the exact time of declaration, which is covered in the next lesson).

4. Reality: How Memory Actually Works

Important Conceptual Shift

While we visualize a 2D array as a nice matrix, computer memory is not a table. RAM is a linear sequence of bytes (like a long 1D tape). Therefore, a 2D array must be "flattened" out to be stored.

On Whiteboard

The computer stores the data row by row sequentially. This is known in computer science as Row-Major Order.

5. Calculating Size and Memory

How much space is your 2D array taking up inside the computer? Let's use our int marks[3][5]; example.

1. Total Number of Elements
Formula: Row Size × Column Size
Example: 3 × 5 = 15 elements
2. Total Memory (Bytes) Allocated
Formula: Total Elements × Size of Datatype
(Assuming a standard architecture where an 'int' = 4 bytes)
Example: 15 elements × 4 bytes = 60 bytes

Instructor Note: Because we only declared the array and didn't assign data to it yet, those 60 allocated bytes currently hold "garbage values".
