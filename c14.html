Deep Dive: The Modulo Operator (%)

While basic math is straightforward, modulo confuses many beginners. Modulo always returns the remainder of a division.

Example 1: a = 5, b = 4

Equation: a % b ➔ 5 % 4

1 (Quotient)
4
5

➔ 4 goes into 5 one time.

➔ Remainder is 1.

Result = 1

Example 2: 4 % 4

➔ 4 goes into 4 exactly one time. Remainder is 0.

Result = 0

Code Dry Run: Integer vs Float

Let's look at exactly how these output in a C program, specifically focusing on division and modulo.

#include <stdio.h>
#include <conio.h>

void main() {
    int a = 10, b = 7;
    clrscr();
    
    printf("a+b = %d", a+b);   // Output: 17
    printf("a-b = %d", a-b);   // Output: 3
    printf("a*b = %d", a*b);   // Output: 70
    
    // 🚨 PAY ATTENTION TO DIVISION & MODULO 🚨
    printf("a/b = %d", a/b);   // Output: 1 (Integer Division!)
    printf("a%%b = %d", a%b);  // Output: 3 (Remainder of 10/7)
    
    getch();
}
💡 Why is 10 / 7 = 1? Because a and b are declared as int, the fractional part is completely truncated (ignored). It does NOT output 1.428. If you want decimals, you must declare them as float and use the %f format specifier!

The 2 Strict Rules of Modulo (%)

The instructor highlights two crucial rules for exams and coding interviews regarding the modulo operator.

❌ Rule 1: No Floating Points Allowed!

You cannot use the modulo operator on floating-point numbers. It will generate an "illegal use of floating point" compiler error. Modulo only works with integers.

✅ Rule 2: The Sign Rule

When dealing with negative numbers, what is the sign of the remainder? The result always takes the sign of the first operand (the Left-Hand Side).

The Sign Rule Proof:

-10 % 7 = -3
(Takes the negative sign from 10)


+10 % -7 = +3
(Takes the positive sign from 10. The -7 is ignored)


-7 % 10 = -7
(Takes the negative sign from 7)

Precedence & Associativity (Arithmetic)

What happens if you have an equation with multiple operators? C evaluates based on Precedence (Priority) and Associativity (Direction).

🏆 Priority 1 (Highest): * , / , %

🥈 Priority 2 (Lowest): + , -

Associativity (Tie-Breaker): Left to Right ( L ➔ R )

Evaluation Example: a + b - c * d

  1. First, c * d is evaluated (Priority 1).
  2. Next, a + b is evaluated (Priority 2, but it's the furthest Left).
  3. Finally, the subtraction - occurs.

2. Assignment Operators (=)

The assignment operator calculates the value on the Right-Hand Side (RHS) and assigns it to the Left-Hand Side (LHS).

⚠️ The Golden Rule: The Left-Hand Side must always be a single variable. You cannot have an expression or constant on the left.

a + b = 0; ➔ ILLEGAL ❌

(Throws an error because a+b is an expression, not a variable)

a = a + b; ➔ LEGAL ✅

(Evaluates RHS a+b, assigns result to LHS a)

Multiple Assignment & Associativity

Unlike arithmetic operators (Left ➔ Right), Assignment operators are evaluated from Right ➔ Left.

Code:
a
←
b
←
c
←
d
←
10

Computer reads: Assign 10 to d. Then d to c. Then c to b. Then b to a.

Shorthand (Compound) Operators

To save time and write cleaner code, C allows you to combine an arithmetic operator with the assignment operator.

Syntax: variable operator= expression;

Instead of a = a + 1; ➔ write a += 1;

Instead of a = a - 2; ➔ write a -= 2;

Instead of a = a % 10; ➔ write a %= 10;

When CAN'T you use shorthand?

You can only use shorthand if the variable on the Left-Hand Side is the exact same as the first variable on the Right-Hand Side.

  • a = b + (c * 10); ➔ Cannot use shorthand (a does not match b).
  • a = a + (c * 10); ➔ Can use shorthand: a += c * 10;
