Context: GATE 2010 Call by Reference

In this session, the instructor breaks down a notoriously tricky question from the GATE 2010 exam. This problem tests your understanding of global variables, pointers, and exactly how call-by-reference modifies memory.

At the start of the video, she notes that in a community poll, 48% of students answered this incorrectly because they tried to solve it in their heads instead of drawing a memory map.

The Problem

Examine the Code

Here is the exact code written on the whiteboard. What do you think the output will be?

#include <stdio.h>

void f(int *p, int *q) {
    p = q;
    *p = 2;
}

int i = 0, j = 1;

int main() {
    f(&i, &j);
    printf("%d %d\n", i, j);
    getchar();
    return 0;
}
The Trap: Almost half of the students guessed the output would be 2 1. Let's draw the memory map to see why they were wrong.
Step 1

Identifying Global Variables

Audio Breakdown: The instructor points to the line int i = 0, j = 1;. She asks, "Are they within f? No. Within main? No. They are outside. It means these are global variables."

Global variables have memory allocated that lasts throughout the entire program and can be accessed by any function.

Step 2 & 3

The Function Call: f(&i, &j)

Execution begins in the main() stack frame. The very first line calls function f(), passing the addresses of i and j (100 and 200 respectively).

Inside the stack frame for function f, two pointer variables p and q are created to catch these addresses.

Step 4

The Tricky Part: p = q;

We are inside function f. The code says p = q;. The instructor explains: "Whatever is the value in q, that would be assigned to p."

The value inside q is the address 200. This overwrites the 100 inside p. Now, both pointers are pointing to j! The arrow pointing to i is erased.

Step 5

Dereferencing: *p = 2;

The next line is *p = 2;. The * is the indirection (dereference) operator. It means: "Go to the address currently stored in p, and change its value to 2."

Because p was just reassigned to address 200, we follow the new path. It points to the global variable j. We cross out the 1 and write 2.

Student Takeaway: Because `p` was reassigned in the previous step, this line modifies global `j`, NOT global `i`. This is why the 48% of students who guessed "2 1" were wrong—they missed the pointer reassignment!

Step 6

Returning to main() & Output

Function f finishes execution and gets popped off the stack. Control returns to main() to execute the print statement: printf("%d %d\n", i, j);

Looking at our final global memory state, the value of i was completely untouched (it remains 0). The value of j was modified to 2.

Output: 0 2

The Instructor's Core Advice

"Don't just see the question and try to find the answer in your brain... draw a visual representation."

Exam questions like this in GATE are designed to be tricky, not necessarily mathematically hard. You must trace every single line of code meticulously using a pen and paper memory map to avoid falling into simple traps.
