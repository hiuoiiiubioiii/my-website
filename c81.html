Null Pointers in C

Complete Class Notes & Visualizations based on the Lecture

1. The Problem: Uninitialized "Wild" Pointers

Before understanding Null Pointers, we must understand the problem they solve. When you declare a pointer but don't initialize it, it doesn't sit empty. It holds a garbage value.

Because a pointer is designed to hold a memory address, the system treats this garbage value as a random address in your RAM.

int *ptr; // Uninitialized Pointer (Wild Pointer)
Pointer (ptr) -858993460
➔
Random RAM ???

⚠️ Danger of Undefined Behavior

If you try to read or write to this random address (dereference it), your program might corrupt OS data, overwrite another program's memory, or crash unpredictably. This is known as Undefined Behavior.

2. The Solution: The Null Pointer

To fix the dangers of Wild Pointers, best practice dictates that if you don't have an address to assign to a pointer immediately, you should initialize it with NULL.

  • What is it? NULL is a special macro defined in headers like <stdio.h> and <stdlib.h>.
  • What is its value? Its integer value is exactly 0.
  • What does it point to? It points to the 0th memory location. The OS reserves this location, meaning it is guaranteed to be an invalid/inaccessible address.
int *ptr = NULL; // Safe, best practice
int *p = 0;      // Technically identical, but NULL is preferred for readability
Pointer (ptr) NULL (0)
➔
🚫 Address 0
(OS Reserved)
The Golden Rule: You CANNOT dereference a Null Pointer. If you write *ptr when ptr is NULL, your program will instantly crash (Segmentation Fault). This is actually a good thing! A guaranteed crash immediately tells you where the bug is, unlike the silent data corruption of a Wild Pointer.

3. Why use them? (Error Handling)

The primary use of Null Pointers is for Error Handling, particularly during Dynamic Memory Allocation (using malloc or calloc).

When you ask the OS for memory at runtime, it might fail (e.g., out of RAM). If malloc fails, it returns NULL.

int *ptr;
// Requesting memory for 5 integers at runtime
ptr = (int*) malloc(5 * sizeof(int));
If Success:
1000
OR
If Fail:
NULL (0)

The Standard C "Null Check" Pattern

Because malloc might return NULL, you must always check the pointer before using it.

if (ptr == NULL) {
    printf("Memory allocation failed!");
    // Handle error: exit program or return
} else {
    // It is safe to use the pointer
    printf("%d", *ptr);
}

4. Comparing Pointers

You can use standard comparison operators (==) on pointers. Understanding how this behaves with Null vs Wild pointers is a common interview question.

Scenario A: Comparing Two Null Pointers

int *ptr1 = NULL;
int *ptr2 = NULL;

if (ptr1 == ptr2) { ... } // ALWAYS TRUE

Since both hold the exact integer value 0, they are guaranteed to be equal.

Scenario B: Comparing Two Wild Pointers

int *ptr1;
int *ptr2;

if (ptr1 == ptr2) { ... } // UNDEFINED / FALSE

Because both pointers grab random garbage data left over in RAM, it is almost mathematically impossible for them to match. This comparison is entirely useless.
