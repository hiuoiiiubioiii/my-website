📝 Question 1: Memory Allocation

The instructor writes the following code on the right side of the whiteboard. The goal is to determine the output of the sizeof() function.

#include <stdio.h>

enum Xenum { C, CPP, Java };

main() {
    enum Xenum var;
    printf("%d", sizeof(var)); 
    return 0;
}
🎙️ Instructor's Explanation

An enum variable can only hold one value at a time from its list of constants. By default, the C compiler treats these enum constants as integers, assigning them sequential values starting from 0.

🧠 Visualization: Under the Hood

Here is how the compiler visually maps the constants to integers in memory:

C 0
CPP 1
Java 2

Because enums are represented as integers under the hood, the size of an enum variable is equal to the size of an standard integer (int) on that specific computer architecture.

The Final Output: The instructor notes we are on a 64-bit machine. On a standard 64-bit architecture, the size of an int is 4 bytes. Therefore, the output printed to the screen is 4.

⚠️ The Scope Trap (Modified Question 1)

The instructor modifies the code to test your knowledge of Scope. She adds a second enum directly below the first one.

enum Xenum { C, CPP, Java };
// Added to the board:
enum Yenum { C, CPP, Java, Xenum };
🚨 Compiler Error!

If you write this, the program will crash. Why? In C, enum constants (the elements inside the curly braces) share the exact same scope as the enum declaration itself.

Because C, CPP, and Java were already defined in Xenum, defining them again in Yenum creates duplicate variables in the same scope, which is illegal in C.

Important Nuance: Notice that she included Xenum inside the brackets of Yenum. This is allowed. Xenum is the tag name of the first enum, not a constant. You are allowed to use a previous enum's tag name as a constant in a new enum.

📝 Homework Assignment 1

enum Xenum { C, CPP, Java, Xenum };

Write this in your IDE and find out the output of sizeof(var).

📝 Question 2: Using typedef with Enums

The instructor now moves to the left side of the whiteboard, writing a new block of code utilizing the typedef keyword.

typedef enum {
    Male,
    Female
} gender;

int main() {
    gender var = Female;
    printf("%d", var);
    return 0;
}

🧠 Visualization: Variable Assignment

Just like in the previous example, the compiler assigns default integer values starting at 0.

Male 0
Female (Assigned to var) 1

Because var was assigned the constant Female, the printf statement will output 1.

📝 Homework Assignment 2

Look closely at the declaration in main():
gender var = Female;

Normally in C, you have to use the enum keyword to declare a variable (e.g., enum gender var = Female;). However, in this code, she omitted the word enum and it doesn't throw an error. Why?

Hint for your notes: The answer is because of the typedef keyword at the top. It creates a new type alias, allowing you to treat gender as a standard data type, bypassing the need to write enum every time!

💡 Key Takeaways & Cheat Sheet

Review these golden rules before your technical interview:

  • Size of an Enum: It is equal to the size of a standard integer (int) on your machine system (usually 4 bytes on modern 64-bit systems), regardless of how many constants are inside the brackets.
  • Default Integer Mapping: Unless you manually specify otherwise, the first constant in an enum is assigned 0, the next is 1, the next is 2, and so on.
  • Scope Clashing Rules: You cannot have two identical enum constants in the same scope. (e.g., You cannot have enum A { RED } and enum B { RED } in the same file block).
  • The Typedef Advantage: Using typedef enum { ... } name; allows you to declare variables simply using name var_name;. This keeps your code clean and removes the requirement to write the word enum repeatedly.
