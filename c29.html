Control Statements in C:
Nested if

Nesting means placing one thing completely inside another. A nested if statement is simply an if statement placed inside another if statement.

We use this when we need to check a secondary condition, but only if a primary condition has already been met. You can nest if statements as many times as you want.

🛒 The Supermarket Analogy

To understand why the inner condition is only checked if the outer is true, think about buying oranges. You cannot check if oranges are available unless you first go to the supermarket!

Outer: if (go to supermarket)
Inner 1: if (oranges are available)
Inner 2: if (good oranges) { BUY }

💻 General Syntax

Always ensure every opening curly brace { has a matching closing curly brace }. This is a common beginner mistake!

if (Condition 1) 
{
    // If Cond 1 is TRUE, control enters this block.
    
    if (Condition 2) 
    {
        // If Cond 1 AND Cond 2 are TRUE
        Inner if block statements;
    }
    else 
    {
        // If Cond 1 is TRUE, but Cond 2 is FALSE
        Inner else block statements;
    }
}
else 
{
    // If Cond 1 is FALSE, jump straight here.
    // Condition 2 is NEVER checked.
    Outer else block statements;
}

Statement X; // Executes no matter what

🗺️ Logic Flowchart

Here is how the computer's logic flows through a nested statement:

Condition 1
If TRUE: Enter Block
Condition 2
If TRUE: Inner IF Block
If FALSE: Inner ELSE Block

If Condition 1 is FALSE initially:

Skip entirely to Outer ELSE Block

⚙️ Practical Program: Salary Bonus

Let's calculate an employee's bonus based on two factors: Age and Salary.

  • Age > 50 & Salary < 60k ➔ Bonus is ₹10,000
  • Age > 50 & Salary ≥ 60k ➔ Bonus is ₹5,000
  • Age ≤ 50 (Any salary) ➔ Bonus is ₹3,000
void main() 
{
    int age, salary;
    
    printf("Enter age & salary: ");
    scanf("%d %d", &age, &salary);
    
    if (age > 50) 
    {
        if (salary < 60000) 
        {
            salary = salary + 10000;
        }
        else 
        {
            salary = salary + 5000;
        }
    }
    else 
    {
        salary = salary + 3000;
    }
    
    printf("Salary = %d\n", salary);
    printf("End of Program");
    getch();
}

🧠 Dry Run (Tracing the Code)

Let's act as the computer and test our program with three different user inputs.

Test Case 1

Input: 53, 45000

1. age > 50 (TRUE)
2. salary < 60k (TRUE)
3. 45k + 10k
> Salary = 55000
Test Case 2

Input: 53, 70000

1. age > 50 (TRUE)
2. salary < 60k (FALSE)
3. 70k + 5k (Inner Else)
> Salary = 75000
Test Case 3

Input: 45, 70000

1. age > 50 (FALSE)
2. Skip outer IF
3. 70k + 3k (Outer Else)
> Salary = 73000

⚡ Pro-Tip: Logical AND (&&)

Sometimes you don't need a nested if. If you just want an action to happen when both conditions are true, you can combine them using the Logical AND operator (&&).

// Instead of nesting:
if (age > 50 && salary < 60000)
{
    salary = salary + 10000;
}

Note: This is great for shortening code, but if you need to perform different actions for different combinations of True/False (like giving a 5k bonus vs a 10k bonus), nested if-else blocks are still required.

⚠️ Common Beginner Mistake

Do NOT place a semicolon immediately after the if() condition.

if (age > 50); <-- FATAL ERROR
{
    // some code
}
else
{
    // some code
}

Why it fails: The semicolon terminates the statement. The computer thinks the if statement is finished right there. When it reaches the else, it crashes with an "Else without a previous if" error.
